# 偏差-方差分解

## 一句话定义

偏差-方差分解是将机器学习模型的泛化误差拆解为偏差、方差和噪声三个组成部分的数学框架，用于诊断模型是过拟合还是欠拟合。

## 核心公式

对于回归任务，泛化误差可以分解为：

$$
\mathbb{E}[(y - \hat{f}(x))^2] = \underbrace{\text{Bias}^2[\hat{f}(x)]}_{\text{偏差}}^2 + \underbrace{\text{Var}[\hat{f}(x)]}_{\text{方差}} + \underbrace{\sigma^2}_{\text{噪声}}
$$

其中：
- **偏差** $\text{Bias}[\hat{f}(x)] = \mathbb{E}[\hat{f}(x)] - f(x)$，衡量模型预测值的期望与真实值的差距
- **方差** $\text{Var}[\hat{f}(x)] = \mathbb{E}[\hat{f}(x)^2] - \mathbb{E}[\hat{f}(x)]^2$，衡量模型预测值的变化幅度
- **噪声** $\sigma^2 = \text{Var}[y]$，数据本身固有的不可约误差

## 详细说明

### 1. 偏差（Bias）

- **定义**：模型预测值的期望与真实值之间的差异
- **来源**：模型假设空间的限制、算法本身的归纳偏置
- **特点**：
  - 高偏差 → 欠拟合（模型太简单，无法捕捉数据模式）
  - 低偏差 → 模型复杂度足够

### 2. 方差（Variance）

- **定义**：模型预测值围绕其期望的波动程度
- **来源**：训练数据的随机性、模型的敏感性
- **特点**：
  - 高方差 → 过拟合（模型太复杂，对训练数据过度敏感）
  - 低方差 → 模型稳定，对训练数据变化不敏感

### 3. 噪声（Noise）

- **定义**：数据本身固有的随机误差，不可消除
- **来源**：测量误差、数据采集过程中的不确定性
- **特点**：
  - 是误差的下界，无法通过改进模型来降低

### 4. 偏差-方差权衡

```
泛化误差
  │
  │   ╲
  │    ╲  方差主导区
  │     ╲───────
  │      ╲
  │       ╲
  │        ╲
  │         ╲ 偏差主导区
  │          ╲────
  └──────────────────→ 模型复杂度
     欠拟合    最优点    过拟合
```

- **简单模型**：高偏差、低方差
- **复杂模型**：低偏差、高方差
- **最佳模型**：在偏差和方差之间找到平衡点

## 代码示例

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline

def bias_variance_simulation(n_samples=100, n_iterations=100, true_degree=3):
    """
    模拟偏差-方差分解实验
    """
    np.random.seed(42)

    # 真实函数：y = sin(2*pi*x) + 噪声
    true_func = lambda x: np.sin(2 * np.pi * x)

    biases = []
    variances = []
    errors = []
    degrees = range(1, 15)

    for degree in degrees:
        test_preds = []

        for _ in range(n_iterations):
            # 生成训练数据和测试数据
            X_train = np.random.uniform(0, 1, n_samples)
            y_train = true_func(X_train) + np.random.normal(0, 0.3, n_samples)
            X_test = np.random.uniform(0, 1, 100)
            y_test = true_func(X_test)

            # 训练模型
            model = Pipeline([
                ('poly', PolynomialFeatures(degree=degree)),
                ('lr', LinearRegression())
            ])
            model.fit(X_train.reshape(-1, 1), y_train)

            # 预测
            y_pred = model.predict(X_test.reshape(-1, 1))
            test_preds.append(y_pred)

        test_preds = np.array(test_preds)
        mean_pred = np.mean(test_preds, axis=0)
        bias = np.mean((mean_pred - y_test) ** 2)
        variance = np.mean(np.var(test_preds, axis=0))
        error = np.mean((test_preds - y_test) ** 2)

        biases.append(bias)
        variances.append(variance)
        errors.append(error)

    return degrees, biases, variances, errors

# 运行模拟
degrees, biases, variances, errors = bias_variance_simulation()

# 绘制结果
plt.figure(figsize=(10, 6))
plt.plot(degrees, biases, 'b-', label='Bias²', linewidth=2)
plt.plot(degrees, variances, 'r-', label='Variance', linewidth=2)
plt.plot(degrees, [b + v + 0.09 for b, v in zip(biases, variances)], 'g--', label='Bias² + Variance', linewidth=2)
plt.xlabel('Model Complexity (Polynomial Degree)')
plt.ylabel('Error')
plt.legend()
plt.title('Bias-Variance Decomposition')
plt.grid(True, alpha=0.3)
plt.show()
```

## 应用场景

### 1. 模型选择

在选择算法或调参时，通过偏差-方差分析判断：
- 线性模型：高偏差、低方差，适合线性可分数据
- 决策树：高方差、低偏差，容易过拟合
- 随机森林：通过集成降低方差

### 2. 集成学习设计

| 方法 | 降低 | 示例 |
|------|------|------|
| Bagging | 方差 | Random Forest |
| Boosting | 偏差 | AdaBoost, Gradient Boosting |
| Stacking | 偏差+方差 | Stacking Classifier |

### 3. 超参数调优

- **正则化参数**：控制模型复杂度，平衡偏差-方差
- **树深度**：深度越大，方差越高
- **学习率**：影响模型收敛的稳定性

## 相关概念

- **过拟合（Overfitting）**：模型在训练集上表现很好，但在测试集上表现差，方差过高
- **欠拟合（Underfitting）**：模型在训练集和测试集上都表现不好，偏差过高
- **泛化误差（Generalization Error）**：模型在新数据上的预期误差
- **正则化（Regularization）**：通过添加惩罚项降低模型复杂度，控制方差
- **交叉验证（Cross Validation）**：评估模型偏差-方差特性的常用方法
