# 逆向强化学习（Inverse Reinforcement Learning, IRL）

## 一句话定义

> 逆向强化学习是从专家演示的轨迹中反推潜在的奖励函数的过程，使得基于该奖励函数的最优策略与专家行为一致。

## 核心公式

### 1. 最大边际 IRL（Max-Margin IRL）

$$\max_{r_\psi, \pi} \quad \sum_{s} \Pr(\text{expert } | s) - \lambda \cdot \text{reg}(r_\psi)$$

约束条件：

$$r_\psi(s) \cdot (\mathbb{E}_{\pi^{expert}}[G(s)] - \mathbb{E}_{\pi}[G(s)]) \geq \Delta(s, \pi)$$

其中 $G(s)$ 为累积回报，$\Delta$ 为边际间隔。

### 2. 能量模型（Energy-Based Model）

将奖励视为轨迹的能量函数：

$$E(\tau) = \sum_{t=0}^{T} r(s_t, a_t)$$

$$p(\tau | \text{expert}) = \frac{e^{-\beta E(\tau)}}{Z}$$

配分函数：

$$Z = \int e^{-\beta E(\tau)} d\tau$$

### 3. GPT-4 从人类反馈中学习（RLHF）

人类偏好损失：

$$\mathcal{L}_{RLHF} = -\mathbb{E}_{(x, y_w, y_l) \sim D}[ \log \sigma( r_\psi(x, y_w) - r_\psi(x, y_l) ) ]$$

其中 $y_w$ 是人类偏好的回答，$y_l$ 是不偏好的回答。

## 详细说明

### 1. 问题背景

标准强化学习：给定奖励函数 $r(s,a)$，寻找最优策略 $\pi^*$。

逆向强化学习：给定专家演示 $\mathcal{D} = \{\tau_1, \tau_2, ...\}$，推断奖励函数 $r(s,a)$。

**为什么重要**：
- 奖励函数设计困难：人类难以显式描述什么是"好"行为
- 同一行为可能由不同奖励函数驱动
- 推断出的奖励函数可用于解释和迁移

### 2. 最大边际 IRL（Max-Margin IRL）

**原理**：寻找一个奖励函数，使得专家轨迹的累积回报至少比任何其他轨迹多一个边际值。

**优化目标**：
$$\min_{r} \quad \frac{1}{2} \|r\|^2 + C \sum_i \xi_i$$

约束：
$$r \cdot (\text{feature}(\tau^{expert}) - \text{feature}(\tau_i)) \geq \Delta(\tau_i) - \xi_i$$

**优点**：理论清晰，边际约束提供可解释性。

**缺点**：当专家轨迹不唯一时可能失败；边际间隔选择敏感。

### 3. 能量模型 IRL

**核心思想**：将轨迹的奖励视为"负能量"，专家轨迹具有最低能量。

$$p(\tau | expert) = \frac{e^{-\beta \cdot r(\tau)}}{\int e^{-\beta \cdot r(\tau)} d\tau}$$

通过对比学习框架训练：

$$\mathcal{L} = -\log \frac{e^{-\beta r(\tau_{expert})}}{e^{-\beta r(\tau_{expert})} + \sum_{i} e^{-\beta r(\tau_i)}}$$

### 4. GPT-4/RLHF（从人类反馈中学习）

**背景**：大规模语言模型（如 GPT-4）的奖励函数难以人工设计。

**三阶段流程**：
1. **监督微调（SFT）**：用人类撰写的高质量回答微调模型
2. **奖励模型训练**：收集人类偏好对，训练奖励模型 $r_\psi(x, y)$
3. **强化学习优化**：用 PPO 算法最大化奖励模型给出的偏好信号

**代码示例（简化版）**：

```python
import torch
import torch.nn as nn

class RewardModel(nn.Module):
    def __init__(self, state_dim, hidden_dim=256):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1)  # 标量奖励
        )

    def forward(self, state, action):
        # 状态-动作对的奖励
        concat = torch.cat([state, action], dim=-1)
        return self.network(concat)

# RLHF 中的奖励模型训练
def train_reward_model(reward_model, preferred_pairs, rejected_pairs):
    """
    preferred_pairs: [(state, preferred_action), ...]
    rejected_pairs: [(state, rejected_action), ...]
    """
    optimizer = torch.optim.Adam(reward_model.parameters(), lr=1e-4)

    for epoch in range(100):
        total_loss = 0
        for (s, a_w), (s, a_l) in zip(preferred_pairs, rejected_pairs):
            r_w = reward_model(s, a_w)
            r_l = reward_model(s, a_l)
            # 偏好损失：偏好的奖励应高于拒绝的
            loss = -torch.log(torch.sigmoid(r_w - r_l))
            total_loss += loss

        optimizer.zero_grad()
        total_loss.backward()
        optimizer.step()

# PPO 阶段使用奖励模型
def ppo_with_reward(env, policy, reward_model, num_episodes=1000):
    for episode in range(num_episodes):
        trajectory = collect_trajectory(env, policy)
        rewards = [reward_model(s, a) for (s, a) in trajectory]
        update_policy_ppo(trajectory, rewards, policy)
```

### 5. 深度 IRL

用深度神经网络表示奖励函数：

$$r_\psi(s, a) = f_\psi(s, a; \theta)$$

适用场景：高维状态空间（图像、视频）。

## 相关概念

- **模仿学习（Behavior Cloning）**：直接复制专家行为，不推断奖励函数
- **逆向强化学习**：从轨迹推断奖励函数
- **RLHF**：使用人类反馈作为奖励信号
- **最大边际方法**：基于间隔的 IRL
- **能量模型**：基于统计物理的 IRL 框架
- **大语言模型对齐**：RLHF 在 LLM 中的应用

## 常见问题

**Q：IRL 的解是唯一的吗？**
A：否。IRL 本质上是不适定的（ill-posed）——多个不同的奖励函数可能产生相同的专家行为。这被称为"奖励等价性"问题。

**Q：IRL 和模仿学习的区别是什么？**
A：IRL 推断奖励函数，可用于解释专家行为、迁移到新环境；模仿学习直接学习策略，无需推断奖励。IRL 可视为更"解耦"的表示。

**Q：RLHF 和传统 IRL 有什么关系？**
A：RLHF 是 IRL 的一个实例——用人类偏好作为"演示"，训练一个奖励模型，然后通过 RL 优化该奖励模型。

**Q：IRL 的计算复杂度如何？**
A：IRL 需要在内部循环中解决强化学习问题（正向问题），因此计算代价很高。深度 IRL 尤其如此，需要结合函数近似和高效 RL 算法。
