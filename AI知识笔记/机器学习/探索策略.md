# 探索策略（Exploration Strategies）

## 一句话定义

> 探索策略是强化学习中解决"探索-利用困境"的方法——智能体需要在探索未知以发现更高回报的动作（探索）与利用当前已知最优动作（利用）之间取得平衡。

## 核心公式

### 1. Epsilon-Greedy

$$\pi(a|s) = \begin{cases} \arg\max_a Q(s, a) & \text{with probability } 1 - \epsilon \\ \text{random action} & \text{with probability } \epsilon \end{cases}$$

### 2. Upper Confidence Bound（UCB）

$$a_t = \arg\max_a \left[ Q(s, a) + c \sqrt{\frac{\ln t}{N(s, a)}} \right]$$

其中 $N(s, a)$ 是状态-动作对的访问次数，$c$ 是置信度参数。

### 3. 熵正则化（Entropy Regularization）

$$\mathcal{L}(\theta) = \mathbb{E}_{\pi_\theta}[\sum_t r(s_t, a_t)] + \alpha \cdot \mathbb{H}(\pi_\theta)$$

$$\mathbb{H}(\pi_\theta) = -\mathbb{E}_{\pi_\theta}[\log \pi_\theta(a|s)]$$

### 4. 内在激励机制（Intrinsic Motivation）

$$r^{total}_t = r^{ext}_t + \beta \cdot r^{intrinsic}_t$$

**基于计数**：
$$r^{intrinsic}_t = \frac{1}{\sqrt{N(s_t)} + c}$$

**基于预测误差（Curiosity）**：
$$r^{intrinsic}_t = \|f(s_t, a_t; \phi) - s_{t+1}\|^2$$

其中 $f$ 是前向动力学模型，$\phi$ 是其参数。

## 详细说明

### 1. 问题背景

**探索-利用困境（Exploration-Exploitation Dilemma）**：

- **利用**：选择当前 $Q$ 值最高的动作，最大化即时回报
- **探索**：尝试新动作，可能发现更高长期回报

盲目利用会导致局部最优；盲目探索会浪费资源。平衡二者是强化学习的核心挑战。

### 2. Epsilon-Greedy

**原理**：以 $\epsilon$ 概率随机选择动作，$1-\epsilon$ 概率选择当前最优动作。

**代码示例**：

```python
import numpy as np
import random

class EpsilonGreedy:
    def __init__(self, n_actions, epsilon=0.1):
        self.n_actions = n_actions
        self.epsilon = epsilon
        self.Q = np.zeros(n_actions)  # Q值估计
        self.N = np.zeros(n_actions)  # 访问计数

    def select_action(self):
        if random.random() < self.epsilon:
            return random.randint(0, self.n_actions - 1)
        else:
            return int(np.argmax(self.Q))

    def update(self, action, reward, alpha=0.1):
        self.N[action] += 1
        # 增量式更新
        self.Q[action] += alpha * (reward - self.Q[action])

# 自适应 epsilon：随时间衰减
def decay_epsilon(epsilon, episode, decay_rate=0.99, min_epsilon=0.01):
    return max(min_epsilon, epsilon * decay_rate ** episode)
```

**优点**：简单易实现。

**缺点**：对所有非最优动作平等对待，不考虑探索价值。

### 3. Softmax / Boltzmann 探索

根据动作的 $Q$ 值概率选择：

$$P(a|s) = \frac{e^{Q(s,a)/\tau}}{\sum_{a'} e^{Q(s,a')/\tau}}$$

$\tau$ 为温度参数：$\tau \to 0$ 趋向贪婪，$\tau \to \infty$ 趋向均匀。

```python
def softmax_select_action(Q, tau=1.0):
    exp_Q = np.exp(Q / tau)
    probs = exp_Q / np.sum(exp_Q)
    return np.random.choice(len(Q), p=probs)
```

### 4. Upper Confidence Bound（UCB）

**原理**：平衡 $Q$ 值估计和估计的不确定性。

$$a_t = \arg\max_a \left[ Q(s, a) + c \sqrt{\frac{\ln t}{N(s, a)}} \right]$$

**置信度项** $\sqrt{\frac{\ln t}{N(s,a)}}$：
- 访问次数越少，不确定性越大，探索优先
- 随时间增长，不确定性项贡献减小

**代码示例**：

```python
import math

class UCB1:
    def __init__(self, n_actions, c=2.0):
        self.n_actions = n_actions
        self.c = c
        self.Q = np.zeros(n_actions)
        self.N = np.zeros(n_actions)
        self.t = 0

    def select_action(self):
        self.t += 1
        # 确保每个动作至少被选择一次
        if 0 in self.N:
            return int(np.argmin(self.N))

        ucb_values = self.Q + self.c * np.sqrt(
            math.log(self.t) / self.N
        )
        return int(np.argmax(ucb_values))

    def update(self, action, reward, alpha=0.1):
        self.N[action] += 1
        self.Q[action] += alpha * (reward - self.Q[action])
```

### 5. 熵正则化（Entropy Regularization）

**原理**：在奖励中加入策略熵项，鼓励探索。

$$\mathcal{L} = \sum_t r(s_t, a_t) + \alpha \cdot \mathbb{H}(\pi_\theta)$$

**效果**：
- 防止策略过早收敛到确定性策略
- 自然地平衡探索与利用
- 等价于最大熵强化学习（MaxEnt RL）

**代码示例**（结合策略梯度）：

```python
class EntropyRegularizedPolicy(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim)
        )
        self.entropy_coef = 0.01

    def forward(self, state):
        logits = self.network(state)
        return F.softmax(logits, dim=-1)

    def loss(self, state, action, advantage):
        probs = self.forward(state)
        # 策略梯度损失
        log_probs = torch.log(probs + 1e-8)
        policy_loss = -(log_probs[range(len(action)), action] * advantage).mean()

        # 熵正则化
        entropy = -(probs * log_probs).sum(dim=-1).mean()
        total_loss = policy_loss - self.entropy_coef * entropy

        return total_loss, entropy
```

### 6. 内在激励机制（Intrinsic Motivation）

**核心思想**：为智能体提供额外的"内在奖励"，激励其探索新状态。

#### 6.1 基于计数的探索

利用访问计数来奖励稀有状态：

$$r^{intrinsic}_t = \frac{1}{\sqrt{N(s_t)} + c}$$

**缺点**：在高维/连续状态空间中不实际。

#### 6.2 基于预测误差的探索（Curiosity-Driven）

$$r^{intrinsic}_t = \|f(s_t, a_t; \phi) - s_{t+1}\|^2$$

$f$ 是前向动力学模型，预测下一个状态。若预测误差大，说明该状态是"意外的"，值得探索。

```python
class CuriosityModule(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=128):
        super().__init__()
        # 前向动力学模型
        self.forward_model = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, state_dim)
        )
        self.optimizer = torch.optim.Adam(self.parameters(), lr=1e-3)

    def compute_intrinsic_reward(self, state, action, next_state):
        pred_next_state = self.forward_model(
            torch.cat([state, action], dim=-1)
        )
        reward = F.mse_loss(pred_next_state, next_state, reduction='none').mean(dim=-1)
        return reward

    def update(self, state, action, next_state):
        self.optimizer.zero_grad()
        reward = self.compute_intrinsic_reward(state, action, next_state)
        (-reward).backward()  # 最大化好奇奖励
        self.optimizer.step()
```

#### 6.3 基于随机网络蒸馏（Random Network Distillation, RND）

$$r^{intrinsic}_t = \|\hat{f}(s_t; \phi) - f(s_t)\|^2$$

使用随机初始化的固定网络作为目标，用可学习网络预测，减少对环境动力学的依赖。

### 7. 深度强化学习中的探索

| 方法 | 描述 | 适用场景 |
|------|------|----------|
| DQN + 经验回放 | $\epsilon$-贪心平衡探索 | 离散动作空间 |
| DQN + Double Q | 减少 Q 值过估计 | 高估问题 |
| A3C + 熵正则化 | 策略网络输出熵作为辅助奖励 | 连续动作空间 |
| PPO + CLIP | 限制策略更新幅度，间接控制探索 | 稳定训练 |
| SAC | 最大熵 RL，自动调整探索 | 连续控制 |
| RND | 随机网络蒸馏 | 稀疏奖励环境 |
| ICM | 内在好奇心模块 | 3D 环境探索 |

## 常见问题

**Q：固定 epsilon 值有什么问题？**
A：固定 $\epsilon$ 无法平衡探索与利用的动态需求。通常使用衰减 epsilon（随训练进行逐渐减少探索）。

**Q：UCB 为什么用对数项？**
A：对数项保证置信度上界随访问次数缓慢增长，使 UCB 能持续探索但逐渐收敛到最优动作。理论上有最优后悔界。

**Q：熵正则化和内在奖励有什么区别？**
A：熵正则化在单步策略优化中加入熵项，隐式鼓励多样性；内在奖励显式奖励"新奇"状态。两者可结合使用。

**Q：在连续动作空间中如何探索？**
A：可用重参数化技巧（SAC）、基于均值-方差的高斯策略、或直接在高维动作空间应用 ICM。

**Q：好奇心驱动探索的局限性？**
A：预测误差可能被环境中无关的噪声主导（如光照变化），需要结合像素级对比学习或特征空间预测。
