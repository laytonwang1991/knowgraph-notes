# 模仿学习（Imitation Learning）

## 一句话定义

> 模仿学习是一种通过观察专家演示来学习策略的机器学习范式，智能体试图直接复制专家在特定状态下的行为，而非通过试错和奖励信号来学习。

## 核心公式

### 1. 行为克隆（Behavior Cloning）

$$\pi_\theta(a|s) = \mathbb{E}_{(s,a) \sim D^{expert}}[a|s]$$

通过监督学习拟合专家策略，损失函数为：

$$\mathcal{L}(\theta) = -\mathbb{E}_{(s,a) \sim D^{expert}}[\log \pi_\theta(a|s)]$$

### 2. DAgger（Dataset Aggregation）

$$D_{i+1} = D_i \cup \{(s, \pi^{expert}(s)) : s \sim \pi_{\theta_i}\}$$

迭代式收集智能体访问的状态，由专家标注对应动作。

### 3. 逆向强化学习损失（基于能量模型）

$$\mathcal{L}_{IRL} = -\mathbb{E}_{(s,a) \sim D^{expert}}[f_\psi(s,a)] + \log \sum_{a'} e^{f_\psi(s,a')}$$

## 详细说明

### 1. 问题背景

在许多真实场景中，设计有效的奖励函数极其困难：

- 自动驾驶：如何量化"安全驾驶"？
- 机器人操作：如何定义"成功抓取"？
- 手术机器人：如何给每个动作打分？

模仿学习通过直接学习专家演示来规避奖励工程问题。

### 2. Behavior Cloning（行为克隆）

**原理**：将模仿学习视为标准的监督学习问题。

**优点**：
- 简单直接，实现容易
- 收敛速度快
- 理论基础完善

**缺点**：
- 分布偏移问题：智能体的策略偏差会导致到达未见状态
- 因果混淆：专家动作可能是对不可观察因素的响应
- 无法超越专家

**代码示例**：

```python
import torch
import torch.nn as nn

class BehaviorCloning(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim)
        )

    def forward(self, state):
        return self.network(state)

    def loss(self, state, expert_action):
        pred_action = self.forward(state)
        return nn.MSELoss()(pred_action, expert_action)

# 训练循环
model = BehaviorCloning(state_dim=64, action_dim=8)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
expert_data = load_expert_demonstrations()

for epoch in range(100):
    for state, action in expert_data:
        loss = model.loss(state, action)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

### 3. DAgger（Dataset Aggregation）

**原理**：迭代式收集智能体策略访问的状态，由专家标注后加入训练集。

**算法流程**：
1. 用当前策略 $\pi_i$ 收集轨迹
2. 对轨迹中的每个状态询问专家获取最优动作
3. 将新数据加入训练集
4. 用聚合数据集训练新策略

**关键洞察**：解决了行为克隆的分布偏移问题。

**代码示例**：

```python
def dagger_loop(policy, expert, env, n_iterations=10):
    dataset = []
    policy_clone = clone_policy(policy)

    for i in range(n_iterations):
        # 1. 用当前策略收集轨迹
        trajectories = collect_trajectories(env, policy_clone)

        # 2. 对每个状态请求专家动作
        for state in extract_states(trajectories):
            expert_action = expert.get_action(state)
            dataset.append((state, expert_action))

        # 3. 用聚合数据训练
        policy_clone = train_on_dataset(dataset)

    return policy_clone
```

### 4. 基于混沌探索的模仿学习

通过随机化的初始状态和扰动动作来扩展专家数据的覆盖范围。

## 相关概念

- **逆向强化学习（IRL）**：从专家轨迹中推断潜在的奖励函数
- **强化学习**：基于奖励信号通过试错学习策略
- **策略梯度**：直接优化策略参数的方法
- **分布偏移**：训练分布与测试分布不一致导致性能下降

## 常见问题

**Q：模仿学习为什么不能超越专家？**
A：模仿学习本质上是回归问题，它学习的是在专家数据覆盖的状态-动作空间中的条件分布。若某个状态从未被专家访问，智能体将无法学到有意义的动作。

**Q：如何评估模仿学习的效果？**
A：通常用专家评估（EL）——让专家观看智能体轨迹并打分；或用成功率、任务完成度等任务特定指标。

**Q：演示数据不足怎么办？**
A：可结合数据增强、DAgger迭代收集、迁移学习等技术。
