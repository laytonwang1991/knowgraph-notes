# 工具调用 (Tool Use / Function Calling)

## 一句话定义

工具调用是 LLM 与外部世界交互的核心能力，使模型能够主动调用外部工具、API 和函数来获取信息、执行操作或完成复杂任务。

## 核心公式/技术要点

### Function Calling 标准格式
```json
{
  "name": "function_name",
  "description": "函数功能描述",
  "parameters": {
    "type": "object",
    "properties": {
      "param1": {"type": "string", "description": "参数描述"},
      "param2": {"type": "number", "description": "参数描述"}
    },
    "required": ["param1"]
  }
}
```

### OpenAI Function Calling 流程
```
用户请求 → LLM判断是否需要工具 → 输出function_call → 执行工具 → 返回结果 → LLM整合回答
```

### 多工具选择
$$P(\text{tool}_k | \text{context}) = \text{softmax}( \text{LLM}(\text{context}, \text{tool}_k) )$$

## 详细说明

### 1. 工具调用原理

#### 传统 LLM 局限
- 知识截止日期
- 无法访问实时数据
- 无法执行外部操作

#### 工具调用扩展
- **RAG (检索增强生成)**: 获取最新知识
- **API 调用**: 执行外部操作
- **代码执行**: 复杂计算和处理

### 2. 主流实现

#### OpenAI Function Calling
```python
import openai

messages = [{"role": "user", "content": "今天北京的天气怎么样？"}]

functions = [
    {
        "name": "get_weather",
        "description": "获取指定城市的天气信息",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "城市名称"},
                "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
            },
            "required": ["city"]
        }
    }
]

response = openai.ChatCompletion.create(
    model="gpt-4",
    messages=messages,
    functions=functions,
    function_call="auto"
)

# 解析函数调用
function_call = response.choices[0].message.function_call
if function_call:
    result = execute_function(function_call)
```

#### Anthropic Tool Use
```python
from anthropic import Anthropic

client = Anthropic()

response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    messages=[{"role": "user", "content": "帮我查询今天的汇率"}],
    tools=[{
        "name": "get_exchange_rate",
        "description": "获取货币汇率",
        "input_schema": {
            "type": "object",
            "properties": {
                "from_currency": {"type": "string"},
                "to_currency": {"type": "string"}
            }
        }
    }]
)
```

#### Google Gemini Function Calling
```python
import google.generativeai as genai

model = genai.GenerativeModel('gemini-pro')

tools = [
    {"name": "get_weather", "description": "获取天气", "parameters": {...}}
]

response = model.generate_content(
    prompt,
    tools=tools,
    tool_config={"function_calling": "auto"}
)
```

### 3. 工具类型分类

| 类型 | 示例 | 用途 |
|------|------|------|
| **信息检索** | Web Search, Wikipedia | 获取实时信息 |
| **计算** | Calculator, Python REPL | 数学运算 |
| **数据查询** | SQL, API | 数据库操作 |
| **文件操作** | Read, Write, Upload | 文件管理 |
| **代码执行** | Code Interpreter | 运行代码 |
| **业务操作** | Email, Calendar | 自动化工作流 |

### 4. 工具调用设计模式

#### 单一工具调用
最简单的模式，一次调用一个工具。

#### 并行工具调用
同时调用多个独立工具。

```python
# 伪代码示例
parallel_calls = [
    tool1.call(args1),
    tool2.call(args2),
    tool3.call(args3)
]
results = await asyncio.gather(*parallel_calls)
```

#### 顺序工具调用
工具之间存在依赖，按顺序执行。

```python
result1 = await tool1.call(args1)
result2 = await tool2.call(result1)  # 依赖 result1
result3 = await tool3.call(result2)  # 依赖 result2
```

### 5. 工具描述 (Tool Description)

高质量的工具描述对效果至关重要：

```json
{
  "name": "search_flights",
  "description": "搜索航班信息。输入出发地、目的地和日期，返回可用航班列表。",
  "parameters": {
    "type": "object",
    "properties": {
      "origin": {
        "type": "string",
        "description": "出发城市代码，如PEK表示北京"
      },
      "destination": {
        "type": "string",
        "description": "目的城市代码，如SHA表示上海"
      },
      "date": {
        "type": "string",
        "description": "出发日期，格式YYYY-MM-DD"
      }
    },
    "required": ["origin", "destination", "date"]
  }
}
```

### 6. 工具调用挑战

| 挑战 | 解决方案 |
|------|----------|
| 幻觉 (Hallucination) | 严格的参数验证 |
| 调用失败 | 重试机制 + 错误处理 |
| 循环调用 | 最大调用次数限制 |
| 上下文长度 | 精简工具返回结果 |
| 安全风险 | 权限控制和审计 |

### 7. MCP (Model Context Protocol)

Anthropic 提出的工具调用标准协议：

```json
// MCP Server 配置
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-server-filesystem", "./data"]
    },
    "weather": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-server-weather"]
    }
  }
}
```
