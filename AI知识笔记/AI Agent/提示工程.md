# 提示工程

## 一句话定义

提示工程是通过精心设计输入文本的结构和内容，引导大语言模型产生期望输出的技术，包括指令设计、示例选择、上下文构建等核心要素。

## 核心公式

### 提示词基本结构

$$
\text{Prompt} = \underbrace{[\text{系统指令}]}_{\text{System}} + \underbrace{[\text{任务描述}]}_{\text{Task}} + \underbrace{[\text{上下文信息}]}_{\text{Context}} + \underbrace{[\text{输入数据}]}_{\text{Input}} + \underbrace{[\text{输出格式}]}_{\text{Format}}
$$

### 零样本提示

$$
P(\text{output} | \text{task}, \text{instruction}) = \text{LLM}(\text{instruction} + \text{task})
$$

### 少样本提示

$$
P(\text{output} | \text{task}, \{(\text{in}_i, \text{out}_i)\}_{i=1}^{k}) = \text{LLM}(\text{instruction} + \{(\text{in}_i, \text{out}_i)\}_{i=1}^{k} + \text{task})
$$

### 思维链提示

$$
\text{Chain-of-Thought} = \text{input} \rightarrow \underbrace{\text{reasoning step 1} \rightarrow \text{reasoning step 2} \rightarrow \cdots}_{\text{hidden thought chain}} \rightarrow \text{output}
$$

## 详细说明

### 1. 提示词设计基础

#### 1.1 清晰明确的指令

```python
# 不好的提示
prompt = "翻译一下"

# 好的提示
prompt = """将以下中文文本翻译成英文。
要求：
1. 保持专业术语的准确性
2. 符合英文表达习惯
3. 输出格式为 JSON: {"original": "...", "translated": "..."}

文本：人工智能正在改变世界。
"""
```

#### 1.2 结构化输出

```python
# 使用结构化格式指定输出
prompt = """分析以下产品评论，返回JSON格式的情感分析结果。

评论: "这个产品超出了我的预期，质量很好，但包装有点简陋。"

输出格式:
{
    "sentiment": "positive|negative|neutral",
    "score": 0.0到1.0之间的分数,
    "pros": ["优点列表"],
    "cons": ["缺点列表"],
    "summary": "一句话总结"
}
"""
```

### 2. 零样本提示 (Zero-shot)

不需要任何示例，直接通过指令引导模型：

```python
def zero_shot_prompt(task: str, instruction: str) -> str:
    """零样本提示模板"""
    return f"""指令: {instruction}

任务: {task}

请按照指令完成上述任务。"""

# 示例
prompt = zero_shot_prompt(
    task="将'the quick brown fox'改为大写",
    instruction="只返回转换后的结果，不要其他解释"
)
# 输出: THE QUICK BROWN FOX
```

### 3. 少样本提示 (Few-shot)

提供少量示例帮助模型理解任务：

```python
def few_shot_prompt(task: str, examples: List[dict], input: str) -> str:
    """少样本提示模板"""
    example_str = "\n\n".join([
        f"输入: {ex['input']}\n输出: {ex['output']}"
        for ex in examples
    ])
    return f"""以下是完成任务的示例:

{example_str}

现在请完成新任务:

输入: {input}
输出:"""

# 示例：情感分类
prompt = few_shot_prompt(
    task="判断文本的情感",
    examples=[
        {"input": "太好吃了！", "output": "positive"},
        {"input": "很失望", "output": "negative"},
        {"input": "还行吧", "output": "neutral"},
    ],
    input="今天运气不错"
)
```

### 4. 思维链提示 (Chain-of-Thought)

引导模型展示推理过程：

```python
def cot_prompt(question: str) -> str:
    """思维链提示"""
    return f"""问题: {question}

请逐步思考这个问题，展示你的推理过程（Think aloud），
最后给出答案。

推理过程:"""

# 或使用自洽性思维链
def self_consistency_cot(question: str, n_samples: int = 5) -> str:
    """自洽性思维链：生成多个推理路径，取最一致的答案"""
    return f"""问题: {question}

请从多个角度思考这个问题，生成{n_samples}种不同的推理路径，
然后比较这些推理，得出最一致的答案。
"""
```

### 5. 上下文学习 (In-Context Learning)

通过在提示中提供相关上下文来提升效果：

```python
class InContextLearning:
    def __init__(self, retriever):
        self.retriever = retriever

    def build_prompt(self, query: str, top_k: int = 3) -> str:
        # 检索相关上下文
        context_docs = self.retriever.search(query, top_k=top_k)

        prompt = f"""基于以下上下文信息回答问题。

上下文:
{self.format_context(context_docs)}

问题: {query}

回答:"""
        return prompt

    def format_context(self, docs: List[Document]) -> str:
        return "\n\n".join([
            f"[文档{i+1}] {doc.content}"
            for i, doc in enumerate(docs)
        ])
```

### 6. 提示工程最佳实践

| 原则 | 做法 | 禁忌 |
|------|------|------|
| 清晰性 | 使用具体明确的指令 | 模糊、歧义的表述 |
| 结构化 | 分点列举、编号列表 | 冗长连段落 |
| 角色设定 | "你是一位资深工程师" | 不设定角色随意发挥 |
| 格式指定 | 明确输出JSON/markdown | 让模型自由发挥 |
| 分解任务 | 分步骤引导 | 一次性复杂要求 |
| 验证输出 | 要求自检或解释 | 盲目接受结果 |

```python
# 综合示例：构建生产级提示词
def build_production_prompt(
    task: str,
    context: str,
    examples: List[dict] = None,
    output_format: str = "text",
    constraints: List[str] = None
) -> str:
    """生产级提示词构建器"""

    parts = []

    # 1. 系统指令
    parts.append("你是一位专业的AI助手，擅长分析和回答问题。")

    # 2. 任务描述
    parts.append(f"## 任务\n{task}")

    # 3. 上下文
    if context:
        parts.append(f"## 上下文\n{context}")

    # 4. 示例
    if examples:
        parts.append("## 示例")
        for ex in examples:
            parts.append(f"- 输入: {ex['input']}")
            parts.append(f"  输出: {ex['output']}")

    # 5. 约束条件
    if constraints:
        parts.append("## 约束\n" + "\n".join([f"- {c}" for c in constraints]))

    # 6. 输出格式
    if output_format == "json":
        parts.append("## 输出格式\n请返回有效的JSON格式，不要包含其他文字。")
    elif output_format == "markdown":
        parts.append("## 输出格式\n使用Markdown格式输出。")

    return "\n\n".join(parts)
```

## 应用场景

1. **文本分类**：情感分析、垃圾邮件检测、主题分类
2. **信息抽取**：命名实体识别、关系抽取、事件提取
3. **文本生成**：文案创作、摘要生成、翻译
4. **问答系统**：阅读理解、知识问答、对话系统
5. **代码任务**：代码生成、代码审查、bug修复
