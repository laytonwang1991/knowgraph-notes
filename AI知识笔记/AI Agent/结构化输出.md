# 结构化输出

## 一句话定义

结构化输出是指通过特定技术和约束，引导大语言模型（LLM）生成符合预定义 schema 的格式化内容，而非自由文本。

## 详细说明

### 1. JSON Mode

JSON Mode 是让 LLM 直接输出合法 JSON 字符串的技术。主流 API（如 OpenAI、Anthropic）提供专门的 `response_format` 参数。

**核心特性：**
- 强制输出合法的 JSON 格式
- 需要在 prompt 中明确说明期望的 JSON 结构
- 可以结合 schema 定义严格约束输出

### 2. Regex 约束

通过正则表达式对输出进行结构约束，引导模型生成特定格式。

**常用场景：**
- 提取特定格式的数据（如手机号、邮箱）
- 限制输出的字符范围
- 引导模型使用特定分隔符

### 3. Grammar Sampling

通过定义上下文无关文法（CFG）或类似形式语法，约束输出的句法结构。

**实现方式：**
- 定义语法规则（如 BNF 形式）
- 在采样过程中根据语法进行剪枝
- 保证输出必定符合语法规范

### 4. Pydantic Validation

使用 Pydantic 等库定义数据模型，让 LLM 输出后自动验证和转换。

**优势：**
- 声明式定义数据结构
- 自动类型转换和验证
- 错误反馈和重试机制

## 代码示例

### 示例 1：JSON Mode（OpenAI）

```python
import json
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一个助手。请以JSON格式输出用户信息。"},
        {"role": "user", "content": "提取以下信息：张三，男，28岁，软件工程师，邮箱 zhangsan@example.com"}
    ],
    response_format={
        "type": "json_object",
        "schema": {
            "type": "object",
            "properties": {
                "name": {"type": "string"},
                "gender": {"type": "string"},
                "age": {"type": "integer"},
                "occupation": {"type": "string"},
                "email": {"type": "string", "format": "email"}
            },
            "required": ["name", "gender", "age"]
        }
    }
)

result = json.loads(response.choices[0].message.content)
print(result)
# {'name': '张三', 'gender': '男', 'age': 28, 'occupation': '软件工程师', 'email': 'zhangsan@example.com'}
```

### 示例 2：Pydantic Validation

```python
from pydantic import BaseModel, Field, ValidationError
from typing import Optional
from openai import OpenAI

client = OpenAI()

class UserProfile(BaseModel):
    name: str = Field(..., min_length=1, max_length=50)
    gender: str = Field(..., pattern="^(男|女|其他)$")
    age: int = Field(..., ge=0, le=150)
    occupation: Optional[str] = None
    email: str = Field(..., pattern=r"^[\w\.-]+@[\w\.-]+\.\w+$")

def extract_user_info(text: str) -> UserProfile:
    """从文本中提取用户信息并验证"""
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": "从以下文本中提取用户信息，以JSON格式输出："},
            {"role": "user", "content": text}
        ],
        response_format={"type": "json_object"}
    )

    data = json.loads(response.choices[0].message.content)
    return UserProfile(**data)  # 自动验证和转换

try:
    user = extract_user_info("张三是一个28岁的男性软件工程师，他的邮箱是 zhangsan@example.com")
    print(f"姓名: {user.name}, 年龄: {user.age}, 邮箱: {user.email}")
except ValidationError as e:
    print(f"验证失败: {e.errors()}")
```

### 示例 3：Regex 约束引导

```python
import re

def extract_phone_numbers(text: str) -> list[str]:
    """使用正则表达式引导模型提取手机号"""
    # 定义手机号格式（中国）
    phone_pattern = r"1[3-9]\d{9}"

    # 在 prompt 中明确格式要求
    prompt = f"""
请从以下文本中找出所有手机号，以 | 分隔输出。
格式要求：必须是 11 位数字，以 1 开头，第二位是 3-9。

文本内容：
{text}

输出示例：13812345678|15912345678|13612345678
"""

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": "你是一个信息提取助手，严格按照指定格式输出。"},
            {"role": "user", "content": prompt}
        ]
    )

    output = response.choices[0].message.content.strip()

    # 验证和清理输出
    phones = [p for p in output.split("|") if re.match(phone_pattern, p)]
    return phones

text = "李四的电话是13812345678，王五的是15987654321，还有13800001111"
phones = extract_phone_numbers(text)
print(phones)  # ['13812345678', '15987654321', '13800001111']
```

## 应用场景

| 场景 | 技术选择 | 说明 |
|------|----------|------|
| API 数据提取 | JSON Mode + Pydantic | 需要严格类型验证的场景 |
| 日志解析 | Regex 约束 | 固定格式的文本提取 |
| 代码生成 | Grammar Sampling | 必须符合语法规范的场景 |
| 配置文件生成 | JSON Mode + Schema | 需要符合特定配置的格式 |
| 表单处理 | Pydantic Validation | 需要多字段验证和转换 |

## 相关概念

- **Prompt 工程**：结构化输出的基础，需要在 prompt 中明确说明格式要求
- **JSON Mode**：API 级别的 JSON 输出支持
- **Pydantic**：Python 中最流行的数据验证库
- **工具调用**：结构化输出是工具调用的基础，函数调用本质就是结构化输出
- **JSON Schema**：用于定义和验证 JSON 结构的规范
