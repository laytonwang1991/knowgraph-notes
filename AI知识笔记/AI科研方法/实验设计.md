# 一句话定义

ML实验设计是一套科学化的实验方法论，通过合理的实验设计、超参数搜索、统计检验和消融实验，确保实验结论的可靠性和可重复性。

---

## 详细说明

### 1. 超参数搜索（Hyperparameter Search）

超参数对模型性能有显著影响，需要系统化的搜索策略。

**搜索方法：**

| 方法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| Grid Search | 小规模参数空间 | 全面覆盖 | 计算开销大 |
| Random Search | 中等规模参数空间 | 效率高 | 可能遗漏最优 |
| Bayesian Optimization | 大规模参数空间 | 高效探索 | 实现复杂 |
| Hyperband | 大规模深度学习 | 早停策略 | 资源分配复杂 |
| Population-based Training | 动态超参数 | 自适应调整 | 需大量实验 |

**关键实践：**
- 使用验证集而非测试集调参
- 设置合理的搜索范围和步长
- 记录每次实验的随机种子
- 考虑超参数之间的交互作用

### 2. A/B测试（A/B Testing）

A/B测试是比较两个模型或方法实际效果的统计方法。

**实验设计要素：**
```
实验组分派
- 控制组（Baseline）
- 实验组（Treatment）
- 样本随机分组

评估指标
- 主要指标（Primary Metric）
- 辅助指标（Secondary Metrics）
- 护栏指标（Guardrail Metrics）

样本量计算
- 预期效应量（Effect Size）
- 统计功效（Power）≥ 0.8
- 显著性水平（α）= 0.05
```

**常见陷阱：**
- 选择偏差：样本分组不随机
- 辛普森悖论：分层数据中的反常结论
- 多重比较：多次检验导致的假阳性
- 新奇效应：用户对新鲜事物的短期偏好

### 3. 统计显著性（Statistical Significance）

统计检验确保观察到的差异不是随机波动。

**常用检验方法：**

| 场景 | 检验方法 |
|------|----------|
| 两组均值比较 | t-test（独立/配对） |
| 多组均值比较 | ANOVA |
| 分类指标比较 | 卡方检验 |
| 非正态分布 | Mann-Whitney U / Wilcoxon |
| 多重检验校正 | Bonferroni / FDR |

**效应量（Effect Size）：**
- Cohen's d：标准化均值差异
- Glass's Δ：实验组相对于控制组的变化
- Pearson r：相关系数
- OR（比值比）：分类问题常用

**置信区间：**
- 报告点估计的同时报告置信区间
- 95%置信区间是最常用选择
- 避免仅依赖p值的误解

### 4. 消融实验（Ablation Study）

消融实验通过逐一移除或修改组件，理解各部分的贡献。

**消融类型：**
- **组件消融**：移除单个模块，观察性能变化
- **参数消融**：调整关键超参数，探索最优配置
- **数据消融**：移除部分训练数据，测试数据多样性
- **输入消融**：逐个移除输入特征，理解特征重要性

**消融设计原则：**
- 控制变量：每次只改变一个因素
- 记录基线：保留完整模型的性能基准
- 分析边际贡献：计算每个组件的性能增益
- 可视化对比：使用图表直观展示消融结果

## 应用场景

### 场景一：超参数调优
- 在验证集上进行系统化超参数搜索
- 使用贝叶斯优化高效探索参数空间
- 最终模型在测试集上评估

### 场景二：模型对比
- 设计A/B测试比较新旧模型
- 确保样本分组随机且代表性
- 进行统计检验确认显著性

### 场景三：论文实验
- 设计全面的消融实验
- 证明每个组件的必要性
- 与现有方法进行公平对比

### 场景四：在线实验
- 设计护栏指标监控副作用
- 分阶段放量降低风险
- 持续监控长期效果

## 相关概念

| 概念 | 说明 |
|------|------|
| 超参数搜索 | 系统化探索模型超参数空间的方法 |
| A/B测试 | 通过统计方法比较两个版本效果的实验 |
| 统计显著性 | 观察结果非随机概率的统计保证 |
| 消融实验 | 通过移除组件理解其贡献的实验方法 |
| 对照实验 | 设置控制组和实验组的比较实验 |
| 交叉验证 | 利用验证集轮换提高评估可靠性的方法 |
| 偏差-方差权衡 | 模型复杂度与泛化能力的平衡 |
| 过拟合 | 模型在训练集上表现过好但泛化能力差 |
