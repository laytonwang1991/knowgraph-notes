# 视频生成模型

## 一句话定义

视频生成模型是一类能够合成时间连续、视觉逼真帧序列的生成模型，通过在扩散或自回归框架中引入时间维度的建模，实现从文本、图像或噪声到动态视频内容的生成。

## 核心公式

### 视频扩散模型前向过程

对于视频帧序列 $x_0, x_1, \ldots, x_T$，前向加噪过程为：

$$
q(x_t^t | x_{t-1}^t) = \mathcal{N}(x_t^t; \sqrt{1-\beta_t} x_{t-1}^t, \beta_t I)
$$

T步后：

$$
x_t^t = \sqrt{\bar{\alpha}_t} x_0^t + \sqrt{1-\bar{\alpha}_t} \epsilon^t, \quad \epsilon^t \sim \mathcal{N}(0, I)
$$

其中 $x_t^t$ 表示第 $i$ 帧在时间步 $t$ 的带噪版本。

### 时空联合注意力

$$
\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
$$

其中 $Q, K, V$ 来自时空维度联合展开的特征。

### 3D VAE重建损失

视频压缩自编码器的重建损失：

$$
\mathcal{L}_{recon} = \|x - \text{Decoder}(\text{Encoder}(x))\|_1
$$

## 详细说明

### 1. 核心挑战

视频生成面临三大核心挑战：

- **时序连贯性（Temporal Consistency）**：连续帧之间需要保持物体运动、光照、遮挡的逻辑一致
- **长程依赖（Long-range Dependency）**：视频开头的动作与结尾可能存在因果关联
- **计算效率（Computational Efficiency）**：视频数据量是图像的T倍，Transformer/UNet参数量和计算成本急剧增长

### 2. 技术路线

视频生成主要有以下几条技术路线：

| 路线 | 代表工作 | 特点 |
|------|----------|------|
| 视频扩散模型 | Make-A-Video, Imagen Video, SVD | 质量高，样本多样，采样慢 |
| 自回归视频生成 | VideoGPT, LVM | 统一框架，可学习压缩 |
| GAN路线 | MoCoGAN, VIDIT | 生成快，但质量不如扩散 |
| 噪声骨架 | Noise to Video, cascaded | 多阶段生成长视频 |

### 3. 视频扩散模型架构

#### 3.1 3D U-Net（时空联合建模）

典型架构将图像空间的2D卷积扩展到3D：

```
输入: (B, C, T, H, W)  # 批量, 通道, 时间, 高度, 宽度
3D卷积核: (k, k, k_t)  # 空间卷积核 + 时间卷积核
```

每个resblock同时在空间和时间维度上操作：

$$
y = \text{Conv3D}(\text{Norm}(x)) + \text{TimeAttn}(\text{Conv3D}(\text{Norm}(x)))
$$

#### 3.2 视频压缩自编码器（Video VAE）

将视频压缩到低维潜空间是处理长视频的关键：

- **空间压缩**：类似图像VAE，压缩H和W维度
- **时间压缩**：在时间维度上采样，减少帧数
- **典型压缩比**：8x8x4（空间8倍，时间4倍）
- **代表工作**：MagicVideo的zeroscope，LaVie的VideoAutoEncoder

#### 3.3 时空分离注意力

为降低计算复杂度，主流方案采用时空分离注意力：

1. **空间自注意力**：在同一帧内建立空间关联
2. **时间注意力**：跨帧建立时间关联（全局或局部窗口）

计算量从 $O((THW)^2)$ 降低到 $O(T H^2 W^2 + H W T^2)$。

### 4. 典型模型解析

#### Make-A-Video（Meta, 2022）

架构要点：
- 基于伪3D卷积的时序建模（在预训练图像模型上扩展）
- MLP-mixer时间建模层
- 引入高质量视频数据集（WebVid-10M）
- 支持图像到视频（I2V）和文本到视频（T2V）

#### Imagen Video（Google, 2022）

架构要点：
- 级联扩散：基础模型 + 多个Temporal超分模型
- 7B参数Video Diffusion Transformer
- 发现T5文本编码器对视频质量提升显著
- 分离空间超分和时间超分

#### Stable Video Diffusion（SVD, Stability AI, 2023）

架构要点：
- 基于SDXL架构的视频化扩展
- 使用Lora微调
- 4秒25帧，15fps
- 成为的开源视频生成基础模型

#### Sora（OpenAI, 2024）

架构要点（根据公开技术报告）：
- 基于DiT（Diffusion Transformer）的视频扩散
- 视频压缩网络：将视频映射到时空潜变量
- Transformer替代U-Net处理潜变量
- 原生支持多种分辨率/宽高比/时长
- 展现了长视频的world model潜力

### 5. 条件生成机制

视频扩散模型支持多种条件：

| 条件类型 | 实现方式 | 代表模型 |
|----------|----------|----------|
| 文本 | T5/CLIP text encoder | Imagen Video |
| 图像/视频 | VAE latent注入 | SVD, I2VGen |
| 姿态 | ControlNet骨骼图 | ControlVideo |
| 深度图 | ControlNet Depth | Depth-to-Video |
| 音频 | Audio-visual联合 | AudioCam |

### 6. 长视频生成策略

| 策略 | 方法 | 优缺点 |
|------|------|--------|
| 自回归 | 逐段生成，后段依赖前段 | 质量好，误差累积 |
| 噪声调度 | 不同阶段用不同噪声水平 | 避免累积误差 |
| 关键帧插值 | 先生成关键帧，再插值 | 节省计算 |
| 分块生成 | 分段独立生成，边缘融合 | 并行化，边界伪影 |

### 7. 优缺点

**优点**：
- 视频扩散模型质量已超越GAN
- 文本控制灵活，可精细调控
- 3D一致性逐渐改善
- 长视频能力持续突破

**缺点**：
- 采样速度极慢（数十到数百步）
- 长视频时序连贯性仍有瑕疵
- 物理规律理解不足（手、遮挡、物理交互）
- 计算资源需求高（百卡训练算力）

## 代码示例

```python
import torch
import torch.nn as nn

class Pseudo3DConv(nn.Module):
    """伪3D卷积：空间卷积 + 时间卷积分离"""
    def __init__(self, in_ch, out_ch, kernel_size=(3, 3, 1), stride=(1, 1, 1)):
        super().__init__()
        # 空间卷积（3x3）
        self.spatial_conv = nn.Conv2d(in_ch, out_ch, kernel_size[0], stride[0], padding=kernel_size[0]//2)
        # 时间卷积（1x1x3）
        self.temporal_conv = nn.Conv3d(out_ch, out_ch, (1, 1, kernel_size[2]), stride=stride)

    def forward(self, x):
        # x: (B, C, T, H, W)
        B, C, T, H, W = x.shape
        # 合并时间和batch维度做空间卷积
        x = x.permute(0, 2, 1, 3, 4).reshape(B * T, C, H, W)
        x = self.spatial_conv(x)
        _, C_out, H_out, W_out = x.shape
        # 恢复维度
        x = x.reshape(B, T, C_out, H_out, W_out).permute(0, 2, 1, 3, 4)
        # 时间卷积
        x = self.temporal_conv(x)
        return x

class TemporalAttention(nn.Module):
    """时间注意力模块"""
    def __init__(self, dim, num_heads=8):
        super().__init__()
        self.attn = nn.MultiheadAttention(dim, num_heads, batch_first=True)
        self.norm = nn.LayerNorm(dim)

    def forward(self, x):
        # x: (B, C, T, H, W)
        B, C, T, H, W = x.shape
        # 全局时间注意力：将空间维度合并
        x = x.permute(0, 2, 3, 4, 1).reshape(B * H * W, T, C)
        x = self.attn(x, x, x)[0]
        x = x.reshape(B, H, W, T, C).permute(0, 4, 3, 1, 2)
        return x

class VideoDiTBlock(nn.Module):
    """视频DiT Block: 空间自注意力 + 时间注意力 + FFN"""
    def __init__(self, dim, num_heads=8):
        super().__init__()
        self.norm1 = nn.LayerNorm(dim)
        self.spatial_attn = nn.MultiheadAttention(dim, num_heads, batch_first=True)
        self.norm2 = nn.LayerNorm(dim)
        self.temporal_attn = TemporalAttention(dim, num_heads)
        self.norm3 = nn.LayerNorm(dim)
        self.ffn = nn.Sequential(
            nn.Linear(dim, dim * 4),
            nn.GELU(),
            nn.Linear(dim * 4, dim)
        )

    def forward(self, x):
        # x: (B*T, H*W, C) 或 (B, C, T, H, W)
        B, C, T, H, W = x.shape
        # 空间自注意力
        h = x.permute(0, 2, 3, 4, 1).reshape(B * T, H * W, C)
        h = self.norm1(h)
        h = self.spatial_attn(h, h, h)[0]
        x = x + h.reshape(B, T, H, W, C).permute(0, 4, 1, 2, 3)
        # 时间注意力
        x = x + self.temporal_attn(self.norm2(x))
        # FFN
        h = x.permute(0, 2, 3, 4, 1).reshape(B * T * H * W, C)
        h = self.ffn(self.norm3(h))
        x = x + h.reshape(B, T, H, W, C).permute(0, 4, 1, 2, 3)
        return x
```

## 应用场景

1. **影视制作**：AI辅助生成特效镜头、B-roll素材，降低制作成本
2. **游戏与虚拟世界**：实时生成游戏过场动画、NPC行为可视化
3. **广告创意**：快速生成概念视频，缩短创意迭代周期
4. **教育和培训**：合成教学视频、模拟危险场景（驾驶、医疗）
5. **数据增强**：生成多样化训练数据，提升视觉模型鲁棒性
6. **个性化内容**：根据用户偏好实时生成定制化视频
7. **虚拟试穿**：时尚行业的服装上身效果视频展示
8. **3D到4D**：结合NeRF技术，从静态3D生成动态4D内容

## 相关概念

- **视频扩散模型（Video Diffusion Model）**：在扩散框架下处理视频帧序列的生成模型
- **时空注意力（Spatiotemporal Attention）**：同时建模空间和时间依赖的注意力机制
- **3D U-Net**：扩展到时间维度的U-Net架构，用于视频语义分割和生成
- **DiT（Diffusion Transformer）**：用Transformer替代U-Net的扩散模型架构
- **视频压缩（Video Compression）**：类似图像VAE，在时间和空间上压缩视频到低维潜空间
- **Make-A-Video**：Meta提出的文本到视频扩散模型
- **Cascaded Diffusion**：多阶段级联扩散，从低分辨率逐步超分到高分辨率
- **时间插值（Temporal Interpolation）**：在关键帧之间插入中间帧
