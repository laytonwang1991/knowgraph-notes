# 本地部署方案

## 一句话定义

本地部署方案是指在个人计算机或私有服务器上运行大语言模型的技术栈，绕过云服务依赖，实现数据隐私控制和离线使用。

## 详细说明

### 主流部署工具对比

| 工具 | 易用性 | 性能 | 量化支持 | 适用场景 |
|------|--------|------|---------|---------|
| Ollama | 极高 | 中等 | 良好 | 快速上手、日常使用 |
| llama.cpp | 高 | 高 | 优秀 | 极致性能、老旧硬件 |
| vLLM | 中等 | 极高 | 良好 | 生产环境、高并发 |
| Ollama + vLLM | 高 | 高 | 良好 | 平衡之选 |

### Ollama

**特点**：开箱即用，一条命令运行模型

```bash
# 安装 (macOS/Linux)
curl -fsSL https://ollama.com/install.sh | sh

# Windows直接下载安装包

# 运行模型
ollama run llama3.2

# 运行其他模型
ollama run mistral
ollama run codellama
ollama pull gemma2:9b
```

**API服务**：

```bash
# 启动API服务（默认11434端口）
ollama serve

# Python调用
import ollama

response = ollama.chat(
    model='llama3.2',
    messages=[
        {'role': 'user', 'content': '解释什么是Docker'}
    ]
)
print(response['message']['content'])
```

### llama.cpp

**特点**：纯C/C++实现，高性能，量化支持极佳

```bash
# 克隆编译
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
mkdir build && cd build
cmake ..
cmake --build . --config Release

# 下载量化模型（如TheBloke的GGUF模型）
# 从HuggingFace下载Q4_K_M量化版本

# 运行
./build/bin/llama-cli -m models/llama-7b-q4_k_m.gguf -p "解释什么是Kubernetes" -t 8 -ngl 99
```

**服务器模式**：

```bash
# 启动HTTP服务
./build/bin/llama-server -m models/llama-7b-q4_k_m.gguf -c 4096 -port 8080

# API调用
curl http://localhost:8080/completion -d '{"prompt": "Hello","n_predict": 128}'
```

### vLLM

**特点**：生产级高性能，支持PagedAttention

```bash
# 安装
pip install vllm

# Python使用
from vllm import LLM, SamplingParams

llm = LLM(model="meta-llama/Llama-3.2-3B-Instruct")

sampling_params = SamplingParams(temperature=0.7, max_tokens=256)

outputs = llm.generate(["解释什么是微服务架构"], sampling_params)

for output in outputs:
    print(output.outputs[0].text)
```

**命令行服务**：

```bash
# 启动服务
vllm serve meta-llama/Llama-3.2-3B-Instruct --host 0.0.0.0 --port 8000
```

### 量化模型选择

| 量化级别 | 文件大小 | 内存需求 | 质量损失 | 推荐场景 |
|---------|---------|---------|---------|---------|
| FP16 | 100% | 高 | 无 | 最高质量需求 |
| Q8_0 | ~50% | 高 | 极小 | 研发/测试 |
| Q6_K | ~40% | 中 | 较小 | 平衡选择 |
| Q5_K_M | ~33% | 中 | 可接受 | 常用推荐 |
| Q4_K_M | ~25% | 低 | 较小 | 内存受限首选 |
| Q4_0 | ~22% | 低 | 中等 | 最低可行 |
| Q3_K_M | ~20% | 极低 | 较明显 | 极致压缩 |
| Q2_K | ~17% | 极低 | 明显 | 仅作演示 |

### 硬件要求

**最低配置（7B模型）**：

- CPU: 4核+
- RAM: 8GB+ (Q4量化)
- 显卡: 非必须（CPU可跑，慢）

**推荐配置（7B模型流畅）**：

- CPU: 6核+
- RAM: 16GB+
- 显卡: NVIDIA 6GB+ VRAM (如RTX 3060)

**13B模型**：

- 显卡: 8GB+ VRAM (如RTX 3080)

**70B模型**：

- 显卡: 24GB+ VRAM (如A100)
- 或多卡并行

## 应用场景

1. **隐私敏感场景**：不希望数据上传云端
2. **离线开发**：无网络环境下的开发调试
3. **成本控制**：长期高频使用避免云服务费用
4. **定制微调**：基于私有数据微调模型
5. **内网部署**：企业内网环境使用

## 相关概念

- **量化(Quantization)**：用较低精度存储模型，减少内存占用
- **GGUF格式**：llama.cpp的模型格式，支持K-quant等高级量化
- **CUDA**：NVIDIA GPU计算平台
- **PagedAttention**：vLLM的高效注意力机制
- **LoRA/QLoRA**：轻量级微调技术
- **ollama pull**：下载模型到本地
