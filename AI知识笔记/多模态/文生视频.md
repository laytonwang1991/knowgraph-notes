# 文生视频（Text-to-Video）

## 一句话定义

文生视频（Text-to-Video）是指通过文本描述生成对应视频序列的深度学习技术，需要同时建模空间语义和时间动态。

## 核心公式

### 1. 时空扩散模型
$$p_\theta(\mathbf{x}_{0:T}|c) = p(\mathbf{x}_T)\prod_{t=1}^{T} p_\theta(\mathbf{x}_{t-1}|\mathbf{x}_t, c)$$

其中 $\mathbf{x}_{0:T} = (\mathbf{x}_0, \mathbf{x}_1, ..., \mathbf{x}_T)$ 是视频序列，$c$ 是文本条件。

### 2. 3D-UNet时空建模
$$\epsilon_\theta(\mathbf{x}_t^V, t, c) = \text{3D-UNet}(\mathbf{x}_t^V; \theta_{\text{spat}}, \theta_{\text{temp}}, \text{Attention}_{S\times T})$$

其中 $\mathbf{x}_t^V \in \mathbb{R}^{B \times C \times F \times H \times W}$ 是视频的潜在表示，$F$ 是帧数。

### 3. 视频潜在编码
$$\mathbf{z}_V = \text{VideoEncoder}(\mathbf{x}^V), \quad \hat{\mathbf{x}}^V = \text{VideoDecoder}(\mathbf{z}_V)$$

类似图像VAE，视频需要专门的时空编码器和解码器。

### 4. Motion Dynamics建模
$$\mathbf{h}_f = \text{MotionEncoder}(\mathbf{x}_f, \mathbf{x}_{f-1}), \quad \text{Motion} = \text{MotionDecoder}(\mathbf{h}_1, ..., \mathbf{h}_F)$$

### 5. 渐进式生成（Temporal Consistency）
$$\hat{\mathbf{x}}_{f+1} = \mathbf{x}_f + \Delta \mathbf{x}_f \cdot \text{Mask}(\text{optical\_flow}_{f \to f+1})$$

## 详细说明

### 技术发展历程
- **2020-2021**：GAN时代 - VGAN, Text2Video, CogVideo
- **2022-2023**：扩散模型崛起 - ModelScope, Tune-A-Video, Gen-2
- **2024**：长视频突破 - Sora, Lumiere, Stable Video Diffusion

### 模型架构分类

#### A. 纯扩散模型
```
文本输入 → CLIP编码器 → 条件潜在扩散（时空3D-UNet）→ 视频解码器 → 视频输出
```
代表：Lumiere, Sora（部分）

#### B. 扩散+Transformer混合
```
文本输入 → T5/PaLM编码器 → DiT（扩散Transformer）→ 视频解码器 → 视频输出
```
代表：Sora, Stable Video Diffusion

#### C. 基于VAE+GAN的方案
```
文本+噪声 → 动作预测模块 → 视频解码器 → 判别器评估 → 输出
```
代表：早期模型 CogVideo

### 关键技术挑战

1. **时序一致性（Temporal Consistency）**
   - 相邻帧之间的内容连贯性
   - 光流一致性约束
   - 运动模糊处理

2. **长视频生成**
   - 长时间动作连贯性
   - 场景一致性问题
   - 内存和计算限制

3. **物理世界模拟**
   - 物体运动符合物理规律
   - 交互物体的物理一致性
   - 重力、光照等物理现象

4. **可控生成**
   - 相机运动控制
   - 角色动作控制
   - 场景编辑控制

### 世界模型视角

Sora展现出的"世界模型"能力：

1. **3D一致性**：在变换相机时保持场景一致性
2. **物体持久性**：当物体离开视野后再出现，保持其属性
3. **物理交互**：模拟物体之间的碰撞、支撑等交互
4. **长期依赖**：保持事件和动作的因果关系

## 代码示例

### 使用ModelScope生成视频

```python
from modelscope.pipelines import pipeline
from modelscope.outputs import OutputKeys
import torch

# 初始化文生视频管道
pipe = pipeline(
    'text-to-video-synthesis',
    model='damo/text-to-video-synthesis',
    device='cuda' if torch.cuda.is_available() else 'cpu'
)

# 生成视频
prompt = "A beautiful sunset over the ocean, waves crashing on the beach, cinematic"
output = pipe({'text': prompt})
video_path = output[OutputKeys.OUTPUT_VIDEO]

print(f"视频已保存至: {video_path}")
```

### 使用Diffusers进行视频生成

```python
from diffusers import StableVideoDiffusionPipeline

    variant="fp16"

pipe = pipe.to("cuda")

# 提供初始图像（可选，用于图生视频）

# 生成视频
with torch.no_grad():
    frames = pipe(
        image,
        num_frames=25,
        decode_chunk_size=8,
        num_inference_steps=25,
        guidance_scale=3.0
    ).frames[0]

# 导出视频
export_to_video(frames, "output_video.mp4", fps=24)
```

### 自定义视频扩散Pipeline

```python
import torch
import torch.nn as nn
from typing import Optional, List

class VideoDiffusionPipeline:
    """简化版视频扩散生成流程"""

    def __init__(self, model_config):
        self.text_encoder = self._load_text_encoder(model_config['text_model'])
        self.video_unet = self._load_video_unet(model_config['unet'])
        self.video_vae = self._load_video_vae(model_config['vae'])
        self.scheduler = DDPMScheduler.from_config(model_config['scheduler'])

    def generate(
        self,
        prompt: str,
        num_frames: int = 16,
        height: int = 256,
        width: int = 256,
        num_inference_steps: int = 50,
        guidance_scale: float = 7.5
    ) -> torch.Tensor:
        # 1. 文本编码
        text_embeddings = self._encode_prompt(prompt)

        # 2. 初始化视频潜在变量
        latents = torch.randn(
            (1, 4, num_frames, height // 8, width // 8),
            device=self.device,
            dtype=text_embeddings.dtype
        )

        # 3. 设置时间步
        self.scheduler.set_timesteps(num_inference_steps)

        # 4. 迭代去噪
        for t in self.scheduler.timesteps:
            # 扩展latents以进行CFG
            latent_model_input = torch.cat([latents] * 2)

            # 预测噪声
            with torch.no_grad():
                noise_pred = self.video_unet(
                    latent_model_input,
                    t,
                    encoder_hidden_states=text_embeddings
                ).sample

            # CFG
            noise_pred_uncond, noise_pred_text = noise_pred.chunk(2)
            noise_pred = noise_pred_uncond + guidance_scale * (
                noise_pred_text - noise_pred_uncond
            )

            # 去噪步骤
            latents = self.scheduler.step(noise_pred, t, latents).prev_sample

        # 5. 解码为视频
        video = self.video_vae.decode(latents)

        return video

    def _encode_prompt(self, prompt: str) -> torch.Tensor:
        """编码文本提示"""
        tokens = self.text_encoder(prompt)
        with torch.no_grad():
            embeddings = self.text_encoder.model(tokens).last_hidden_state
        return embeddings
```

### 视频风格迁移

```python
def apply_style_to_video(
    video_frames: List[Image.Image],
    style_prompt: str,
    strength: float = 0.75
) -> List[Image.Image]:
    """将指定风格应用到视频的每一帧"""

    style_pipe = StableDiffusionImg2ImgPipeline.from_pretrained(
        "runwayml/stable-diffusion-v1-5",
        torch_dtype=torch.float16
    ).to("cuda")

    styled_frames = []

    for frame in tqdm(video_frames, desc="应用风格"):
        # 使用ControlNet保持结构
        processed_frame = style_pipe(
            prompt=style_prompt,
            image=frame,
            strength=strength,
            guidance_scale=7.5
        ).images[0]

        styled_frames.append(processed_frame)

    return styled_frames
```

## 相关概念

### 核心概念
- **时空建模（Spatial-Temporal Modeling）**：同时考虑空间信息和时间维度的建模
- **光流（Optical Flow）**：描述像素在连续帧之间的运动
- **关键帧（Key Frame）**：定义动作轨迹的关键帧
- **过渡（Transition）**：场景或动作之间的平滑过渡

### 相关技术
- **图生视频（Image-to-Video）**：基于初始图像生成视频
- **视频续写（Video Extension）**：延长现有视频
- **视频编辑（Video Editing）**：修改视频的特定元素
- **视频风格迁移（Video Stylization）**：改变视频的艺术风格

### 技术对比
- **短视频 vs 长视频**：短视频（<10秒）质量更稳定，长视频需要更强的时序建模
- **可控 vs 随机**：可控生成（ControlNet、相机控制）vs 完全随机生成
- **真实感 vs 风格化**：写实风格 vs 动漫/艺术风格
