# 注意力机制变体

## 一句话定义

注意力机制变体是通过**稀疏化、线性近似、分块计算**等方法改进标准 Softmax Attention，在保持核心能力的同时，将 $O(n^2)$ 时间复杂度和空间复杂度降低到更高效的等级。

---

## 核心公式

### 标准 Softmax Attention

$$
\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

**复杂度：**
- 时间：$O(n^2 \cdot d)$
- 空间（KV Cache）：$O(n \cdot d)$

### Flash Attention

$$
\text{FlashAttention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

**改进：通过分块计算和融合核函数，避免物化完整的 $n \times n$ 注意力矩阵**

$$
\text{分块大小}: B = \frac{M}{4d} \quad (M \text{ 为 SRAM 大小})
$$

### Linear Attention（线性注意力）

$$
\text{LinearAttention}(Q, K, V) = \phi(Q) \cdot (\phi(K)^T \cdot V)
$$

**复杂度：**
- 时间：$O(n \cdot d^2)$
- 空间：$O(d^2)$

---

## 详细说明

### 1. Flash Attention

**一句话定义：** 通过**分块计算（Tile）** 和**核函数融合（Kernel Fusion）**，在 GPU HBM 和 SRAM 之间高效流动数据，避免物化 $O(n^2)$ 的完整注意力矩阵。

#### 核心思想

```
标准 Attention：                      Flash Attention：
┌─────────────────────┐              ┌─────────────────────┐
│ 计算完整的 QK^T      │              │ 分块加载 Q, K 到 SRAM│
│ 占用 O(n²) HBM      │              │ 分块计算 exp(QK^T/√d)│
│ 再与 V 相乘          │              │ 在线 softmax 归一化  │
└─────────────────────┘              │ 累加到输出           │
                                     └─────────────────────┘
```

#### 在线 Softmax 技巧

$$
m_i = \max(m_{i-1}, \frac{q_i^T k_j}{\sqrt{d}})
$$
$$
\ell_i = \ell_{i-1} \cdot e^{m_{i-1} - m_i} + e^{\frac{q_i^T k_j}{\sqrt{d}} - m_i}
$$

#### 性能提升

| 指标 | 标准 Attention | Flash Attention |
|------|---------------|-----------------|
| 时间复杂度 | $O(n^2)$ | $O(n^2)$（但实际加速 2-4x） |
| 空间复杂度 | $O(n^2)$ | $O(n)$ |
| GPU 内存 | $O(n^2)$ | $O(n)$ |
| 长序列支持 | 受限于显存 | 可处理 64K+ tokens |

#### 代码实现

```python
import torch
from flash_attn import flash_attn_func

# Flash Attention 调用示例
q = torch.randn(2, 8, 512, 64, dtype=torch.float16, device='cuda')  # (batch, heads, seq, dim)
k = torch.randn(2, 8, 512, 64, dtype=torch.float16, device='cuda')
v = torch.randn(2, 8, 512, 64, dtype=torch.float16, device='cuda')

# Flash Attention 前向计算
output = flash_attn_func(q, k, v, dropout_p=0.0, softmax_scale=None, causal=True)
# output: (2, 8, 512, 64)
```

### 2. Sparse Attention（稀疏注意力）

**一句话定义：** 通过**局部窗口、跳跃连接、随机采样**等模式，只计算部分 token 对之间的注意力，将稀疏性引入 Transformer。

#### 稀疏模式

| 模式 | 图示 | 复杂度 |
|------|------|--------|
| **滑窗注意力（Window）** | `[[[W W W]]`<br>` [[W W W]]` | $O(n \cdot w)$ |
| **全局注意力（Global）** | `[[G G G]`<br>` [G A A]` | $O(g^2 + n \cdot w)$ |
| **随机注意力（Random）** | 随机选择 $r$ 个连接 | $O(n \cdot r)$ |
| **块注意力（Block）** | 分块独立计算 | 可配置 |

#### Longformer 稀疏模式

$$
\text{Attention}_{Longformer}(i, j) =
\begin{cases}
\frac{q_i^T k_j}{\sqrt{d}} & \text{if } j \in \text{window}(i) \cup \{global\} \\
-\infty & \text{otherwise}
\end{cases}
$$

#### 代码示例

```python
class SparseAttention(nn.Module):
    """简化稀疏注意力实现"""
    def __init__(self, d_model, nhead, window_size=64, num_global=2):
        super().__init__()
        self.d_model = d_model
        self.nhead = nhead
        self.window_size = window_size
        self.num_global = num_global

        self.qkv = nn.Linear(d_model, d_model * 3)
        self.proj = nn.Linear(d_model, d_model)

    def forward(self, x):
        B, N, C = x.shape
        qkv = self.qkv(x).reshape(B, N, 3, self.nhead, C // self.nhead)
        q, k, v = qkv.unbind(dim=2)

        # 创建稀疏掩码
        mask = torch.zeros(N, N, device=x.device)
        # 全局注意力（首尾 token）
        mask[:self.num_global, :] = 1
        mask[-self.num_global:, :] = 1
        # 滑窗注意力
        for i in range(N):
            start = max(0, i - self.window_size // 2)
            end = min(N, i + self.window_size // 2 + 1)
            mask[i, start:end] = 1

        # 计算注意力（简化版）
        scale = (C // self.nhead) ** -0.5
        attn = (q @ k.transpose(-2, -1)) * scale
        attn = attn.masked_fill(mask == 0, float('-inf'))
        attn = F.softmax(attn, dim=-1)

        return self.proj((attn @ v).transpose(1, 2).reshape(B, N, C))
```

### 3. Linear Attention（线性注意力）

**一句话定义：** 通过**核函数近似** $\text{Softmax}(QK^T) \approx \phi(Q) \cdot \phi(K)^T$，将矩阵乘法顺序改变，打破 $O(n^2)$ 瓶颈。

#### 核心推导

$$
\text{Attention}(Q, K, V) = \text{Softmax}(QK^T) \cdot V
$$

设 $\text{Softmax}(x) \approx \tau(x)$（某核函数），则：

$$
\text{Attention} \approx \tau(QK^T) V = \phi(Q) (\phi(K)^T V)
$$

**关键：先算 $\phi(K)^T V$（$O(n \cdot d^2)$），再与 $\phi(Q)$ 相乘（$O(n \cdot d^2)$）**

#### 常见核函数

| 核函数 | 公式 | 特点 |
|--------|------|------|
| **ReLU** | $\phi(x) = \text{ReLU}(x)$ | 简单有效 |
| **Exp** | $\phi(x) = \exp(x)$ | 保持非负性 |
| **Performer** | $\phi(x) = \exp(-\|\|x\|\|^2) \cdot \exp(Wx)$ | 随机特征映射 |

#### 代码示例

```python
class LinearAttention(nn.Module):
    """线性注意力（Performer 风格）"""
    def __init__(self, d_model, nhead, num_features=256):
        super().__init__()
        self.d_model = d_model
        self.nhead = nhead
        self.num_features = num_features

        # 随机特征映射矩阵
        self.W = nn.Parameter(torch.randn(d_model, num_features) / math.sqrt(d_model))

        self.qkv = nn.Linear(d_model, d_model * 3)
        self.proj = nn.Linear(d_model, d_model)

    def forward(self, x):
        B, N, C = x.shape

        # 随机特征映射
        qkv = self.qkv(x).reshape(B, N, 3, self.nhead, C // self.nhead)
        q, k, v = qkv.unbind(dim=2)

        # 映射到特征空间
        q_prime = F.relu(q @ self.W)
        k_prime = F.relu(k @ self.W)

        # 线性注意力
        kv = torch.einsum('bnhd,bnhe->bhde', k_prime, v)
        z = k_prime.sum(dim=1, keepdim=True)
        output = torch.einsum('bnhd,bhde->bnhe', q_prime, kv / z)
        output = output.reshape(B, N, C)

        return self.proj(output)
```

### 4. Grouped Query Attention (GQA)

**一句话定义：** 将 **Query 分组**，每组共享一组 **Key-Value**，在保持多 Query 头表达能力的同时，大幅减少 KV 缓存。

#### 动机

- Multi-Head Attention：$n_{heads}$ 组 Q，每组独立 K、V → **计算量大，KV 缓存大**
- Multi-Query Attention：1 组 Q，1 组 K、V → **效率高，但表达能力下降**
- GQA：$n_{query\_heads}$ 组 Q，$n_{kv\_heads}$ 组 K、V → **平衡**

#### 公式

$$
\text{Multi-Head Attention}_{\text{GQA}}: \text{Concat}(\text{head}_1, \ldots, \text{head}_{n_q}) W^O
$$

其中每个 $\text{head}_i$ 共享 $K, V$（按组划分）：

$$
\text{head}_i = \text{Attention}(q_i, K_{i \mod n_{kv}}, V_{i \mod n_{kv}})
$$

#### 对比

| 注意力类型 | Query 头数 | KV 头数 | KV 缓存 | 表达能力 |
|------------|-----------|---------|---------|----------|
| Multi-Head (MHA) | $h$ | $h$ | $2 \times h \times d$ | 最高 |
| Multi-Query (MQA) | $h$ | 1 | $2 \times d$ | 中等 |
| Grouped-Query (GQA) | $h$ | $g \ll h$ | $2 \times g \times d$ | 高 |

#### 代码示例

```python
class GroupedQueryAttention(nn.Module):
    """分组查询注意力"""
    def __init__(self, d_model, nhead, n_kv_heads):
        super().__init__()
        self.nhead = nhead
        self.n_kv_heads = n_kv_heads
        self.head_dim = d_model // nhead

        assert nhead % n_kv_heads == 0, "nhead must be divisible by n_kv_heads"
        self.n_groups = nhead // n_kv_heads

        self.q_proj = nn.Linear(d_model, d_model)
        self.k_proj = nn.Linear(d_model, self.n_kv_heads * self.head_dim)
        self.v_proj = nn.Linear(d_model, self.n_kv_heads * self.head_dim)
        self.o_proj = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B, N, C = x.shape

        q = self.q_proj(x).reshape(B, N, self.nhead, self.head_dim)
        k = self.k_proj(x).reshape(B, N, self.n_kv_heads, self.head_dim)
        v = self.v_proj(x).reshape(B, N, self.n_kv_heads, self.head_dim)

        # 扩展 KV 头以匹配 Q 头
        k = repeat(k, 'b s kv h d -> b s (kv g) h d', g=self.n_groups)
        v = repeat(v, 'b s kv h d -> b s (kv g) h d', g=self.n_groups)

        # 标准的注意力计算
        scale = self.head_dim ** -0.5
        attn = torch.einsum('bqhd,bkhd->bhqk', q, k) * scale
        if mask is not None:
            attn = attn.masked_fill(mask == 0, float('-inf'))
        attn = F.softmax(attn, dim=-1)

        out = torch.einsum('bhqk,bkhd->bqhd', attn, v).reshape(B, N, C)
        return self.o_proj(out)
```

### 5. Multi-Query Attention vs Grouped Query Attention

| 特性 | MHA | MQA | GQA |
|------|-----|-----|-----|
| KV 头数 | = Q 头数 | 1 | $g$ (2-8) |
| KV 缓存 | 最大 | 最小 | 中等 |
| 推理速度 | 慢 | 最快 | 快 |
| 表达能力 | 最强 | 最弱 | 可比 MHA |
| 代表模型 | BERT | GPT-1/2 | LLaMA 2+, Mistral |

## 应用场景

| 场景 | 推荐注意力 | 理由 |
|------|-----------|------|
| 超长序列（>32K） | Flash Attention + Sparse | 显存友好，支持长上下文 |
| 实时对话 | GQA / MQA | 减少 KV 缓存，降低延迟 |
| 长文档理解 | Longformer / BigBird | 稀疏模式处理超长文档 |
| 线性复杂度需求 | Linear Attention | 理论 $O(n)$ 复杂度 |
| 高精度任务 | MHA / GQA | 保持完整表达能力 |
| 边缘设备部署 | GQA + 量化 | 效率与质量平衡 |

## 相关概念

- **KV Cache**：缓存 Key-Value 对，加速自回归生成
- **SRAM / HBM**：GPU 内存层级，决定 Flash Attention 分块大小
- **Softmax 近似**：线性注意力的核心技巧
- **Rotary Position Embedding (RoPE)**：常与各类注意力结合使用
- **Attention Sink**：某些 token 吸引大量注意力的现象
