# 语言模型评估

## 一句话定义

语言模型评估是量化衡量语言模型性能的一系列标准与方法，通过概率度量、生成质量和任务表现等多个维度判断模型能力的系统性过程。

## 核心公式

### Perplexity（困惑度）

$$PPL = \left( \prod_{i=1}^{N} \frac{1}{P(w_i | w_1, ..., w_{i-1})} \right)^{\frac{1}{N}} = \exp\left( -\frac{1}{N} \sum_{i=1}^{N} \log P(w_i | w_1, ..., w_{i-1}) \right)$$

困惑度越低，模型对语言的建模能力越强。

### BLEU Score

$$BLEU = BP \cdot \exp\left( \sum_{n=1}^{N} w_n \log p_n \right)$$

其中 $BP$ 为短句惩罚，$p_n$ 为 n-gram 精确率，$w_n$ 为权重（通常均匀权重）。

### 区分度（Discriminability）

$$D = \frac{1}{K} \sum_{k=1}^{K} \frac{1}{|S_k|} \sum_{i=1}^{|S_k|} \frac{P_{LM}(w_i^{(k)} | w_1^{(k)}, ..., w_{i-1}^{(k)})}{P_{ref}(w_i^{(k)} | w_1^{(k)}, ..., w_{i-1}^{(k)})}$$

## 详细说明

### 1. 概率类指标

- **Perplexity**：衡量模型预测下一个词的"不确定度"，等价于语言分支系数的指数。是最基础的内蕴评估指标。
- **Bits per Character (BPC)**：每个字符用多少比特编码，在字符级模型中与 Perplexity 可相互转换。
- **Entrophy**：香农熵的变体，衡量语言分布的随机性。

### 2. 生成质量指标

- **BLEU**：n-gram 精确率的加权几何平均，衡量生成文本与参考文本的词汇重叠度。主要用于机器翻译和文本生成。
- **ROUGE**：召回导向的评估指标，衡量生成内容对参考内容的覆盖程度，常用于文本摘要。
- **METEOR**：基于词匹配的评估，对同义词和词干变化更宽容。
- **chrF**：基于字符 n-gram 的评估，对形态丰富的语言效果更好。

### 3. 零样本评估（Zero-Shot Evaluation）

- 在未见过的任务上直接评估模型能力，反映模型的泛化性。
- 常用基准：MMLU（多任务语言理解）、Big-Bench、HellaSwag（常识推理）、ARC（推理挑战）。
- 不需要任何微调，直接以文本提示的方式让模型作答。

### 4. 下游任务评估（Downstream Task Evaluation）

- **NLU 任务**：文本分类、情感分析、问答、自然语言推理（NLI）。
- **NLG 任务**：机器翻译、摘要生成、对话生成。
- **标准数据集**：SuperGLUE、GLUE、SQuAD、CoNLL 系列。
- 通常使用微调后的模型进行评估。

### 5. 人类评估（Human Evaluation）

- 最可靠但成本最高的评估方式。
- 指标：流畅性、一致性、信息正确性、相关性、有用性。
- 常用成对比较（Pairwise Comparison）或单一评分（Single Score）。
- 受限于主观性、标注者差异和规模。

## 代码示例

### Python 计算 Perplexity

```python
import math
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

def calculate_perplexity(model, tokenizer, text, device="cuda"):
    encodings = tokenizer(text, return_tensors="pt")
    encodings = {k: v.to(device) for k, v in encodings.items()}

    with torch.no_grad():
        outputs = model(**encodings, labels=encodings["input_ids"])
        neg_log_likelihood = outputs.loss * encodings["input_ids"].size(1)

    ppl = torch.exp(neg_log_likelihood / encodings["input_ids"].size(1))
    return ppl.item()

# 示例
model_name = "gpt2"
model = AutoModelForCausalLM.from_pretrained(model_name).to("cuda")
tokenizer = AutoTokenizer.from_pretrained(model_name)

text = "The quick brown fox jumps over the lazy dog."
ppl = calculate_perplexity(model, tokenizer, text)
print(f"Perplexity: {ppl:.2f}")
```

### Python 计算 BLEU Score

```python
from nltk.translate.bleu_score import sentence_bleu, SmoothingFunction

reference = [["这", "是", "一", "个", "测试"]]
candidate = ["这", "是", "一", "个", "示例"]

# 使用平滑函数避免 0 BLEU
smooth = SmoothingFunction().method1
bleu = sentence_bleu(reference, candidate, smoothing_function=smooth)
print(f"BLEU: {bleu:.4f}")
```

## 应用场景

1. **模型选择与对比**：在多个候选模型中选择最优模型，需要统一的评估标准。
2. **模型迭代优化**：在训练过程中持续监控 Perplexity 等指标，判断模型收敛情况。
3. **能力诊断**：通过特定任务评估发现模型的强项与弱项（如长文本推理、数学能力）。
4. **安全与对齐评估**：检测模型的有害输出、幻觉问题和偏见。
5. **基准测试竞赛**：以排行榜形式推动模型能力提升（SuperGLUE、OpenCompass）。

## 相关概念

- **语言模型 (Language Model)**：定义语言序列概率分布的模型，是评估的对象。
- **Perplexity**：基于交叉熵的评估指标，衡量模型的不确定度。
- **BLEU**：基于 n-gram 重叠的生成质量评估指标。
- **零样本学习 (Zero-Shot Learning)**：模型在完全未见过任务上的泛化能力测试。
- **下游任务 (Downstream Task)**：实际应用任务，如分类、翻译、问答等。
- **指令微调 (Instruction Tuning)**：通过指令数据提升模型遵循任务要求的能力。
- **人类评估 (Human Evaluation)**：最接近真实使用体验的评估方式。
