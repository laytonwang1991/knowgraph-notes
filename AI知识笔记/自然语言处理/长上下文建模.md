# 长上下文建模

## 一句话定义

长上下文建模是指使语言模型能够在超出标准上下文窗口（如 4096 tokens）的超长文本上高效执行注意力计算并保持关键信息检索能力的一系列建模技术与架构改进。

## 核心公式

### 稀疏注意力分数

$$A_{ij}^{sparse} = \begin{cases} \frac{\exp(Q_i \cdot K_j / \sqrt{d})}{\sum_{j \in \mathcal{S}(i)} \exp(Q_i \cdot K_j / \sqrt{d})} & \text{if } j \in \mathcal{S}(i) \\ 0 & \text{otherwise} \end{cases}$$

其中 $\mathcal{S}(i)$ 是 token $i$ 的稀疏邻域集合。

### 层次化注意力（Hierarchy Attention）

$$\alpha_{ij}^{(l)} = \text{Attention}(H_i^{(l)}, H_j^{(l)}), \quad H_i^{(l)} = \text{Pool}\left( \sum_{k \in \text{seg}(i)} H_k^{(l-1)} \right)$$

每一层对上一层同一段内的表示进行池化，实现层级递进的信息汇聚。

### 递归 Transformer 状态传递

$$\mathbf{h}_t^{(i)} = \text{TransformerBlock}(\mathbf{h}_t^{(i-1)}, \mathbf{c}_{t-1}^{(i)})$$

$$\mathbf{c}_t^{(i)} = \text{Summary}(\mathbf{h}_1^{(i)}, ..., \mathbf{h}_t^{(i)})$$

每段处理完后，将压缩状态 $\mathbf{c}_t^{(i)}$ 传入下一段作为上下文。

### Mamba SSM 状态更新

$$\mathbf{h}_t = \mathbf{A} \mathbf{h}_{t-1} + \mathbf{B} \mathbf{x}_t$$
$$\mathbf{y}_t = \mathbf{C} \mathbf{h}_t$$

其中 $\mathbf{A}, \mathbf{B}, \mathbf{C}$ 为选择性参数矩阵，由输入动态决定。

## 详细说明

### 1. 稀疏注意力机制

#### 滑动窗口注意力（Sliding Window Attention）

- 每个 token 只关注其周围 $w$ 个 token（如 $w = 512$）。
- 复杂度从 $O(n^2)$ 降至 $O(n \cdot w)$。
- 适合捕捉局部语言结构，但无法建模远距离依赖。

#### 全局注意力（Global Attention）

- 预先指定某些特殊 token（如 [CLS]、问题 token）拥有全剧注意力。
- 这些 token 作为"信息汇总器"汇聚全文信息。
- 适用于文档级分类、问答等需要全局感知任务。

#### 随机注意力（Random Attention）

- 每个 token 以固定概率 $p$ 随机关注其他 token。
- 提供额外的全局连接，有助于信息传播。
- BigBird 和 LONGformer 都使用了随机注意力。

#### 膨胀注意力（Dilated Attention）

- 类似膨胀卷积，在窗口内跳过固定间隔的 token。
- 扩大感受野的同时保持计算量可控。
- 可在局部窗口内捕获更长距离的依赖模式。

### 2. 层次化注意力（Hierarchical Attention）

- **分段处理**：将长文档分为多个 segment（如每段 512 tokens）。
- **层级结构**：段内注意力 -> 段间注意力 -> 文档级表示。
- **代表模型**：HLT（层次长程Transformer）、文档级 BERT。
- **优势**：天然支持任意长度文档，内存占用恒定。
- **挑战**：跨段边界信息传递需要额外的跨段连接机制。

### 3. 递归 Transformer（Recurrent Transformer）

- **核心思想**：将长文档切分为固定大小的段落，逐段处理并维护一个压缩的"记忆状态"。
- **代表工作**：Transformer-XL（记忆机制）、Compressive Transformer。
- **记忆机制**：将上一段的隐藏状态作为当前段的额外输入。
- **优势**：理论上可以处理无限长度的序列。
- **局限**：记忆状态有损压缩，远距离信息可能丢失。

### 4. 状态空间模型（State Space Models, SSM）

#### 从 RNN 到 SSM

- SSM 是连续状态的离散时间系统，比 RNN 更容易并行训练。
- Mamba 引入选择性机制，使状态转移矩阵 $\mathbf{A}$ 依赖于输入。
- 理论上支持 $O(n)$ 推理和线性复杂度的长距离依赖。

#### 选择性机制（Selection Mechanism）

$$\mathbf{B}_t = \mathbf{S}_B(\mathbf{x}_t), \quad \mathbf{C}_t = \mathbf{S}_C(\mathbf{x}_t), \quad \Delta_t = \tau_{\Delta}(\mathbf{S}_{\Delta}(\mathbf{x}_t))$$

通过输入依赖的门控机制，SSM 可以选择性地保留或遗忘信息。

#### 与 Transformer 的比较

| 维度 | Transformer | SSM (Mamba) |
|------|-------------|-------------|
| 推理复杂度 | $O(n^2)$ | $O(n)$ |
| 训练并行性 | 高度并行 | 线性并行 |
| 远程依赖 | 全注意机制 | 选择性压缩 |
| 任务适配性 | 通用 | 需要针对性优化 |

### 5. 检索增强的长上下文

- **RAG（Retrieval-Augmented Generation）**：将长文档分块后检索相关片段。
- **密度**：通过 DPR 等密集检索找到最相关的 chunks。
- **重排序**：使用 Cross-Encoder 对候选 chunks 重排序。
- **迭代检索**：多步迭代式检索以满足复杂问答需求。

## 代码示例

### 自实现稀疏注意力（局部窗口 + 全局）

```python
import torch
import torch.nn.functional as F
import math

def sparse_attention(Q, K, V, window_size=512, global_indices=None):
    """
    Q, K, V: (batch, heads, seq, dim)
    global_indices: 全局 token 的索引列表
    """
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)

    seq_len = Q.size(2)
    mask = torch.zeros_like(scores)

    # 局部窗口注意力掩码
    for i in range(seq_len):
        start = max(0, i - window_size)
        end = min(seq_len, i + window_size + 1)
        mask[:, :, i, start:end] = float('-inf')

    # 全局注意力掩码（设为 0，允许全部注意）
    if global_indices is not None:
        for idx in global_indices:
            mask[:, :, idx, :] = 0
            mask[:, :, :, idx] = 0

    scores = scores + mask
    attn_weights = F.softmax(scores, dim=-1)
    return torch.matmul(attn_weights, V)

# 示例
batch, heads, seq, dim = 2, 12, 2048, 64
Q = torch.randn(batch, heads, seq, dim)
K = torch.randn(batch, heads, seq, dim)
V = torch.randn(batch, heads, seq, dim)
output = sparse_attention(Q, K, V, window_size=256, global_indices=[0, 1])
```

### 递归 Transformer 伪代码

```python
def recurrent_forward(model, segments, memory=None, seg_len=512):
    """
    segments: list of token segments
    memory: previous segment's compressed state
    """
    all_outputs = []

    for seg in segments:
        if memory is not None:
            # 将记忆状态拼接到当前段
            seg = torch.cat([memory, seg], dim=1)

        output = model(seg)
        all_outputs.append(output[:, -seg_len:, :])  # 取最后 seg_len 个 token

        # 更新记忆状态
        memory = compress(output[:, :-seg_len, :])  # 压缩已处理的 token

    return all_outputs, memory

def compress(tensor, num_layers=2):
    """简单压缩：取平均后通过线性层投影"""
    compressed = tensor.mean(dim=1)  # (batch, dim)
    return compressed.unsqueeze(1)   # (batch, 1, dim)
```

### Mamba（SSM）简化实现

```python
import torch
import torch.nn as nn

class SimpleMambaBlock(nn.Module):
    """简化的 Mamba SSM 块"""
    def __init__(self, d_model, d_state=16, d_conv=4):
        super().__init__()
        self.d_model = d_model
        self.d_state = d_state

        # 输入投影
        self.x_proj = nn.Linear(d_model, d_state * 2, bias=False)
        self.dt_proj = nn.Linear(d_state, d_model, bias=True)

        # SSM 参数
        self.A = nn.Parameter(torch.randn(d_state, d_model))
        self.D = nn.Parameter(torch.ones(d_model))

    def forward(self, x):
        # x: (batch, seq, d_model)
        xz = self.x_proj(x)
        x_inner, z = xz.chunk(2, dim=-1)

        # 离散化 SSM（简化版）
        dt = F.softplus(self.dt_proj(x_inner.mean(dim=1)))  # (batch, d_state)

        # 选择性扫描（简化）
        y = torch.einsum('bsd,dm->bsm', x, self.A.sigmoid() * dt.unsqueeze(1))
        y = y + self.D * x

        # Gated residual
        return torch.nn.functional.silu(z) * y
```

## 应用场景

1. **超长文档理解**：处理整本图书、全套法律文件、多年历史记录。
2. **基因序列分析**：处理数千个氨基酸组成的长链蛋白质序列。
3. **视频理解**：将长视频的帧序列作为长上下文处理（视频字幕、视频摘要）。
4. **代码仓库分析**：理解整个代码库的依赖关系和架构。
5. **科学文献综述**：快速综述某领域数十年的相关论文。
6. **多模态长程依赖**：图像中远处区域之间的关系（如分割、检测）。
7. **具身智能**：机器人任务规划中需要理解长序列动作指令。

## 相关概念

- **稀疏注意力 (Sparse Attention)**：通过只计算部分 token 对注意力来降低 $O(n^2)$ 复杂度。
- **层次化注意力 (Hierarchical Attention)**：通过多层级表示逐步汇聚信息，适合超长文档。
- **递归 Transformer (Recurrent Transformer)**：通过记忆状态在段落间传递信息，支持无限长度。
- **状态空间模型 (SSM)**：选择性状态更新，理论上线性的时间和空间复杂度。
- **Mamba**：结合 SSM 选择性机制的新架构，兼顾效率和任务性能。
- **FlashAttention**：高效的注意力实现，是长上下文建模的计算基础。
- **上下文窗口 (Context Window)**：模型单次前向传播能处理的最大 token 数。
- **位置编码 (Positional Encoding)**：注入序列位置信息，对长上下文尤其关键。
