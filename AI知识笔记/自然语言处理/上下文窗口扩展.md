# 上下文窗口扩展

## 一句话定义

上下文窗口扩展是指通过改进位置编码或注意力机制，使大型语言模型能够有效处理远超其训练长度的超长文本序列的技术集合。

## 核心公式

### 线性位置插值（Linear Position Interpolation, LLaMA）

$$\text{pos} \rightarrow \frac{\text{pos}}{\alpha}, \quad \alpha = \frac{\text{新上下文长度}}{\text{原始上下文长度}}$$

将绝对位置索引按比例缩放，使模型在扩展后的位置范围内平滑外推。

### RoPE（Rotary Position Embedding）扩展因子

$$\mathbf{q}_i^{(new)} = \mathbf{R}_i^{(new)} \mathbf{x}_i, \quad \mathbf{R}_i^{(new)} = \bigoplus_{j=1}^{d/2} \begin{pmatrix} \cos(\theta_j \cdot m_i) & -\sin(\theta_j \cdot m_i) \\ \sin(\theta_j \cdot m_i) & \cos(\theta_j \cdot m_i) \end{pmatrix}$$

其中 $m_i$ 为扩展后的位置，$\theta_j = \theta_0 \cdot d^{-2j/\theta_0}$ 为旋转角度基数。

### YaRN 缩放

$$\text{温度缩放}: t \rightarrow t / s, \quad s = \sqrt{\frac{d}{d + 2 \log(\beta / \pi)}}$$

$$\text{位置缩放}: m \rightarrow m \cdot \frac{\beta}{\pi}$$

在注意力计算前对旋转角度进行缩放，减少高频维度的信息纠缠。

### BIGBIRD 稀疏注意力模式

$$\text{Attention}(i, j) = \begin{cases} \text{全局注意力} & \text{if } i \in \mathcal{G} \text{ or } j \in \mathcal{G} \\ \text{滑动窗口} & \text{if } |i - j| \leq w \\ \text{随机连接} & \text{if } j \in \mathcal{R}_i \\ 0 & \text{otherwise} \end{cases}$$

## 详细说明

### 1. 位置编码扩展

#### 位置插值（Position Interpolation, PI）

- **原理**：将超出原始训练范围的远距离位置映射到训练过的范围内。
- **方法**：线性缩放位置索引，例如将 4096 扩展到 32768 时，所有位置除以 8。
- **优点**：简单有效，保留大部分原有能力。
- **缺点**：对极长上下文（>100x）效果下降明显。

#### YaRN（Yet another RoPE extensioN）

- **核心改进**：在 RoPE 的基础上引入缩放因子，解决长距离位置编码退化问题。
- **关键洞察**：需要同时缩放注意力温度和位置，而不是单纯缩放位置。
- **效果**：在 128k 上下文长度下仍保持良好性能。

#### 梅花布局（NTK-aware Scaling）

- 将缩放分散到多个"位"（bits）上，而非线性插值单一维度。
- 在高频维度使用少量插值，在低频维度使用全量插值。
- 代表性工作：DynamiCIT、CLEX。

### 2. 注意力机制改造

#### LONGformer

- **设计**：局部窗口注意力 + 全局注意力 + 随机注意力。
- **窗口大小**：典型设置为 512（局部）+ 全文全局 tokens。
- **复杂度**：$O(n)$ 而非 $O(n^2)$，适合处理 16k+ 长度文档。
- **代表模型**：Longformer、BART-Long。

#### BigBird

- **设计**：类似 LONGformer 的稀疏模式，但使用更严谨的随机连接证明。
- **理论保证**：稀疏随机注意力可以逼近全注意力的效果。
- **代表模型**：BigBird-Pegasus（用于摘要任务）。

#### FlashAttention

- **核心思想**：通过 IO-aware 的 tiling 和 softmax 融合，减少 HBM 和 SRAM 之间的数据搬运。
- **加速比**：2-4x 端到端加速，同时降低内存占用至 $O(n)$。
- **兼容性**：无缝支持 RoPE 等位置编码方案。

### 3. 状态空间模型路径

- **Mamba（SSM）**：选择性状态空间模型，理论上支持无限长度上下文。
- **优势**：计算量与序列长度呈线性关系。
- **局限**：在需要精确信息检索的任务上略弱于注意力机制。

### 4. 微调策略

- **持续预训练（Continual Pretraining）**：在扩展上下文数据上继续训练。
- **数据配比**：混合短文本（保持原有能力）和长文本（学习新模式）。
- **中继训练（Relay Training）**：渐进式扩展，先到 16k，再 到 32k，逐步扩展。

## 代码示例

### PyTorch 实现 RoPE 缩放

```python
import torch
import math

def precompute_freqs_cis(dim: int, end: int, theta: float = 10000.0, scale: float = 1.0):
    """预计算旋转位置编码的频率矩阵"""
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2).float() / dim))
    t = torch.arange(end) * scale  # 应用缩放因子
    freqs = torch.outer(t, freqs)
    freqs = torch.polar(torch.ones_like(freqs), freqs)  # 转为复数形式
    return freqs

def apply_rotary_emb(q, k, freqs_cis):
    """将 RoPE 应用于 query 和 key"""
    q_complex = torch.view_as_complex(q.float().reshape(*q.shape[:-1], -1, 2))
    k_complex = torch.view_as_complex(k.float().reshape(*k.shape[:-1], -1, 2))
    q_rotated = q_complex * freqs_cis.unsqueeze(1)
    k_rotated = k_complex * freqs_cis.unsqueeze(1)
    return torch.view_as_real(q_rotated).flatten(-2), torch.view_as_real(k_rotated).flatten(-2)

# 示例：应用位置插值缩放
seq_len_orig = 2048
seq_len_new = 8192
scale = seq_len_orig / seq_len_new  # 线性插值缩放因子
freqs_cis = precompute_freqs_cis(dim=64, end=seq_len_new, scale=scale)
```

### HuggingFace Transformers 使用 Longformer

```python
from transformers import LongformerTokenizer, LongformerModel
import torch

tokenizer = LongformerTokenizer.from_pretrained("allenai/longformer-base-4096")
model = LongformerModel.from_pretrained("allenai/longformer-base-4096")

# LONGformer 需要配置注意力模式
config = model.config
config.attention_window = [512] * 12  # 每层窗口大小
config.attention_mode = "longformer"

text = "这是一段很长的文本..." * 500  # 模拟长文档
inputs = tokenizer(text, return_tensors="pt", max_length=4096, truncation=True)

with torch.no_grad():
    outputs = model(**inputs)
print(outputs.last_hidden_state.shape)
```

## 应用场景

1. **长文档分析**：法律合同、学术论文、技术文档的全文理解与问答。
2. **多轮对话**：在长对话历史中保持上下文连贯性（>100 轮）。
3. **代码补全**：处理完整代码文件而非截断的代码片段。
4. **长距离关系抽取**：跨段落、跨章节的关系和实体关联分析。
5. **视频/音频字幕处理**：将长时序数据作为上下文输入模型。
6. **RAG（检索增强生成）**：将多个检索文档拼接为长上下文进行生成。

## 相关概念

- **位置编码 (Positional Encoding)**：为序列中的每个位置添加位置信息的机制，是上下文扩展的基础。
- **RoPE (Rotary Position Embedding)**：旋转位置编码，通过旋转操作注入位置信息，支持高效的线性缩放。
- **位置插值 (Position Interpolation)**：将长距离位置索引压缩到训练范围内。
- **稀疏注意力 (Sparse Attention)**：只计算部分 token 对之间注意力的机制，大幅降低计算复杂度。
- **FlashAttention**：IO-aware 的高效注意力实现，兼顾速度和内存。
- **状态空间模型 (SSM)**：Mamba 等新型模型架构，理论上是线性复杂度的选择。
- **上下文窗口 (Context Window)**：模型一次能处理的最大 token 数量。
