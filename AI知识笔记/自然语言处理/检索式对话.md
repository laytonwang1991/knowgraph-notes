# 检索式对话系统

## 一句话定义

检索式对话系统通过在预先构建的对话库中匹配与用户输入最相关的回复来响应，而非生成新的文本。

## 核心公式

**双塔编码（Bi-Encoder）**：

$$
\mathbf{u} = \text{Enc}_U(\text{user\_input}), \quad \mathbf{r} = \text{Enc}_R(\text{response})
$$

**匹配分数（余弦相似度）**：

$$
\text{score}(\mathbf{u}, \mathbf{r}) = \frac{\mathbf{u} \cdot \mathbf{r}}{\|\mathbf{u}\| \|\mathbf{r}\|}
$$

**交互编码（Cross-Encoder）**：

$$
\text{score}(\text{user}, \text{response}) = \text{MLP}(\text{BERT}([\text{CLS}] \text{user} [\text{SEP}] \text{response}))
$$

**多轮上下文向量**：

$$
\mathbf{c}_t = \text{Agg}(\mathbf{u}_1, \mathbf{u}_2, \ldots, \mathbf{u}_t)
$$

## 详细说明

### 1. 系统架构

检索式对话系统的核心流程：

```
用户输入 → 编码 → 向量检索 → 候选排序 → 最优回复
```

**离线阶段**：
1. 构建对话库（QA对或多轮对话）
2. 使用编码器将所有候选回复向量化
3. 存储至向量索引（Faiss、Milvus等）

**在线阶段**：
1. 将用户输入编码为向量
2. ANN 近似最近邻检索
3. 重排序（Rerank）
4. 返回最优回复

### 2. 双塔模型（Bi-Encoder）

**结构**：

- 用户编码器：处理用户输入，输出向量 $\mathbf{u}$
- 回复编码器：处理候选回复，输出向量 $\mathbf{r}$
- 两者通常共享参数

**优点**：
- 推理速度快，适合大规模检索
- 离线预计算候选向量

**缺点**：
- 用户和回复无交互，信息损失

### 3. 交互模型（Cross-Encoder）

**结构**：

将用户输入和候选回复拼接后，一起通过Transformer编码器

**优点**：
- 深度交互建模，效果更好

**缺点**：
- 推理速度慢，需要对每个候选单独计算

### 4. 多轮上下文处理

单轮对话仅考虑当前输入，多轮对话需要整合历史上下文：

| 方法 | 说明 |
|------|------|
| 直接拼接 | 将历史 utterances 用特殊符号连接 |
| 层次化编码 | Utterance Encoder + Context Encoder |
| 记忆网络 | 使用外部记忆存储对话历史 |
| Transformer 聚合 | 用 self-attention 融合历史 |

### 5. 匹配模型演进

| 模型 | 年份 | 特点 |
|------|------|------|
| DSSM | 2013 | 深度语义匹配，双塔结构 |
| SMN | 2017 | 匹配矩阵 + CNN |
| DAM | 2018 | Transformer-style 交互 |
| BERT | 2019 | 预训练 + Cross-Encoder |
| SimBERT | 2020 | 检索 + 生成一体化 |

## 代码示例

### Bi-Encoder 实现

```python
import torch
import torch.nn as nn
from transformers import BertModel, BertTokenizer

class BiEncoder(nn.Module):
    def __init__(self, model_name="bert-base-chinese"):
        super().__init__()
        self.bert = BertModel.from_pretrained(model_name)
        self.pooler = nn.Linear(768, 768)

    def encode_user(self, input_ids, attention_mask):
        outputs = self.bert(input_ids=input_ids, attention_mask=attention_mask)
        # 使用 [CLS] 向量
        cls_output = outputs.last_hidden_state[:, 0]
        return self.pooler(cls_output)

    def encode_response(self, input_ids, attention_mask):
        # 与 encode_user 共享编码器
        return self.encode_user(input_ids, attention_mask)

    def forward(self, user_ids, user_mask, resp_ids, resp_mask):
        user_vec = self.encode_user(user_ids, user_mask)
        resp_vec = self.encode_response(resp_ids, resp_mask)
        # 余弦相似度
        user_vec = torch.nn.functional.normalize(user_vec, dim=-1)
        resp_vec = torch.nn.functional.normalize(resp_vec, dim=-1)
        similarity = torch.sum(user_vec * resp_vec, dim=-1)
        return similarity
```

### 向量检索（Faiss）

```python
import faiss
import numpy as np

class VectorRetriever:
    def __init__(self, dim=768, top_k=10):
        self.index = faiss.IndexFlatIP(dim)  # Inner Product = Cosine when normalized
        self.top_k = top_k
        self.response_list = []

    def build_index(self, response_vectors, responses):
        """构建索引"""
        vectors = np.array(response_vectors).astype('float32')
        faiss.normalize_L2(vectors)  # 归一化
        self.index.add(vectors)
        self.response_list = responses

    def search(self, query_vector, top_k=None):
        """检索"""
        query = np.array([query_vector]).astype('float32')
        faiss.normalize_L2(query)
        top_k = top_k or self.top_k
        scores, indices = self.index.search(query, top_k)
        results = [(self.response_list[i], scores[0][j])
                   for j, i in enumerate(indices[0])]
        return results
```

### 多轮上下文处理

```python
def build_context(utterances, max_turns=5, sep_token=" [SEP] "):
    """构建多轮上下文"""
    # 取最近 max_turns 轮对话
    recent = utterances[-max_turns:] if len(utterances) > max_turns else utterances
    context = sep_token.join(recent)
    return context

class HierarchicalEncoder(nn.Module):
    def __init__(self, bert_model):
        super().__init__()
        self.bert = bert_model

    def encode_utterance(self, text):
        inputs = self.bert(**text)
        # [CLS] 向量
        return inputs.last_hidden_state[:, 0]

    def encode_context(self, utterance_vectors):
        # 使用 self-attention 聚合
        stacked = torch.stack(utterance_vectors, dim=1)  # [B, T, H]
        # 简化的聚合：取平均
        context_vec = stacked.mean(dim=1)
        return context_vec
```

## 应用场景

1. **客服机器人**：企业级FAQ问答系统
2. **社交对话**：情感陪伴机器人的检索回复
3. **知识库问答**：从问答库中检索最相关的答案
4. **垂直领域助手**：法律、医疗等专业领域咨询
5. **智能音箱**：预设回复的快速响应

## 相关概念

- **生成式对话**：端到端生成回复，而非检索
- **双塔模型**：用户和回复分别编码的检索架构
- **向量检索**：使用 ANN 加速大规模向量搜索
- **重排序**：先用ANN粗排，再用Cross-Encoder精排
- **BERT**：预训练语言模型，可用于检索编码
