# 分词 (Tokenization)

## 一句话定义

分词是将连续文本序列分割成离散语言单元（词、词素或子词）的预处理步骤，是NLP管道中的首个关键环节。

## 核心公式

### 基于词典的正向最大匹配 (MM)

```
设词典D，句子S[1:n]，最大词长L
for i = 1 to n:
    for j = min(n-i+1, L) down to 1:
        if S[i:i+j] ∈ D:
            output S[i:i+j]
            i += j
            break
```

### N-gram 分词概率

```
P(w1,w2,...,wm) ≈ ∏ P(wi|wi-n+1,...,wi-1)
```

### BPE (Byte Pair Encoding) 合并规则

```
merge_count = count of all adjacent pairs → select most frequent pair → merge
```

## 详细说明

### 1. 分词类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 词级分词 | 分割成完整词语 | 英文等空格分隔语言 |
| 子词分词 | 分割成更小单元 | 形态丰富语言、神经网络模型 |
| 字符级分词 | 分割成单个字符 | 未知词处理、跨语言模型 |

### 2. 主流分词算法

**正向最大匹配 (MM)**
- 从左到右贪婪匹配最长词
- 简单高效，但可能产生歧义

**双向最大匹配 (BiMM)**
- 同时执行正向和反向最大匹配
- 选择词数最少或单字词最少的结果

**条件随机场 (CRF)**
- 序列标注方法，输出BMES标签
- 可学习上下文特征，精度高

**子词分词 (BPE/WordPiece/BBPE)**
- BPE: 统计合并高频字节对
- WordPiece: 基于语言模型选择合并
- 解决OOV问题，适合预训练模型

### 3. 中文分词的特殊挑战

- 词边界模糊：歧义字段（如"商品和服务"）
- 未登录词问题：人名、地名、机构名
- 嵌套歧义：多粒度理解需求

### 4. 分词质量评估

```
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1 = 2 × Precision × Recall / (Precision + Recall)
```
