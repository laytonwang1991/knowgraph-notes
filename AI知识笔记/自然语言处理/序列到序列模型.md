# 序列到序列模型 (Sequence to Sequence, Seq2Seq)

## 一句话定义

Seq2Seq是一种编码器-解码器架构，将任意长度的输入序列编码为语义向量，再由解码器生成任意长度的目标序列。

## 核心公式

### 编码器

```
h_t = f(x_t, h_{t-1})
c = q({h_1, h_2, ..., h_n})
其中:
  h_t = 时刻t的隐藏状态
  f = RNN/LSTM/GRU变换函数
  c = 上下文向量(最终语义表示)
```

### 解码器

```
s_t = g(y_{t-1}, s_{t-1}, c)
P(y_t|y_{<t}, c) = softmax(W·s_t)
其中:
  s_t = 解码器隐藏状态
  y_{t-1} = 上一个输出
  c = 上下文向量
```

### 注意力机制

```
a_{ti} = align(s_{t-1}, h_i) = exp(score(s_{t-1}, h_i)) / Σ exp(score(s_{t-1}, h_j))
c_t = Σ a_{ti} × h_i

score函数选择:
  - Dot: s^T · h
  - MLP: v^T · tanh(W·s + U·h)
  - Bilinear: s^T · W · h
```

### 训练损失

```
L = -Σ log P(y_t|y_{<t}, x)
  = -Σ log softmax(W·s_t + b)[y_t]
```

## 详细说明

### 1. 架构概述

```
输入序列: X = (x1, x2, ..., xn)
     ↓
[编码器 Encoder]
     ↓
上下文向量: c = encoder(x1...xn)
     ↓
[解码器 Decoder]
     ↓
输出序列: Y = (y1, y2, ..., ym)
```

### 2. 编码器类型

| 类型 | 特点 | 代表模型 |
|------|------|----------|
| RNN | 顺序计算，梯度问题 | basic Seq2Seq |
| LSTM | 门控机制，长程依赖 | Google NMT |
| GRU | 简化门控，计算高效 | GRU-NMT |
| BiRNN | 双向捕获上下文 | 通用 |
| Transformer | 并行计算，全局注意力 | 主流 |

### 3. 解码策略

**贪婪解码 (Greedy)**
```
y_t = argmax P(y|y_{<t}, c)
```
- 简单但误差累积

**束搜索 (Beam Search)**
```
维护K个最优候选序列
每步扩展K个分支，保留Top-K
```
- 平衡质量和效率

**随机采样 (Sampling)**
```
从概率分布随机采样
可配合Temperature控制多样性
```
- 高多样性，适合创意生成

### 4. 注意力改进

**自注意力 (Self-Attention)**
- 输入序列内部的注意力
- 捕获词间依赖关系

**多头注意力 (Multi-Head)**
```
MultiHead(Q,K,V) = Concat(head_1,...,head_h) × W^O
head_i = Attention(Q×W_i^Q, K×W_i^K, V×W_i^V)
```

### 5. 典型应用

| 应用 | 输入 | 输出 |
|------|------|------|
| 机器翻译 | 英文句子 | 中文句子 |
| 文本摘要 | 长文档 | 短摘要 |
| 问答系统 | 问题+上下文 | 答案 |
| 对话系统 | 用户消息 | 机器人回复 |
