# 词嵌入

## 一句话定义

> 词嵌入是将离散的词语映射到连续稠密向量空间的技术，让语义相似的词拥有相似的向量表示。

## 核心公式

### Word2Vec Skip-gram

目标函数（负采样）：
$$
\mathcal{L} = \sum_{t=1}^{T} \sum_{c \in C(t)} \log \sigma(v_c^T v_t) + \sum_{j=1}^{k} \mathbb{E}_{w_j \sim P_n(w)} [\log \sigma(-v_{w_j}^T v_t)]
$$

其中 $v_t$ 是中心词向量，$v_c$ 是上下文词向量。

## 详细说明

### 1. 为什么需要词嵌入？

**One-Hot 编码的问题：**
- 维度高且稀疏
- 无法表达语义相似性
- 无法进行运算

**词嵌入的优势：**
- 维度低（通常100-300维）
- 稠密表示
- 语义相似 → 向量相似

### 2. Word2Vec

Google 2013年提出的词嵌入方法，包含两种模型：

**Skip-gram：** 用中心词预测上下文
```
the cat sits on the [mat]
中心词"cat" → 预测"the", "sits", "on", "the", "mat"
```

**CBOW：** 用上下文预测中心词
```
the [cat] sits on the mat
上下文"the", "sits", "on", "the", "mat" → 预测"cat"
```

### 3. 预训练词嵌入

| 模型 | 年份 | 特点 |
|------|------|------|
| Word2Vec | 2013 | 高效，静态词向量 |
| GloVe | 2014 | 全局共现统计 + 局部上下文 |
| FastText | 2016 | 子词嵌入，处理OOV |
| ELMo | 2018 | 动态词向量，上下文相关 |

### 4. 代码示例

```python
from gensim.models import Word2Vec

# 训练语料
sentences = [
    ['cat', 'sat', 'on', 'the', 'mat'],
    ['dog', 'sat', 'on', 'the', 'log'],
]

# 训练Word2Vec模型
model = Word2Vec(
    sentences,
    vector_size=100,    # 词向量维度
    window=2,          # 上下文窗口
    min_count=1,      # 最小词频
    workers=4,        # 并行线程数
    sg=1              # 1=Skip-gram, 0=CBOW
)

# 获取词向量
vector = model.wv['cat']

# 找相似词
similar = model.wv.most_similar('cat')
```

### 5. 现代方法

Transformer时代，BERT等模型产生**上下文相关的词嵌入**：
- 每个词在不同上下文中有不同向量
- 动态表示，比静态词嵌入更强大
