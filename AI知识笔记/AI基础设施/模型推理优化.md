# 模型推理优化

## 一句话定义

模型推理优化是通过各种技术手段降低深度学习模型的推理延迟、提高吞吐量、减少内存占用的系统性工程实践。

## 详细说明

### 1. 批量推理（Batch Inference）

批量推理是指将多个推理请求合并为一个批次同时处理，利用GPU的并行计算能力提高吞吐量。

**核心原理：**
- GPU在处理矩阵运算时，批量越大，硬件利用率越高
- 减少了每个样本的启动开销（kernel launch overhead）
- 内存访问模式更友好，提高显存带宽利用率

**实现方式：**
```python
import torch

# 动态批量：累积请求直到达到 batch_size 上限
def dynamic_batch_inference(model, requests, max_batch_size=32, max_wait_ms=10):
    """
    动态批量推理
    - requests: 待推理的输入列表
    - max_batch_size: 最大批量大小
    - max_wait_ms: 最大等待时间（毫秒）
    """
    batch = []
    start_time = time.time()

    while requests:
        batch.append(requests.pop(0))

        # 达到最大批量或等待超时，执行推理
        if len(batch) >= max_batch_size or \
           (time.time() - start_time) * 1000 >= max_wait_ms:

            with torch.no_grad():
                outputs = model(torch.stack(batch))
            batch = []
            start_time = time.time()

    # 处理剩余请求
    if batch:
        with torch.no_grad():
            outputs = model(torch.stack(batch))

    return outputs
```

### 2. KV Cache

KV Cache 是 Transformer 模型推理中的关键技术，通过缓存已计算的键值对来避免重复计算。

**核心原理：**
- 自注意力机制中，每个token需要 attending 到之前所有token
- 生成第 N 个 token 时，无需重新计算前 N-1 个 token 的 Key 和 Value
- 将空间复杂度从 O(N²) 降低到 O(N)

**实现方式：**
```python
class KVCache优化推理:
    """带 KV Cache 的自注意力推理"""

    def __init__(self, model):
        self.k_cache = {}
        self.v_cache = {}

    def forward_with_cache(self, layer_idx, q, k, v, past_kv=None):
        """
        带 KV Cache 的前向传播
        - q: 当前 query (1, num_heads, seq_len, head_dim)
        - k, v: 当前 key/value
        - past_kv: 历史的 KV cache
        """
        if past_kv is not None:
            # 拼接历史 cache 和当前输入
            k = torch.cat([past_kv[0], k], dim=-2)
            v = torch.cat([past_kv[1], v], dim=-2)

        # 更新 cache
        self.k_cache[layer_idx] = k
        self.v_cache[layer_idx] = v

        # 计算注意力
        attn_weights = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(q.shape[-1])
        attn_weights = F.softmax(attn_weights, dim=-1)
        attn_output = torch.matmul(attn_weights, v)

        return attn_output, (k, v)

    def generate(self, model, prompt_tokens, max_new_tokens=100):
        """使用 KV Cache 生成文本"""
        input_ids = torch.tensor(prompt_tokens).unsqueeze(0)

        for _ in range(max_new_tokens):
            with torch.no_grad():
                output = model(input_ids)

            # 只取最后一个 token 的 logits
            next_token_logits = output[:, -1, :]
            next_token = torch.argmax(next_token_logits, dim=-1, keepdim=True)
            input_ids = torch.cat([input_ids, next_token], dim=-1)

            # KV Cache 自动在 model 内部维护
            if next_token.item() == model.config.eos_token_id:
                break

        return input_ids
```

### 3. 连续批处理（Continuous Batching）

连续批处理（又称 Iteration-level Scheduling）解决了静态批处理的资源利用率问题。

**核心原理：**
- 传统静态批处理：整个批次完成后才能处理新请求（First Finished, First Scheduled）
- 连续批处理：单个序列完成后立即释放，新序列可以立即加入
- 大幅提高 GPU 利用率，特别是在序列长度差异大的场景

**实现方式：**
```python
from typing import List, Dict
import heapq

class ContinuousBatchingScheduler:
    """
    连续批处理调度器
    - 以 iteration 为单位调度，而非请求级别
    - 完成的序列立即退出，新序列立即加入
    """

    def __init__(self, model, max_batch_size=32):
        self.model = model
        self.max_batch_size = max_batch_size
        self.running_seqs: List[Sequence] = []
        self.pending_seqs: List[Sequence] = []

    def add_request(self, request_id, prompt_tokens):
        """添加新请求"""
        seq = Sequence(request_id, prompt_tokens)
        self.pending_seqs.append(seq)

    def step(self):
        """
        执行一步推理
        - 调度新序列加入
        - 对当前批次执行一步 forward
        - 移除已完成的序列
        """
        # 1. 调度：从 pending 中补充新序列到 running
        while len(self.running_seqs) < self.max_batch_size and self.pending_seqs:
            seq = self.pending_seqs.pop(0)
            self.running_seqs.append(seq)

        if not self.running_seqs:
            return []

        # 2. 构建当前批次的输入
        batch_input_ids = [seq.input_ids for seq in self.running_seqs]

        # 3. 执行推理（截断到当前最大长度以节省计算）
        max_len = max(len(ids) for ids in batch_input_ids)
        padded_input = torch.stack([
            self._pad_sequence(ids, max_len) for ids in batch_input_ids
        ])

        with torch.no_grad():
            outputs = self.model(padded_input)

        # 4. 处理输出，更新序列
        finished_seqs = []
        for i, seq in enumerate(self.running_seqs):
            next_token_logits = outputs[i, -1, :]
            next_token = torch.argmax(next_token_logits, dim=-1).item()

            seq.input_ids.append(next_token)

            if next_token == seq.eos_token_id:
                finished_seqs.append(seq)

        # 5. 移除已完成的序列
        self.running_seqs = [s for s in self.running_seqs if s not in finished_seqs]

        return finished_seqs
```

### 4. 投机解码（Speculative Decoding）

投机解码通过使用小模型预测多个token，然后大模型验证的策略加速推理。

**核心原理：**
- 使用小模型（draft model）快速生成候选 token 序列
- 大模型（verifier model）并行验证这些候选
- 接受率通常很高，大幅减少大模型的 forward 次数

**实现方式：**
```python
class SpeculativeDecoding:
    """
    投机解码实现
    - draft_model: 小而快的模型
    - verifier_model: 大而准确的模型
    """

    def __init__(self, draft_model, verifier_model, gamma=4):
        self.draft = draft_model
        self.verifier = verifier_model
        self.gamma = gamma  # 每次猜测的 token 数量

    def generate(self, prompt, max_new_tokens=100):
        input_ids = torch.tensor(prompt).unsqueeze(0)
        generated = list(input_ids[0].numpy())

        while len(generated) < max_new_tokens:
            # Step 1: Draft model 生成 gamma 个候选 token
            draft_input = torch.tensor(generated).unsqueeze(0)
            draft_outputs = []

            with torch.no_grad():
                for _ in range(self.gamma):
                    output = self.draft(draft_input)
                    next_token = torch.argmax(output[:, -1, :], dim=-1).item()
                    draft_outputs.append(next_token)
                    draft_input = torch.cat([draft_input, torch.tensor([[next_token]])], dim=-1)
                    generated.append(next_token)

                    if next_token == self.draft.config.eos_token_id:
                        break

            # Step 2: Verifier model 并行验证所有候选
            candidate_ids = torch.tensor([generated[-len(draft_outputs):]]).unsqueeze(0)

            with torch.no_grad():
                verifier_output = self.verifier(candidate_ids)

            # Step 3: 验证并调整
            accepted_tokens = []
            for i, draft_tok in enumerate(draft_outputs):
                # 对比 draft 和 verifier 的概率分布
                draft_prob = F.softmax(self.draft.last_logits, dim=-1)[0, -1]
                verifier_prob = F.softmax(verifier_output.logits[0, -(len(draft_outputs)-i):], dim=-1)

                # 验证每个 token
                if torch.argmax(verifier_prob, dim=-1)[-1] == draft_tok:
                    accepted_tokens.append(draft_tok)
                else:
                    # 首次拒绝，后续的也都拒绝
                    break

            # Step 4: 采样最终 token（基于 verifier 分布）
            if accepted_tokens:
                final_token = torch.multinomial(F.softmax(verifier_output.logits[0, -1], dim=-1), 1).item()
                generated = generated[:-len(draft_outputs)] + accepted_tokens + [final_token]

        return generated
```

## 应用场景

| 场景 | 推荐技术 | 预期收益 |
|------|----------|----------|
| LLM 文本生成 | KV Cache + 连续批处理 | 延迟降低 50%+，吞吐提升 3-5x |
| 多模型 Ensemble | 批量推理 | GPU 利用率提升 2-3x |
| 长序列推理 | KV Cache + 稀疏注意力 | 内存降低 70%+ |
| 实时对话系统 | 投机解码 + 连续批处理 | 首 token 延迟降低 60%+ |
| 批量数据处理 | 静态批量推理 | 吞吐提升 10x+ |

## 相关概念

- **ONNX Runtime**: 通用的跨平台推理引擎，支持多种优化
- **TensorRT**: NVIDIA 的高性能推理优化库
- **量化部署**: 通过 INT8/FP16 量化进一步加速
- **模型压缩**: 知识蒸馏、剪枝等技术减小模型体积
- **Flash Attention**: 注意力机制的 IO 优化实现
