# 学习率调度

## 一句话定义

学习率调度是在训练过程中根据预设策略动态调整学习率的技术，旨在平衡收敛速度与最终模型性能。

## 详细说明

### 什么是学习率调度

学习率是控制模型权重更新步幅的超参数。固定学习率可能导致：初期收敛太慢，或后期在最优解附近震荡。学习率调度通过在训练不同阶段使用不同学习率来解决这一问题。

### 常见调度策略

#### 1. Warmup（预热）

在训练初期将学习率从极小值逐渐增加到目标值。

- **线性预热**：学习率线性递增
- **阶梯预热**：分阶段递增
- **优点**：防止早期梯度爆炸，稳定训练
- **适用场景**：大Batch训练、BERT等大型模型

#### 2. Cosine Decay（余弦退火）

学习率按余弦函数曲线从峰值衰减到最小值。

```
η_t = η_min + (η_max - η_min) × (1 + cos(πt/T)) / 2
```

- **优点**：平滑衰减，适合图像分类
- **变体**：带Warmup的余弦退火（Cosine with Warmup）

#### 3. Linear Decay（线性衰减）

学习率随步数线性下降。

```
η_t = η_max - (η_max - η_min) × t/T
```

- **优点**：实现简单
- **缺点**：衰减过于均匀

#### 4. Step Decay（阶梯衰减）

每隔固定 epoch 学习率下降固定比例。

```python
lr = lr × drop_rate  # 每N个epoch执行一次
```

- **优点**：实现简单，适合大规模训练
- **缺点**：不夠平滑

#### 5. OneCycle（单周期）

分为预热阶段（提升学习率）和稳定阶段（降低学习率），总周期不超过一个epoch。

- **优点**：加速收敛，减少训练时间
- **研究支持**：Smith (2018) 表明 OneCycle 可达到更高准确率

### 学习率调度的实现要点

1. **起始学习率选择**：基于经验或小规模实验
2. **最小学习率**：通常设为起始值的 1/100 到 1/1000
3. **调度时机**：每个 epoch 后或每个 batch 后更新
4. **配合优化器**：学习率调度器通常包装现有优化器

## 代码示例

```python
import torch
from torch.optim.lr_scheduler import CosineAnnealingWarmRestarts, StepLR, OneCycleLR

# 余弦退火
scheduler = CosineAnnealingWarmRestarts(
    optimizer,
    T_0=10,           # 第一个周期长度
    T_mult=2,         # 周期倍数
    eta_min=1e-6      # 最小学习率
)

# 阶梯衰减
scheduler = StepLR(
    optimizer,
    step_size=30,
    gamma=0.1         # 每30 epoch学习率乘0.1
)

# OneCycle（PyTorch Lightning风格）
scheduler = OneCycleLR(
    optimizer,
    max_lr=1e-3,
    epochs=100,
    steps_per_epoch=len(train_loader),
    pct_start=0.3,    # 前30%用于预热
    anneal_strategy='cos'
)

# 训练循环
for epoch in range(num_epochs):
    train(...)
    scheduler.step()
```

### 自定义学习率调度器

```python
class WarmupCosineScheduler:
    def __init__(self, optimizer, warmup_epochs, total_epochs, base_lr, min_lr=0):
        self.optimizer = optimizer
        self.warmup_epochs = warmup_epochs
        self.total_epochs = total_epochs
        self.base_lr = base_lr
        self.min_lr = min_lr

    def step(self, epoch):
        if epoch < self.warmup_epochs:
            lr = self.base_lr * (epoch + 1) / self.warmup_epochs
        else:
            progress = (epoch - self.warmup_epochs) / (self.total_epochs - self.warmup_epochs)
            lr = self.min_lr + (self.base_lr - self.min_lr) * (1 + math.cos(math.pi * progress)) / 2
        for param_group in self.optimizer.param_groups:
            param_group['lr'] = lr
        return lr
```

## 应用场景

| 场景 | 推荐策略 | 原因 |
|------|---------|------|
| 大模型预训练（BERT、GPT） | Warmup + Cosine Decay | 稳定训练，防止早期梯度爆炸 |
| 图像分类（ResNet等） | Cosine Annealing | 平滑衰减，提升泛化能力 |
| 强化学习 | Linear Decay | 平衡探索与利用 |
| 小数据集快速实验 | Step Decay | 实现简单，调参方便 |
| 超参数搜索 | OneCycle | 训练快速，效果稳定 |

## 相关概念

- **自适应学习率优化器**（Adam、RMSprop）：自身具有学习率适应特性，但配合调度器效果更佳
- **学习率区间搜索**（LR Range Test）：确定最佳学习率范围
- **梯度裁剪**：与学习率调度配合，防止梯度爆炸
- **早停**（Early Stopping）：学习率衰减到一定程度后停止训练
