# 梯度裁剪

## 一句话定义

梯度裁剪是通过限制梯度范数或值的范围来防止梯度爆炸的技术，是训练深度神经网络（尤其是RNN）的关键技巧。

## 详细说明

### 什么是梯度裁剪

在深度神经网络反向传播中，梯度逐层累积可能导致数值爆炸（梯度 > 阈值），使训练发散。梯度裁剪将超过阈值的梯度"裁剪"到合理范围，保证训练稳定。

### 梯度裁剪方法

#### 1. 全局范数裁剪（Gradient Clipping by Norm）

最常用的方法，将所有参数的梯度向量整体缩放。

```python
# 伪代码
total_norm = sqrt(sum(grad_i^2 for all grad_i))
clip_coef = max_norm / total_norm
if clip_coef < 1:
    grad = grad * clip_coef
```

```python
# PyTorch实现
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

**优点**：
- 保持梯度方向信息
- 自适应缩放比例
- 适合大规模模型

#### 2. 值裁剪（Gradient Clipping by Value）

直接限制每个梯度值的范围。

```python
# 裁剪到[-clip_value, clip_value]
torch.nn.utils.clip_grad_value_(model.parameters(), clip_value=1.0)
```

**缺点**：破坏梯度方向信息，可能影响收敛

#### 3. 梯度累积（Gradient Accumulation）

在不增大batch size的情况下模拟大batch训练效果。

```python
# 积累N个mini-batch的梯度，再更新一次
accumulation_steps = 4
for i, (inputs, labels) in enumerate(dataloader):
    outputs = model(inputs)
    loss = criterion(outputs, labels) / accumulation_steps
    loss.backward()

    if (i + 1) % accumulation_steps == 0:
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        optimizer.step()
        optimizer.zero_grad()
```

**应用场景**：
- GPU显存不足时的替代方案
- 需要大effective batch size的场景

#### 4. Gradient Checkpointing（梯度检查点）

用时间换空间的技术，通过重新计算中间激活值来减少显存占用。

```python
# PyTorch实现
from torch.utils.checkpoint import checkpoint

class Model(nn.Module):
    def forward(self, x):
        # 对计算密集的层使用checkpoint
        x = checkpoint(self.layer1, x)
        x = checkpoint(self.layer2, x)
        return x
```

**原理**：
- 前向传播时不保存所有中间激活值
- 反向传播时重新计算所需激活值
- 显存减少约 1/3 到 1/2

### 梯度裁剪的使用场景

| 场景 | 推荐方法 | 阈值建议 |
|------|---------|---------|
| RNN/LSTM/GRU | 范数裁剪 | 1.0 - 5.0 |
| Transformer | 范数裁剪 | 1.0 - 5.0 |
| 图像分类（CNN） | 通常不需要 | - |
| 强化学习 | 范数裁剪 | 0.5 - 2.0 |
| GAN训练 | 范数裁剪 | 0.01 - 1.0 |

### RNN中梯度裁剪的重要性

RNN由于循环结构，梯度在时间步上累积，极易发生梯度爆炸：

```
h_t = tanh(W·h_{t-1} + U·x_t)
```

- 梯度沿时间步反向传播时，每步乘以权重矩阵
- 如果权重矩阵特征值 > 1，梯度指数增长
- **LSTM/GRU的门控机制能缓解，但梯度裁剪仍是保险措施**

## 代码示例

### 基本使用

```python
import torch
import torch.nn as nn
import torch.optim as optim

# 训练循环
for epoch in range(num_epochs):
    for batch in dataloader:
        optimizer.zero_grad()

        inputs, labels = batch
        outputs = model(inputs)
        loss = criterion(outputs, labels)

        loss.backward()

        # 梯度裁剪 - 防止梯度爆炸
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

        optimizer.step()
```

### RNN完整示例

```python
import torch
import torch.nn as nn

class SimpleRNN(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.rnn = nn.RNN(embed_dim, hidden_dim, batch_first=True)
        self.fc = nn.Linear(hidden_dim, vocab_size)

    def forward(self, x):
        embedded = self.embedding(x)
        output, hidden = self.rnn(embedded)
        return self.fc(output)

# 训练函数
def train_epoch(model, dataloader, optimizer, criterion, device):
    model.train()
    total_loss = 0

    for batch in dataloader:
        batch = batch.to(device)
        optimizer.zero_grad()

        outputs = model(batch)
        # 假设预测下一个token
        loss = criterion(outputs[:, :-1].contiguous().view(-1, outputs.size(-1)),
                         batch[:, 1:].contiguous().view(-1))

        loss.backward()

        # RNN必须使用梯度裁剪
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=5.0)

        optimizer.step()
        total_loss += loss.item()

    return total_loss / len(dataloader)
```

### 自定义裁剪策略

```python
def gradient_clipping(model, max_norm, norm_type=2):
    """自定义梯度裁剪"""
    total_norm = 0.0

    # 计算总梯度范数
    for p in model.parameters():
        if p.grad is not None:
            param_norm = p.grad.data.norm(norm_type)
            total_norm += param_norm.item() ** norm_type

    total_norm = total_norm ** (1.0 / norm_type)
    clip_coef = max_norm / (total_norm + 1e-6)

    if clip_coef < 1:
        for p in model.parameters():
            if p.grad is not None:
                p.grad.data.mul_(clip_coef)

    return total_norm
```

## 应用场景

| 场景 | 是否需要裁剪 | 原因 |
|------|------------|------|
| RNN文本生成 | 必须 | 时间步累积导致梯度爆炸 |
| Transformer | 推荐 | 深层网络可能有风险 |
| ResNet等CNN | 通常不需要 | BatchNorm + Skip Connection缓解 |
| GAN | 必须 | 判别器梯度易爆炸 |
| 强化学习 | 必须 | 策略梯度方差大 |

## 相关概念

- **梯度消失**：与梯度爆炸相对，通常由Sigmoid/Tanh引起
- **LSTM/GRU**：通过门控机制缓解RNN的梯度问题
- **残差连接**：缓解深层网络的梯度问题
- **Batch Normalization**：稳定训练，但不完全替代梯度裁剪
- **权重衰减**（L2正则）：与梯度裁剪有协同作用
