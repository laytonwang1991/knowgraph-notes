# 分布式深度学习训练

## 一句话定义

> 分布式深度学习训练通过将训练任务分配到多个GPU或多个计算节点上，利用并行处理能力加速大规模模型的训练，同时管理参数同步和通信开销。

## 核心公式

### 数据并行梯度同步

$$
W_{new} = W_{old} - \eta \cdot \frac{1}{N} \sum_{i=1}^{N} \nabla W_i
$$

其中 $N$ 为GPU数量，每个GPU计算局部梯度后通过AllReduce同步。

### ZeRO Stage 1-3

| Stage | 优化内容 | 内存节省 |
|-------|---------|---------|
| ZeRO-1 | 分片optimizer states | 4x |
| ZeRO-2 | 分片gradients + optimizer states | 8x |
| ZeRO-3 | 分片parameters + gradients + optimizer states | 最大 |

## 详细说明

### 1. 并行策略

**数据并行 (Data Parallelism)**
- 每个GPU持有完整模型副本
- 不同batch数据分配到不同GPU
- 梯度同步后更新参数
- 适合GPU数量多、模型可单卡加载

**模型并行 (Model Parallelism)**
- 模型拆分到多个GPU
- 流水线并行：不同层在不同GPU
- 张量并行：层内分片
- 适合超大模型无法单卡加载

**混合并行 (Hybrid Parallelism)**
- 多节点数据并行 + 节点内模型并行
- 结合两者优势

### 2. 通信策略

**Ring-AllReduce**
```
GPU0 → GPU1 → GPU2 → GPU3 → GPU0
```
- 带宽最优
- 无中心节点瓶颈

**Tree-AllReduce**
- 层级聚合
- 延迟更低但带宽不均衡

### 3. 同步方式

| 方式 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| Synchronous | 等待所有GPU梯度完成后同步更新 | 收敛稳定 | GPU空闲等待 |
| Asynchronous | 各GPU独立更新 | GPU利用率高 | 可能不稳定 |

### 4. 主流框架

**PyTorch DDP**
```python
import torch.distributed as dist

dist.init_process_group(backend='nccl')
model = nn.parallel.DistributedDataParallel(model)
```

**Horovod (Uber)**
```python
import horovod.torch as hvd

hvd.init()
model = hvd.DistributedOptimizer(model)
```

**DeepSpeed (Microsoft)**
- ZeRO优化器
- 混合精度训练
- Gradient Checkpointing

### 5. 训练配置建议

| 参数 | 推荐值 |
|------|--------|
| 通信后端 | NCCL (GPU) / Gloo (CPU) |
| 梯度累积 | 配合大batch使用 |
| 混合精度 | FP16/BF16 |
| 异步预取 | DataLoader workers > 4 |

## 优缺点

| 优点 | 缺点 |
|------|------|
| 线性加速比 (理想情况) | 通信开销 |
| 支持超大模型训练 |调试复杂 |
| 资源利用率高 | 同步开销 |
