# 结构化稀疏

## 一句话定义

结构化稀疏以规则的结构（如N:M模式、Channel、Head等）为单位进行稀疏化，在保持硬件友好计算的同时实现模型压缩与加速。

## 核心公式

### 1. N:M 稀疏（N:M Sparsity）

在每M个连续权重中，保留N个非零值：

$$
\forall i \in [0, \frac{|\mathbf{w}|}{M} - 1], \quad \text{nnz}(\mathbf{w}_{i \cdot M : (i+1) \cdot M}) = N
$$

- NVIDIA A100支持2:4稀疏（每4个元素保留2个非零）
- 稀疏矩阵可无损压缩为紧凑格式，2:4稀疏加速比为2x

### 2. 结构化稀疏度（Structured Sparsity Ratio）

$$
s_{struct} = 1 - \frac{\text{num\_structural\_units\_kept}}{\text{total\_structural\_units}}
$$

- Channel Pruning: 以通道为单位的稀疏度
- Filter Pruning: 以卷积核为单位的稀疏度

### 3. Channel稀疏

设卷积层输入通道为 $C_{in}$，输出通道为 $C_{out}$：

$$
y_{:, :, c_{out}} = \sum_{c_{in}=0}^{C_{in}-1} w_{c_{out}, c_{in}, :, :} \ast x_{:, :, c_{in}}
$$

当某通道 $c_{in}$ 被剪枝时，$w_{:, c_{in}, :, :} = 0$，对应输入特征图也可跳过

### 4. Head稀疏（Multi-Head Attention）

$$
\text{Attention}_{pruned} = \text{softmax}\left(\frac{(W_Q^H \cdot X) \odot M_Q \cdot ((W_K^H \cdot X) \odot M_K)^T}{\sqrt{d_k}}\right)
$$

- $M_Q, M_K$：Head维度的二元mask矩阵
- 通过移除不重要的Head实现稀疏

## 详细说明

### 1. N:M 稀疏

- **概念**：每M个元素中保留N个非零值，形成规则的稀疏模式
- **硬件支持**：NVIDIA A100/A30 GPU通过Tensor Core支持2:4稀疏
- **优势**：
  - 索引开销极低
  - 可实现2倍加速而无精度损失
  - 与量化结合效果更好（2:4稀疏 + INT8）
- **应用**：卷积层、全连接层、嵌入表

### 2. Channel Pruning（通道剪枝）

- **概念**：将整个输入或输出通道置零
- **实现方式**：
  - 依赖重要性分析（梯度、激活、泰勒展开）
  - 通道的L1范数、几何中位数等指标
- **优点**：
  - 直接减少计算量（MACs减少与剪枝比例成正比）
  - 减少特征图宽度，内存带宽同步降低
- **挑战**：需要联合剪枝多层以保持精度

### 3. Head Pruning（注意力头剪枝）

- **概念**：在Multi-Head Attention中移除不重要的Head
- **应用场景**：Transformer架构（大语言模型）
- **重要性指标**：
  - Head的注意力一致性（不同Head间的相关性）
  - 梯度贡献
  - 移除后的精度损失
- **实践**：
  - GPT系列模型中部分Head被证明冗余
  - LLaMA模型支持动态Head稀疏

### 4. MLP稀疏（FFN层稀疏）

- **概念**：Transformer中FFN（前馈网络）的中间层稀疏
- **结构**：FFN通常为 [Linear → activation → Linear]
- **稀疏类型**：
  - 神经元稀疏：第二个Linear层的行稀疏
  - 激活稀疏：ReLU/GELU后的零值比例高
- **特点**：
  - FFN通常占Transformer参数的2/3
  - MLP稀疏潜力巨大

### 5. 结构化稀疏 vs 非结构化稀疏

| 特性 | 结构化稀疏 | 非结构化稀疏 |
|------|-----------|-------------|
| 粒度 | 结构（channel/head/N:M） | 单个权重 |
| 加速效果 | 硬件直接加速 | 需要稀疏库支持 |
| 精度损失 | 相对较大 | 相对较小 |
| 实现难度 | 需要结构感知 | 简单幅度剪枝即可 |
| 适用硬件 | NVIDIA GPU、专用芯片 | 通用GPU |

### 6. 稀疏模式选择

| 模式 | 压缩比 | 加速潜力 | 精度影响 | 适用场景 |
|------|--------|----------|----------|----------|
| 2:4稀疏 | 50% | 2x | 极小 | NVIDIA Ampere+ |
| 4:8稀疏 | 50% | 取决于硬件 | 小 | 专用加速器 |
| Channel 50% | 50% | ~1.5-2x | 中等 | 边缘部署 |
| Head Pruning | 变量 | ~1.2-1.5x | 小-中 | LLM |

## 代码示例

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

def n_m_sparsify(weight, n, m):
    """
    将权重转换为N:M稀疏格式（每m个元素保留n个）

    Args:
        weight: 权重张量 [*, m] 最后维度必须是m的倍数
        n: 保留的非零元素个数
        m: 每组的元素个数
    """
    original_shape = weight.shape
    last_dim = original_shape[-1]

    # 确保最后一维是m的倍数
    assert last_dim % m == 0, f"Last dimension {last_dim} must be divisible by {m}"

    # reshape为 (..., num_groups, m)
    num_groups = last_dim // m
    weight_flat = weight.reshape(-1, m)

    # 对每组，保留绝对值最大的n个元素
    _, topk_indices = torch.topk(torch.abs(weight_flat), k=n, dim=1)

    # 创建mask
    mask = torch.zeros_like(weight_flat)
    mask.scatter_(1, topk_indices, 1.0)

    # 应用mask
    sparse_weight = weight_flat * mask

    return sparse_weight.reshape(original_shape)


class NM SparseLinear(nn.Module):
    """支持N:M稀疏的线性层"""
    def __init__(self, in_features, out_features, n=2, m=4):
        super().__init__()
        self.in_features = in_features
        self.out_features = out_features
        self.n = n
        self.m = m
        self.weight = nn.Parameter(torch.randn(out_features, in_features))

    def forward(self, x):
        # 训练时使用动态稀疏（保持梯度流动）
        if self.training:
            sparse_weight = n_m_sparsify(self.weight, self.n, self.m)
        else:
            # 推理时直接使用稀疏权重（已在外部预处理）
            sparse_weight = self.weight
        return F.linear(x, sparse_weight, self.bias)


def channel_prune_conv(conv_layer, pruning_ratio=0.3):
    """
    Channel Pruning：基于L1范数剪枝卷积层通道

    Args:
        conv_layer: 卷积层 (nn.Conv2d)
        pruning_ratio: 要剪枝的通道比例
    """
    # 计算每个输出通道的L1范数（衡量重要性）
    weight = conv_layer.weight.data  # [out_channels, in_channels, kH, kW]
    channel_importance = torch.norm(weight, dim=(1, 2, 3))  # [out_channels]

    # 确定阈值
    threshold = torch.quantile(channel_importance, pruning_ratio)

    # 保留重要通道
    keep_mask = channel_importance > threshold
    keep_indices = torch.where(keep_mask)[0]

    print(f"Pruning {conv_layer.in_channels - len(keep_indices)}/{conv_layer.in_channels} channels")

    # 创建新卷积层
    new_conv = nn.Conv2d(
        in_channels=len(keep_indices),
        out_channels=conv_layer.out_channels,
        kernel_size=conv_layer.kernel_size,
        stride=conv_layer.stride,
        padding=conv_layer.padding,
        dilation=conv_layer.dilation,
        groups=conv_layer.groups,
        bias=conv_layer.bias is not None
    )

    # 复制保留的权重
    new_conv.weight.data = conv_layer.weight.data[keep_indices]
    if conv_layer.bias is not None:
        new_conv.bias.data = conv_layer.bias.data.clone()

    return new_conv, keep_indices


def head_pruning_attention(attn_weights, head_importance, pruning_ratio=0.2):
    """
    注意力Head剪枝

    Args:
        attn_weights: 注意力权重 [batch, num_heads, seq_len, seq_len]
        head_importance: 各Head的重要性分数 [num_heads]
        pruning_ratio: 剪枝比例
    """
    num_heads = attn_weights.shape[1]
    threshold = torch.quantile(head_importance, pruning_ratio)

    # 保留重要Head
    keep_mask = head_importance > threshold
    keep_indices = torch.where(keep_mask)[0]

    print(f"Pruning {num_heads - len(keep_indices)}/{num_heads} attention heads")

    # 返回保留的Head
    return attn_weights[:, keep_indices, :, :], keep_indices
