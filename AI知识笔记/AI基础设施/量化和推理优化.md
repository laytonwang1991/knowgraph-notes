# 量化和推理优化

## 一句话定义

> 量化和推理优化通过将模型参数和计算从高精度浮点转换为低精度表示，结合算子融合、内存优化等技术，显著提升推理速度并降低资源消耗。

## 核心公式

### 对称量化

$$
X_{int8} = \text{round}\left(\frac{X_{fp32}}{scale}\right), \quad scale = \frac{\max(|X_{fp32}|)}{127}
$$

反量化：
$$
X_{fp32} = X_{int8} \times scale
$$

### 非对称量化

$$
X_{int8} = \text{round}\left(\frac{X_{fp32} - Z}{scale}\right), \quad Z = \text{round}\left(-\frac{\min(X_{fp32})}{scale}\right)
$$

### 量化误差分析

$$
\text{SNR}_{dB} = 10 \log_{10}\left(\frac{\text{Var}(X)}{\text{Var}(X - \hat{X})}\right)
$$

## 详细说明

### 1. 量化精度类型

| 数据类型 | 数值范围 | 精度损失 | 加速比 |
|---------|---------|---------|--------|
| FP32 | $3.4 \times 10^{38}$ | 无 | 1x |
| FP16 | 65504 | 极小 | ~1.5x |
| BF16 | $3.4 \times 10^{38}$ | 极小 | ~1.5x |
| INT8 | $[-128, 127]$ | 较小 | ~2-4x |
| INT4 | $[-8, 7]$ | 中等 | ~4-8x |
| INT2 | $[-2, 1]$ | 较大 | ~8-16x |

### 2. 量化方法

**训练后量化 (Post-Training Quantization)**
- 无需重新训练
- 使用校准数据集
- 简单但可能有精度损失

```python
from torch.quantization import quantize_dynamic

model = quantize_dynamic(
    model, {nn.Linear}, dtype=torch.qint8
)
```

**量化感知训练 (Quantization-Aware Training)**
- 在训练中模拟量化
- 精度更好
- 需要修改训练流程

```python
model.qconfig = torch.quantization.get_default_qconfig('fbgemm')
torch.quantization.prepare(model, inplace=True)
# 训练
torch.quantization.convert(model, inplace=True)
```

### 3. TensorRT优化流程

```
ONNX Model → Parser → Network Definition
    ↓
Calibration (INT8) → Builder → Engine
    ↓
Serialize/Deserialize → Inference
```

**INT8 Calibration**
```python
# 使用校准数据集
trt_builder = builder.create_network(1 << int(trt.NetworkDefinitionCreationFlag.EXPLICIT_BATCH))
# 收集激活范围
trt_int8_calibrator = MyCalibrator(train_loader)
```

### 4. 推理优化技术

**算子融合**
- 减少kernel launch开销
- 减少中间结果HBM访问
- 典型融合：Conv+BN+ReLU

**动态shape优化**
- 支持可变输入维度
- 减少padding开销

**内存复用 (Memory Pool)**
- 算子间共享显存
- 减少分配释放开销

**Batching策略**
| 方式 | 延迟 | 吞吐 | 适用场景 |
|------|------|------|---------|
| Static Batching | 高 | 高 | 离线批处理 |
| Dynamic Batching | 中 | 中 | 在线服务 |
| Continuous Batching | 低 | 最高 | LLM推理 |

### 5. KV Cache量化

LLM推理中的Key/Value缓存量化：
$$
\text{KV}_{quant} = \text{round}\left(\frac{\text{KV}_{fp16}}{\text{scale}}\right)
$$
- 显存占用大幅降低
- 滑动窗口attention配合

### 6. 量化精度验证

```python
# 对比FP32和INT8输出
with torch.no_grad():
    fp32_out = model_fp32(input)
    int8_out = model_int8(input)
    diff = (fp32_out - int8_out).abs().max()
    print(f"Max quantization error: {diff}")
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 推理速度显著提升 | 精度可能下降 |
| 显存占用减少 | 需要校准数据 |
| 部署成本降低 | 某些算子不支持INT8 |
