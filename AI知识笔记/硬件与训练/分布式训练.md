# 分布式训练

## 一句话定义

分布式训练是将深度学习模型训练任务分散到多个计算设备（GPU/节点）上执行，通过并行化计算和梯度同步来实现大规模模型的高效训练。

## 核心公式

### 数据并行通信量

$$Comm_{DDP} = 2 \times \frac{Parameter\_Size \times 4 \times (N-1)}{N} \quad (bytes/batch)$$

其中：
- $N$：GPU数量
- 参数以FP32存储（4字节）
- 每 batch 发送和接收各一次

### ZeRO通信量（ZeRO-3）

$$Comm_{ZeRO-3} = Comm_{DDP} \times \frac{1}{N}$$

### Pipeline并行气泡率

$$Bubble\_Ratio = \frac{(P-1) \times (M-1)}{M}$$

其中：
- $P$：Pipeline阶段数
- $M$：Micro-batch数量

### 有效吞吐量

$$Speedup_{effective} = \frac{N}{1 + (N-1) \times \frac{Comm}{Comp}}$$

### 2. 数据并行（Data Parallelism）

#### 2.1 经典数据并行（DDP）

每个GPU持有完整的模型副本，处理不同的数据分片：

```
DDP架构（4 GPU）：
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ GPU 0   │  │ GPU 1   │  │ GPU 2   │  │ GPU 3   │
│ Model   │  │ Model   │  │ Model   │  │ Model   │
│ Data 1/4│  │ Data 2/4│  │ Data 3/4│  │ Data 4/4│
└────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘
     │            │            │            │
     ▼            ▼            ▼            ▼
  Forward     Forward      Forward      Forward
     │            │            │            │
     ▼            ▼            ▼            ▼
   Loss        Loss         Loss         Loss
     │            │            │            │
     ▼            ▼            ▼            ▼
  Backward    Backward     Backward     Backward
     │            │            │            │
     └────────────┴─────┬──────┴────────────┘
                        ▼
              AllReduce (梯度平均)
                        │
     ┌──────────────────┴──────────────────┐
     ▼            ▼            ▼            ▼
   Optimizer   Optimizer   Optimizer   Optimizer
   Update      Update      Update      Update
```

#### 2.2 通信优化：AllReduce算法

```python
# Ring-AllReduce 示意
def ring_allreduce(send_buf, recv_buf, rank, world_size):
    """环形AllReduce实现"""
    n = world_size
    send_recv_offset = 0

    # Reduce-Scatter阶段
    for k in range(1, n):
        src = (rank - k + n) % n
        dst = (rank - k + 1 + n) % n
        # 从src接收并累加
        recv_chunk(send_buf, src, chunk_id=k)
        reduce(send_buf, recv_buf)

    # All-Gather阶段
    for k in range(1, n):
        src = (rank + k) % n
        dst = (rank + k + 1) % n
        # 发送完整数据
        send_chunk(send_buf, dst)
```

### 3. ZeRO（Zero Redundancy Optimizer）

ZeRO通过分片优化器状态、梯度和参数来消除数据并行中的内存冗余。

#### ZeRO三个阶段

```
ZeRO内存优化对比：

DDP (每个GPU完整复制)：
┌────────────────────────────────────┐
│  Optimizer States    │  12GB       │
│  Gradients          │  12GB       │
│  Model Parameters   │  12GB       │
│  Activations        │   8GB       │
│  Total per GPU     │  44GB       │
└────────────────────────────────────┘

ZeRO-1 (优化器状态分片)：
┌────────────────────────────────────┐
│  Optimizer States    │  3GB (1/4)  │
│  Gradients          │  12GB       │
│  Model Parameters   │  12GB       │
│  Total per GPU     │  27GB       │
└────────────────────────────────────┘

ZeRO-2 (优化器状态+梯度分片)：
┌────────────────────────────────────┐
│  Optimizer States    │  3GB (1/4)  │
│  Gradients          │  3GB (1/4)  │
│  Model Parameters   │  12GB       │
│  Total per GPU     │  18GB       │
└────────────────────────────────────┘

ZeRO-3 (全量分片)：
┌────────────────────────────────────┐
│  Optimizer States    │  3GB (1/4)  │
│  Gradients          │  3GB (1/4)  │
│  Model Parameters   │  3GB (1/4)  │
│  Total per GPU     │   9GB       │
└────────────────────────────────────┘
```

#### ZeRO通信量分析

$$Comm_{ZeRO-1} = Comm_{DDP}$$

$$Comm_{ZeRO-2} = Comm_{DDP} = \frac{2 \times P \times 4}{N}$$

$$Comm_{ZeRO-3} = \frac{Comm_{DDP}}{N}$$

其中 $P$ 为参数量，$N$ 为GPU数量。

### 4. 模型并行（Model Parallelism）

#### 4.1 张量并行（Tensor Parallelism, TP）

将单个层的参数矩阵切分到多个GPU：

```
Column Parallel (Linear层):
┌─────────┐     ┌─────────┐
│   A     │     │   B     │
│(Column) │     │(完整)   │
└────┬────┘     └────┬────┘
     │              │
     ▼              ▼
  Y₁ = AX₁      Y₂ = AX₂
     │              │
     └──────┬───────┘
            ▼
      All-Gather

Row Parallel (Linear层):
┌─────────┐     ┌─────────┐
│   A     │     │   B     │
│(完整)   │     │(Row)    │
└────┬────┘     └────┬────┘
     │              │
     ▼              ▼
  Y₁ = A₁X      Y₂ = A₂X
     │              │
     └──────┬───────┘
            ▼
       Reduce-Scatter
```

#### 4.2 流水线并行（Pipeline Parallelism）

将模型按层切分到不同GPU，形成生产-消费流水线：

```
Standard Pipeline (4 stage, 8 micro-batch):

Timeline:
GPU0: [F0][F0][F0][F0][F0][F0][F0][F0][B0][B0][B0][B0]
GPU1:     [ ][F1][F1][F1][F1][F1][F1][F1][F1][B1][B1][B1][B1]
GPU2:         [ ][ ][F2][F2][F2][F2][F2][F2][F2][F2][B2][B2][B2]
GPU3:             [ ][ ][ ][F3][F3][F3][F3][F3][F3][F3][F3][B3][B3]

Bubble:  ████

优化后 Interleaved Pipeline:
GPU0: [F0][  ][F0][  ][F0][  ][F0][  ][B0][  ][B0][  ][B0][  ][B0]
GPU1:   [F1][F0][  ][F1][F1][  ][F1][F1][  ][B1][B1][  ][B1][B1]
...
```

### 5. FSDP（Fully Sharded Data Parallel）

FSDP是PyTorch原生的大模型训练方案，结合了ZeRO和DDP的优点：

```
FSDP工作流程：

1. Forward阶段：
   - 每个rank拥有完整参数 →scatter → 计算 → gather

2. Backward阶段：
   - 梯度计算 → scatter reduce → 本地优化器更新

内存公式：
$$Mem_{FSDP} = \frac{M_{model} + M_{optimizer} + M_{gradients} + M_{activation}}{N}$$

其中：
- $M_{model}$: 模型参数（分片后 = 原始/N）
- $M_{optimizer}$: 优化器状态（分片后 = 原始/N）
- $M_{gradients}$: 梯度（分片后 = 原始/N）
- $M_{activation}$: 激活值（取决于batch size）
```

## 代码示例

### PyTorch DDP基础实现

```python
import os
import torch
import torch.nn as nn
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

def setup(rank, world_size):
    """初始化分布式环境"""
    os.environ['MASTER_ADDR'] = 'localhost'
    os.environ['MASTER_PORT'] = '12355'
    dist.init_process_group("nccl", rank=rank, world_size=world_size)
    torch.cuda.set_device(rank)

def cleanup():
    """清理分布式环境"""
    dist.destroy_process_group()

class SimpleModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(512, 2048),
            nn.ReLU(),
            nn.Linear(2048, 1024),
            nn.ReLU(),
            nn.Linear(1024, 10)
        )

    def forward(self, x):
        return self.net(x)

def train(rank, world_size):
    setup(rank, world_size)

    # 创建模型并移到GPU
    model = SimpleModel().cuda(rank)
    model = DDP(model, device_ids=[rank])

    optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
    criterion = nn.CrossEntropyLoss()

    # 模拟训练
    for epoch in range(10):
        for batch_idx in range(100):
            inputs = torch.randn(32, 512).cuda(rank)
            targets = torch.randint(0, 10, (32,)).cuda(rank)

            optimizer.zero_grad()
            outputs = model(inputs)
            loss = criterion(outputs, targets)
            loss.backward()
            optimizer.step()

        if rank == 0:
            print(f"Epoch {epoch}, Loss: {loss.item()}")

    cleanup()

# 启动脚本
# torchrun --nproc_per_node=4 train_ddp.py
```

# 定义混合精度策略
mixed_precision_policy = MixedPrecision(
    param_dtype=torch.float16,
    reduce_dtype=torch.float16,
    buffer_dtype=torch.float16,
)

def create_model():
    from transformers import BertModel
    return BertModel.from_pretrained('bert-large-uncased')

# FSDP配置
model = create_model()

# 自动包装策略（按transformer层包装）
auto_wrap_policy = functools.partial(

    max_tokens=1e8
)

### 流水线并行（DeepSpeed Pipeline）

```python
import deepspeed

# DeepSpeed配置
ds_config = {
    "train_batch_size": 64,
    "train_micro_batch_size_per_gpu": 4,
    "pipeline": {
        "parallel_size": 8,        # 8个流水线阶段
        "partitions": [1, 2, 3, 2] # 每层分配到哪个GPU
    },
    "zero_optimization": {
        "stage": 3,
        "contiguous_gradients": True
    },
    "fp16": {
        "enabled": True,
        "loss_scale": 0
    }
}

# 模型分层定义
class PipelineModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.embedding = nn.Embedding(50000, 1024)
        self.layers = nn.ModuleList([
            TransformerLayer() for _ in range(24)
        ])
        self.head = nn.Linear(1024, 50000)

    def forward(self, input_ids):
        hidden = self.embedding(input_ids)
        for layer in self.layers:
            hidden = layer(hidden)
        return self.head(hidden)

# DeepSpeed流水线训练
model, optimizer, _, _ = deepspeed.initialize(
    model=PipelineModel(),
    config=ds_config
)

# 流水线执行
for batch in dataloader:
    loss = model(batch['input_ids'])
    model.backward(loss)
    model.step()
```

### 梯度累积（用于大Batch训练）

```python
import torch
import torch.distributed as dist

class GradientAccumulator:
    """梯度累积器，支持分布式训练"""

    def __init__(self, model, accum_steps=4):
        self.model = model
        self.accum_steps = accum_steps
        self.step_count = 0

    def backward_and_step(self, loss, optimizer):
        # 缩放损失以实现梯度累积
        scaled_loss = loss / self.accum_steps
        scaled_loss.backward()

        self.step_count += 1

        if self.step_count % self.accum_steps == 0:
            # 同步梯度（在DDP中自动处理）
            optimizer.step()
            optimizer.zero_grad()

# 使用示例
accumulator = GradientAccumulator(model, accum_steps=8)

for batch in dataloader:
    outputs = model(batch)
    loss = criterion(outputs, targets)

    # 无需手动同步，backward()会自动触发AllReduce
    accumulator.backward_and_step(loss, optimizer)
```
