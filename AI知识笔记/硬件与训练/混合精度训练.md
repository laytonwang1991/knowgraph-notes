# 混合精度训练

## 一句话定义

混合精度训练通过在神经网络的不同组件使用不同精度的浮点数（通常为FP16/BF16和FP32），在保持模型精度的同时显著提升训练速度并降低显存占用。

## 核心公式

### 浮点数表示范围

$$FP16: Range = \pm 65504 \quad (5.96 \times 10^4)$$

$$BF16: Range = \pm 3.39 \times 10^{38}$$

### 动态范围对比

$$Dynamic\_Range_{FP16} = \frac{2^{16}}{2^{24}} = 2^{-8} \quad vs \quad Dynamic\_Range_{BF16} = \frac{2^{128}}{2^{112}} = 2^{16}$$

### Loss Scaling公式

$$Grad_{scaled} = \frac{Grad_{actual}}{Scale}$$

$$Grad_{actual} = Grad_{scaled} \times Scale$$

### 混合精度加速比（理论）

$$Speedup \approx \frac{2 \times BW_{FP16}}{BW_{FP32}} \approx 2-8\times$$

## 详细说明

### 1. 浮点数格式对比

#### 1.1 FP16（半精度）

```
FP16结构：Sign(1bit) + Exponent(5bit) + Mantissa(10bit)

表示范围：
- 规格化: ±6.55e4
- 非规格化: ±6.10e-5
- 精度: ~3.3 十进制位

内存占用: 16 bits (2 bytes)
```

#### 1.2 BF16（Brain Float）

```
BF16结构：Sign(1bit) + Exponent(8bit) + Mantissa(7bit)

与FP32相同的指数范围，牺牲尾数精度

优势：
- 指数位与FP32相同，动态范围一致
- 避免FP16的溢出问题
- 适合深度学习训练

内存占用: 16 bits (2 bytes)
```

#### 1.3 FP32（单精度）

```
FP32结构：Sign(1bit) + Exponent(8bit) + Mantissa(23bit)

表示范围：
- 规格化: ±3.4e38
- 精度: ~7.2 十进制位

内存占用: 32 bits (4 bytes)
```

#### 1.4 精度格式对比表

| 格式 | 指数位 | 尾数位 | 精度 | 动态范围 | 深度学习用途 |
|------|--------|--------|------|----------|-------------|
| FP32 | 8 | 23 | 高 | 大 | Master Weight |
| FP16 | 5 | 10 | 中 | 小 | Forward/Backward |
| BF16 | 8 | 7 | 中 | 大 | Forward/Backward |
| FP8 (E4M3) | 4 | 3 | 低 | 中 | 推理加速 |
| FP8 (E5M2) | 5 | 2 | 低 | 大 | 梯度存储 |

### 2. 混合精度训练原理

#### 2.1 为什么需要混合精度？

```
训练流程中的精度需求：

Forward Pass:
  Input(FP16) → Layer(FP16) → Output(FP16)
                ↓
          Loss计算

Backward Pass:
  Loss(FP16) → Gradients(FP16) → 参数更新

Optimizer Update:
  Gradients(FP16) → FP32 Master Copy → FP16 Parameters
```

**问题**：直接使用FP16训练会导致：
1. 梯度下溢（underflow）：梯度值小于FP16最小可表示数
2. 精度不足：累加误差影响收敛

**解决方案**：维护FP32 Master Weight

#### 2.2 Master Weight机制

```
混合精度训练流程：

1. Forward:
   FP32 Master Weight → Cast to FP16 → FP16 Forward → FP16 Output

2. Loss Computation:
   FP16 Loss

3. Backward:
   FP16 Loss → FP16 Gradients (可能下溢!)
       ↓
   Loss Scaling放大梯度
       ↓
   FP16 Scaled Gradients

4. Update:
   Scaled Gradients → Update FP32 Master Weight
       ↓
   Convert to FP16 for next iteration
```

### 3. Loss Scaling详解

#### 3.1 为什么需要Loss Scaling？

FP16的梯度范围为 $2^{-24}$ 到 $2^{16}$，即 $6 \times 10^{-8}$ 到 $65504$。

大部分神经网络的梯度值远小于此范围，会下溢为0。

#### 3.2 动态Loss Scaling算法

```python
# 动态Loss Scaling策略
class DynamicLossScaler:
    def __init__(self, scale_factor=2.0, init_scale=2**16):
        self.scale = init_scale
        self.scale_factor = scale_factor
        self.found_inf = False
        self.max_scale = 2**24

    def has_inf_or_nan(self, grad):
        return torch.isinf(grad).any() or torch.isnan(grad).any()

    def update_scale(self, found_inf):
        if found_inf:
            # 遇到inf/nan，减少scale
            self.scale /= self.scale_factor
        else:
            # 连续正常，增加scale
            self.scale *= self.scale_factor
            self.scale = min(self.scale, self.max_scale)
```

#### 3.3 Loss Scaling效果示意

```
梯度分布示意（未缩放 vs 缩放）:

未缩放:
        │
        │    ████
        │   ██████
        │  ████████
        │ ██████████
        └─────────────────
        0        梯度值

FP16最小值 → 梯度全部下溢!

缩放后（Scale=1024）:
        │
        │         ████
        │        ██████
        │       ████████
        │      ██████████
        └─────────────────
        0           梯度值×1024

→ 梯度落入FP16表示范围
```

### 4. AMP（Automatic Mixed Precision）

PyTorch的AMP库自动处理精度转换和Loss Scaling。

#### 4.1 GradScaler工作流程

```
┌─────────────────────────────────────────────────┐
│                 训练循环                         │
├─────────────────────────────────────────────────┤
│ 1. Forward (FP16自动转换)                        │
│    - 检查每个操作的输入类型                       │
│    - 适当操作保持FP32                           │
│    - Matmul, Conv → FP16                       │
│    - Softmax, LayerNorm → FP32                 │
├─────────────────────────────────────────────────┤
│ 2. Loss Scaling                                 │
│    - loss_scaler.scale(loss)                   │
├─────────────────────────────────────────────────┤
│ 3. Backward (FP16计算)                          │
│    - scaled_loss.backward()                    │
├─────────────────────────────────────────────────┤
│ 4. Gradient检查                                 │
│    - 检查inf/nan                               │
│    - unscale gradients                         │
├─────────────────────────────────────────────────┤
│ 5. Optimizer Step                               │
│    - update FP32 master weights                │
│    - scaler.update()                          │
└─────────────────────────────────────────────────┘
```

#### 4.2 支持FP16的操作

```
自动转换为FP16的操作：
✓ 矩阵乘法 (matmul, linear, bmm)
✓ 卷积 (conv1d, conv2d, conv3d)
✓ 循环神经网络 (lstm, gru)
✓ 注意力机制 (multi_head_attention)

保持FP32的操作：
✗ 损失函数中的logits累计
✗ BatchNorm统计（mean, var）
✗ Softmax操作
✗ LayerNorm中的统计量
✗ 优化器状态 (momentum, variance)
```

## 代码示例

### PyTorch AMP基础用法

```python
import torch
import torch.nn as nn
from torch.cuda.amp import autocast, GradScaler

# 定义模型
model = MyModel().cuda()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.CrossEntropyLoss()

# 创建GradScaler（自动处理loss scaling）
scaler = GradScaler()

for epoch in range(10):
    for batch_idx, (inputs, targets) in enumerate(dataloader):
        inputs = inputs.cuda()
        targets = targets.cuda()

        optimizer.zero_grad()

        # 自动混合精度训练
        with autocast(dtype=torch.float16):
            outputs = model(inputs)
            loss = criterion(outputs, targets)

        # Scales loss, calls backward(), unscales gradients
        scaler.scale(loss).backward()

        # 检查梯度并更新
        scaler.step(optimizer)
        scaler.update()

        if batch_idx % 100 == 0:
            print(f"Epoch {epoch}, Batch {batch_idx}, Loss: {loss.item()}")
```

### 自定义训练循环（细粒度控制）

```python
from torch.cuda.amp import autocast, GradScaler

class AMPTrainer:
    def __init__(self, model, optimizer, config):
        self.model = model
        self.optimizer = optimizer
        self.scaler = GradScaler(
            init_scale=2**16,  # 初始缩放因子
            growth_factor=2.0,  # 增长因子
            backoff_factor=0.5,  # 缩减因子
            growth_interval=2000  # 增长间隔
        )
        self.config = config

    def training_step(self, batch):
        inputs, targets = batch

        # 重置梯度
        self.optimizer.zero_grad()

        # Forward with AMP
        with autocast(
            enabled=True,
            dtype=torch.float16,
            grad_scaler=self.scaler
        ):
            outputs = self.model(inputs)
            loss = self.criterion(outputs, targets)

        # Backward with scaling
        self.scaler.scale(loss).backward()

        # Gradient clipping
        self.scaler.unscale_(self.optimizer)
        torch.nn.utils.clip_grad_norm_(
            self.model.parameters(),
            self.config.max_grad_norm
        )

        # Optimizer step
        self.scaler.step(self.optimizer)
        self.scaler.update()

        return loss

    def criterion(self, outputs, targets):
        return nn.functional.cross_entropy(outputs, targets)
```

### BF16混合精度训练

```python
import torch
from torch.cuda.amp import autocast, GradScaler

# PyTorch 1.10+ 支持BF16
model = MyModel().cuda()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3)
scaler = GradScaler()

for batch in dataloader:
    inputs, targets = batch
    inputs = inputs.cuda()
    targets = targets.cuda()

    optimizer.zero_grad()

    # 使用BF16进行训练
    with autocast(dtype=torch.bfloat16):
        outputs = model(inputs)
        loss = nn.functional.cross_entropy(outputs, targets)

    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

### TensorFlow混合精度

```python
import tensorflow as tf
from tensorflow.keras import mixed_precision

# 1. 设置混合精度策略
mixed_precision.set_global_policy('mixed_float16')

# 2. 配置优化器（使用FP32优化器状态）
optimizer = tf.keras.optimizers.Adam(learning_rate=1e-3)

# 3. 损失函数（输出层使用FP32）
class MyModel(tf.keras.Model):
    def train_step(self, data):
        x, y = data
        with tf.GradientTape() as tape:
            y_pred = self(x, training=True)
            # 损失计算使用FP32
            loss = self.compiled_loss(y, y_pred)

        # 计算梯度
        gradients = tape.gradient(loss, self.trainable_variables)

        # 梯度裁剪
        gradients, _ = tf.clip_by_global_norm(gradients, 5.0)

        # 更新权重
        optimizer.apply_gradients(zip(gradients, self.trainable_variables))

        # 更新指标
        self.compiled_metrics.update_state(y, y_pred)

        return {m.name: m.result() for m in self.metrics}

    # 输出层保持FP32
    def call(self, inputs):
        x = self.encoder(inputs)
        return tf.keras.layers.Dense(
            self.num_classes,
            activation='softmax',
            dtype='float32'  # 输出层使用FP32
        )(x)

# 训练
model = MyModel()
model.compile(optimizer=optimizer, loss='sparse_categorical_crossentropy')
model.fit(train_dataset, epochs=10)
```

## 应用场景

### 1. 大模型训练

| 模型 | 参数量 | 使用精度 | 训练硬件 |
|------|--------|---------|---------|
| GPT-3 | 175B | FP16/BF16 | 1000+ V100 |
| PaLM | 540B | BF16 | 6144 TPU |
| LLaMA 2 | 70B | FP16 | 2048 A100 |
| Stable Diffusion | 1B | FP16 | 32-256 A100 |

### 2. 混合精度训练注意事项

**适用场景：**
- 大规模模型训练（参数>1B）
- 显存受限时
- 追求训练速度时

**不适用场景：**
- 小模型训练（可能无显著加速）
- 回归任务（需要高精度）
- 数值敏感的科学计算

### 3. 精度选择指南

```
精度选择决策树：

是否需要训练？
├── 是 → 模型规模 > 1B?
│       ├── 是 → BF16 (推荐) / FP16
│       └── 否 → 可用FP16
└── 否 → 推理部署
        ├── 延迟敏感? → INT8/FP8
        └── 精度优先 → FP16/BF16
```
