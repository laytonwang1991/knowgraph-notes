# 实体链接

## 一句话定义

实体链接（Entity Linking）是将文本中出现的实体提及（mention）准确识别并链接到知识图谱中对应实体的技术，是连接非结构化文本与结构化知识图谱的桥梁。

## 核心公式

### 候选生成（Candidate Generation）
$$C(m) = \{e \in KG \mid e \text{ 可以对应文本片段 } m\}$$

**核心思想**：对于给定 mention $m$，从知识图谱中检索所有可能对应的候选实体集合 $C(m)$。

### 全局消歧目标函数
$$e^* = \arg\max_{e \in C(m)} \sum_{e' \neq e} \text{co-occurrence}(e, e')$$

**核心思想**：利用文档中所有 mentions 的协同信息，选择使全局一致性最大的实体。

### BERT 实体链接模型评分
$$s(e \mid m, ctx) = \text{MLP}([BERT(m, ctx); \text{Type}(e); \text{Embedding}(e)])$$

**核心思想**：融合 mention 文本表示、实体类型信息和实体嵌入进行统一打分。

### Cross-Encoder 排序模型
$$P(e \mid m) = \sigma(W \cdot \text{BERT}([CLS]m[SEP]e[SEP]))$$

**核心思想**：将 mention 和候选实体描述拼接，通过 Cross-Encoder 联合编码后输出链接概率。

## 详细说明

### 1. 什么是实体链接？

实体链接是知识图谱应用中的核心技术之一。当用户在文本中提到"苹果"时，可能指的是水果苹果、公司苹果（Apple Inc.）或其他含义。实体链接的任务就是根据上下文语境，将文本中的 mention（如"苹果"）准确链接到知识图谱中唯一的实体节点（如 Apple Inc.）。

**实体链接的输入输出**：

- **输入**：文本语料 + 知识图谱
- **输出**：每个 mention 链接到的知识图谱实体（带置信度）

**典型任务pipeline**：

1. **命名实体识别（NER）**：从文本中识别出所有实体提及（mention）
2. **候选生成（Candidate Generation）**：为每个 mention 生成候选实体列表
3. **候选消歧（Disambiguation）**：从候选实体中选择最合适的实体
4. **无链接预测（ NIL Link Prediction）**：判断 mention 是否应该被链接

### 2. 候选生成方法

候选生成的目标是为每个 mention 高召回率地生成候选实体集合。

**方法一：字典匹配**

- 构建 mention 到实体的映射字典（如 "Apple" -> {Apple Inc., Apple fruit, Apple Records}）
- 基于字符串相似度（编辑距离、Jaccard）进行模糊匹配
- **优点**：速度快，准确率高
- **缺点**：覆盖率受字典规模限制，未登录 mention 无法处理

**方法二：搜索引擎检索**

- 将 mention 作为查询，在知识图谱或外部语料中检索相关实体
- 利用 TF-IDF、BM25 等文本检索算法排序
- **优点**：能处理未见过的 mention
- **缺点**：可能引入噪声，需要较大计算量

**方法三：嵌入检索**

- 将 mention 和实体编码为向量，利用向量相似度检索候选
- 使用预训练语言模型（BERT、Sentence-BERT）编码
- **优点**：能捕捉语义相似性
- **缺点**：依赖高质量嵌入模型

### 3. 候选消歧方法

候选消歧是实体链接的核心环节，目的是从多个候选实体中选择最合适的一个。

**方法一：局部上下文消歧**

仅利用 mention 所在句子/段落的上下文信息进行消歧。

**特征包括**：
- 上下文词与候选实体描述的语义相似度
- 候选实体的类型与上下文的兼容性
- mention 的表面形式与实体名称的匹配程度

**典型模型**：
- TF-IDF 加权词匹配
- BERT Cross-Encoder：将 mention 和候选实体描述拼接后联合编码

**方法二：全局文档级消歧**

利用文档中所有 mentions 之间的语义关联信息进行协同消歧。

**核心假设**：同一文档中的实体在语义上是相关的（主题一致性和实体一致性）。

**典型方法**：
- **LinkKGE**：将实体链接建模为知识图谱补全问题，利用链接预测进行消歧
- **多莉（DWIE）**：同时建模实体识别和实体链接
- **Mentions graph**：构建 mention-entity 二分图，通过图算法进行全局推理

**协同消歧损失**：
$$\mathcal{L}_{global} = -\sum_{d \in D} \log P(e_d \mid ctx_d)$$

其中 $e_d$ 为文档 $d$ 中所有 mentions 的链接结果。

**方法三：预训练语言模型方法**

利用大型预训练模型学习上下文感知的实体表示。

**典型模型**：
- **GENRE**：基于 BART 的自回归实体链接，生成式建模实体名称
- **ELQ**：联合进行命名实体识别和实体链接的 Query-focused 方法
- **BLINK**：使用双编码器架构，分别编码 mention 上下文和候选实体

### 4. 无链接处理（NIL Link Prediction）

并非所有 mention 都应该被链接到知识图谱中的实体。

**无链接的类型**：

- **NIL Unknown**：mention 指向的实体确实不在知识图谱中
- **NIL New**：mention 指向新实体，需要在知识图谱中新建节点
- **NIL Not-Entity**：mention 不是实体（如普通名词的提及）

**处理方法**：
- 设定置信度阈值，低于阈值的视为无链接
- 训练专门的 NIL 分类器
- 在知识图谱中创建对应的 NIL 实体节点

### 5. 端到端联合模型

传统 pipeline 方式存在错误累积问题，近年来端到端联合模型成为主流。

**联合建模优势**：

- 避免 NER 错误传播到链接阶段
- 共享编码层，提升计算效率
- 支持跨任务的信息交互

**典型架构**：

```
Input Text → BERT Encoder → NER Head → Linker Head
                          ↓
                   [Joint Training]
```

- **Input**：原始文本 token 序列
- **Encoder**：BERT 或类似预训练模型
- **NER Head**：识别 mention 的起止位置和实体类型
- **Linker Head**：为每个 mention 预测链接实体

## 代码示例

### 基于 BERT 的候选消歧模型

```python
import torch
import torch.nn as nn
from transformers import BertModel, BertTokenizer

class BERTEntityLinker(nn.Module):
    def __init__(self, bert_model_name="bert-base-uncased", num_entity_types=10):
        super(BERTEntityLinker, self).__init__()
        self.bert = BertModel.from_pretrained(bert_model_name)
        self.tokenizer = BertTokenizer.from_pretrained(bert_model_name)
        hidden_size = self.bert.config.hidden_size

        # 候选消歧分类器
        self.disambig_classifier = nn.Sequential(
            nn.Linear(hidden_size * 3, hidden_size),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(hidden_size, 1)
        )

        # 实体类型分类
        self.type_classifier = nn.Linear(hidden_size, num_entity_types)

    def encode_mention_context(self, text, mention_start, mention_end):
        """编码包含 mention 的上下文"""
        inputs = self.tokenizer(
            text,
            return_tensors="pt",
            truncation=True,
            max_length=512
        )

        # 获取 mention 位置的表示
        outputs = self.bert(**inputs)
        sequence_output = outputs.last_hidden_state

        # 取 mention 对应 token 的平均表示
        mention_repr = sequence_output[0, mention_start:mention_end].mean(dim=0)
        cls_repr = sequence_output[0, 0]  # [CLS] token

        return torch.cat([cls_repr, mention_repr, outputs.pooler_output[0]])

    def score_candidate(self, context_repr, candidate_repr):
        """计算 mention 上下文与候选实体的匹配分数"""
        combined = torch.cat([context_repr, candidate_repr,
                              context_repr * candidate_repr])
        return self.disambig_classifier(combined)

    def forward(self, texts, mention_starts, mention_ends,
                candidate_embeddings, candidate_mask):
        # 编码上下文
        batch_size = len(texts)
        context_reprs = []
        for i in range(batch_size):
            ctx_repr = self.encode_mention_context(
                texts[i], mention_starts[i], mention_ends[i]
            )
            context_reprs.append(ctx_repr)

        context_reprs = torch.stack(context_reprs)  # (batch, hidden)

        # 计算每个候选的分数
        # candidate_embeddings: (batch, num_candidates, hidden)
        scores = torch.einsum('bd,bnd->bn',
                              context_reprs, candidate_embeddings)

        # 应用 mask 并 softmax
        scores = scores.masked_fill(candidate_mask == 0, -1e9)
        probs = torch.softmax(scores, dim=-1)

        return probs
```

### 候选生成示例

```python
from collections import defaultdict

class CandidateGenerator:
    def __init__(self, knowledge_graph):
        self.kg = knowledge_graph
        self.mention_to_entities = defaultdict(set)
        self._build_mention_index()

    def _build_mention_index(self):
        """构建 mention 到实体的索引"""
        for entity_id, entity in self.kg.entities.items():
            # 索引实体名称的各种变体
            for name in [entity.name] + entity.aliases:
                self.mention_to_entities[name.lower()].add(entity_id)
                # 也索引 n-gram
                tokens = name.lower().split()
                for n in [2, 3]:
                    for ngram in self._get_ngrams(tokens, n):
                        self.mention_to_entities[ngram].add(entity_id)

    def _get_ngrams(self, tokens, n):
        return [' '.join(tokens[i:i+n]) for i in range(len(tokens)-n+1)]

    def generate(self, mention, top_k=10):
        """为 mention 生成候选实体"""
        mention_lower = mention.lower()

        # 精确匹配
        candidates = self.mention_to_entities.get(mention_lower, set())

        # 模糊匹配（如果精确匹配结果不足）
        if len(candidates) < top_k:
            for m, ents in self.mention_to_entities.items():
                if m in mention_lower or mention_lower in m:
                    candidates.update(ents)

        # 限制返回数量
        return list(candidates)[:top_k]

    def get_entity_description(self, entity_id):
        """获取实体描述用于排序"""
        entity = self.kg.entities[entity_id]
        return f"{entity.name}: {entity.description}"
```

## 应用场景

### 知识图谱构建

从非结构化文本（新闻、论文、网页）中抽取知识并融入知识图谱。

**Pipeline**：
1. NER 识别文本中的实体
2. 实体链接将识别结果对接到已有知识图谱
3. 新实体/关系补充到知识图谱中

**典型系统**：DBpedia 抽取系统、谷歌 Knowledge Graph 构建

### 智能问答系统

将用户问题中的实体链接到知识图谱，实现知识驱动的问答。

**问题理解**：识别用户问题中询问的实体和关系
**答案生成**：沿着知识图谱的路径推理得到答案

**示例**：
- 输入："苹果公司的创始人是谁？"
- 链接："苹果公司" -> Apple Inc.
- 推理：Apple Inc. -> hasFounder -> Steve Jobs

### 语义搜索

提升搜索引擎的语义理解能力，提供更精准的搜索结果。

**实体卡片**：搜索实体名称时展示实体摘要信息
**语义扩展**：根据链接的实体扩展查询意图
**实体关联**：展示与搜索实体相关的其他实体

### 信息抽取与文本挖掘

从大规模语料中抽取结构化知识。

**关系抽取**：配合实体链接提高关系抽取准确率
**事件抽取**：链接事件涉及的核心实体
**舆论监控**：追踪特定实体在媒体中的出现和情感倾向

### 生物医学领域

医学文献中的实体链接（如基因、疾病、药物）是医学知识图谱构建的核心。

**医学命名实体识别**：识别疾病名称、药物名称、基因名称
**UMLS 医学叙词表**：将文本链接到标准医学叙词表实体
