# 知识图谱构建

## 一句话定义

知识图谱构建是将非结构化或半结构化数据转化为结构化知识的过程，通过实体抽取、关系抽取、实体链接和知识融合等关键技术，形成可计算和推理的知识网络。

## 核心公式

### 1. 命名实体识别（NER）

$$
P(y_1, y_2, ..., y_n | x_1, x_2, ..., x_n) = \prod_{i=1}^{n} P(y_i | y_{i-1}, x_i)
$$

BiLSTM-CRF 模型的解码公式：

$$
s(\mathbf{x}, \mathbf{y}) = \sum_{i=1}^{n} \mathbf{W}_{y_i}^{EMIT} \cdot \mathbf{h}_i + \mathbf{T}_{y_{i-1}, y_i} + \mathbf{s}_{START, y_1} + \mathbf{s}_{y_n, END}
$$

### 2. 关系抽取

$$
\hat{r} = \arg\max_{r \in R} P(r | e_1, e_2, \mathbf{c}) = \arg\max_{r \in R} \sigma(\mathbf{w}_r \cdot [\mathbf{h}_{e_1}; \mathbf{h}_{e_2}; \mathbf{h}_\mathbf{c}])
$$

注意力机制增强的关系抽取：

$$
\alpha_{ij} = \frac{\exp(\mathbf{v}^T \tanh(\mathbf{W}_1 \mathbf{h}_i + \mathbf{W}_2 \mathbf{h}_j))}{\sum_k \exp(\mathbf{v}^T \tanh(\mathbf{W}_1 \mathbf{h}_i + \mathbf{W}_2 \mathbf{h}_k))}
$$

### 3. 实体链接

$$
\hat{e} = \arg\max_{e \in \mathcal{E}(m)} P(e | m, c) = \arg\max_{e \in \mathcal{E}(m)} P(e | m) \cdot P(c | e)
$$

其中 $m$ 是 mention，$c$ 是上下文，$\mathcal{E}(m)$ 是候选实体集。

### 4. 知识融合（实体对齐）

$$
\text{Sim}(e_1, e_2) = \alpha \cdot \text{AttrSim}(e_1, e_2) + (1-\alpha) \cdot \text{StructSim}(e_1, e_2)
$$

基于表示学习的对齐：

$$
\mathcal{L} = \sum_{(e, e^+, e^-)} \max(0, \gamma - \cos(f(e), f(e^+)) + \cos(f(e), f(e^-)))
$$

## 详细说明

### 1. 实体抽取（Named Entity Recognition）

#### 1.1 任务定义

从文本中识别出命名实体，并将其分类到预定义类别（人物PER、地点LOC、机构ORG等）。

#### 1.2 技术演进

| 阶段 | 方法 | 代表模型 | 优缺点 |
|------|------|----------|--------|
| 早期 | 规则+词典 | 正则表达式 | 准确率高，覆盖率低 |
| 中期 | 统计学习 | CRF, HMM | 泛化能力强，特征工程复杂 |
| 当前 | 深度学习 | BiLSTM-CRF, BERT | 端到端，效果好，依赖数据 |

#### 1.3 BERT + CRF 模型

```python
from transformers import BertModel
import torch.nn as nn

class BertCRF(nn.Module):
    def __init__(self, vocab_size, tag_size, hidden_size=768):
        super().__init__()
        self.bert = BertModel.from_pretrained('bert-base-chinese')
        self.classifier = nn.Linear(hidden_size, tag_size)
        self.crf = CRF(tag_size)

    def forward(self, x, tags=None, mask=None):
        outputs = self.bert(x)
        emissions = self.classifier(outputs.last_hidden_state)
        if tags is not None:
            loss = -self.crf(emissions, tags, mask=mask)
            return loss
        return self.crf.decode(emissions, mask=mask)
```

### 2. 关系抽取

#### 2.1 任务定义

从文本中识别两个实体之间的语义关系，例如"张三是北京大学教授"中的"任职"关系。

#### 2.2 方法分类

- **流水线方法**：先识别实体，再判断关系
- **联合抽取**：端到端同时抽取实体和关系
- **远程监督**：利用已有知识库自动标注数据

#### 2.3 PCNN 模型

Piecewise CNN 将句子按实体位置分成三段：

$$
\mathbf{c}_1 = \text{CNN}(\mathbf{x}_{1:p}), \quad \mathbf{c}_2 = \text{CNN}(\mathbf{x}_{p+1:q}), \quad \mathbf{c}_3 = \text{CNN}(\mathbf{x}_{q+1:n})
$$

$$
\mathbf{c} = [\mathbf{c}_1; \mathbf{c}_2; \mathbf{c}_3]
$$

### 3. 实体链接

#### 3.1 任务流程

```
文本输入 → 实体识别 → 候选生成 → 候选排序 → 实体消歧 → 链接输出
```

#### 3.2 候选生成

- 基于别名字典匹配
- 基于搜索引擎的模糊匹配
- 基于知识库索引（如 Elasticsearch）

#### 3.3 候选排序

$$
P(e | m, c) = \frac{\exp(\text{MLP}([\mathbf{e}; \mathbf{c}; |\mathbf{e} - \mathbf{c}|]))}{\sum_{e' \in \mathcal{E}(m)} \exp(\text{MLP}([\mathbf{e'}; \mathbf{c}; |\mathbf{e'} - \mathbf{c}|]))}
$$

### 4. 知识融合

#### 4.1 挑战

- **数据异构**：不同来源的数据模式不同
- **实体冗余**：同一实体有多种表示
- **冲突消解**：不同来源的数据可能冲突
- **规模庞大**：数十亿级别的实体

#### 4.2 技术框架

| 阶段 | 主要技术 |
|------|----------|
| 数据预处理 | 清洗、分词、标准化 |
| 实体对齐 | 表示学习、规则匹配 |
| 属性融合 | 置信度加权、冲突消解 |
| 质量评估 | 准确率、召回率、一致性 |

#### 4.3 知识融合算法

**基于规则的融合：**
- 属性优先级排序
- 来源置信度加权
- 投票机制

**基于机器学习的融合：**
- 特征工程 + 分类器
- 表示学习 + 相似度计算
- 图神经网络捕获结构信息

## 代码示例

### 基于 BERT 的关系抽取

```python
import torch
from transformers import BertModel, BertTokenizer
import torch.nn as nn

class BertRelationExtraction(nn.Module):
    def __init__(self, num_relations, hidden_size=768):
        super().__init__()
        self.bert = BertModel.from_pretrained('bert-base-chinese')
        self.dropout = nn.Dropout(0.1)
        self.classifier = nn.Linear(hidden_size, num_relations)

    def forward(self, input_ids, attention_mask, token_type_ids):
        outputs = self.bert(
            input_ids=input_ids,
            attention_mask=attention_mask,
            token_type_ids=token_type_ids
        )
        pooled_output = outputs.pooler_output
        pooled_output = self.dropout(pooled_output)
        logits = self.classifier(pooled_output)
        return logits

# 训练
model = BertRelationExtraction(num_relations=13)
optimizer = torch.optim.AdamW(model.parameters(), lr=2e-5)

for batch in dataloader:
    outputs = model(**batch)
    loss = nn.CrossEntropyLoss()(outputs, batch['labels'])
    loss.backward()
    optimizer.step()
```

### 实体对齐（表示学习）

```python
class EntityAlignment(nn.Module):
    def __init__(self, embedding_dim):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(embedding_dim * 2, embedding_dim),
            nn.ReLU(),
            nn.Linear(embedding_dim, 1)
        )

    def forward(self, entity1_emb, entity2_emb):
        combined = torch.cat([entity1_emb, entity2_emb], dim=-1)
        similarity = torch.sigmoid(self.encoder(combined))
        return similarity
```

### 端到端实体链接

```python
class EntityLinker:
    def __init__(self, kb, mention_detector, ranker):
        self.kb = kb
        self.mention_detector = mention_detector
        self.ranker = ranker

    def link(self, text):
        # 1. 实体识别
        mentions = self.mention_detector.detect(text)

        results = []
        for mention in mentions:
            # 2. 候选生成
            candidates = self.kb.get_candidates(mention)

            # 3. 候选排序
            best_entity = self.ranker.rank(mention, candidates, text)

            # 4. NIL 处理

                results.append((mention, None))
            else:
                results.append((mention, best_entity))

        return results
```

## 应用场景

### 1. 通用知识图谱

| 图谱 | 规模 | 主要来源 |

### 2. 垂直领域应用

- **医疗领域**：药品-疾病-症状知识图谱
- **金融领域**：公司-人物-投资关系图谱
- **法律领域**：法条-案例-罪名知识图谱
- **教育领域**：知识点-课程-教材关系图谱

### 3. 企业知识管理

- 内部文档知识抽取
- 项目经验知识沉淀
- 组织架构知识建模

## 相关概念

- **命名实体识别（NER）**：识别文本中的实体边界和类型
- **关系抽取**：识别实体间的语义关系
- **实体链接**：将文本实体消歧到知识库
- **知识融合**：合并多源知识，消除冗余
- **知识表示学习**：将知识映射到低维向量空间
- **远程监督**：利用已有知识库自动标注训练数据
