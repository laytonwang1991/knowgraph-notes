# 增量学习（Incremental Learning）

## 一句话定义

增量学习是指模型能够随着时间持续学习新数据或新任务，同时保持对旧知识的记忆，避免灾难性遗忘的机器学习范式。

## 核心公式

增量学习的优化目标可以形式化为：

$$
\min_{\theta} \sum_{t=1}^{T} \alpha_t \cdot L_{task_t}(\theta)
$$

其中 $T$ 是任务总数，$\alpha_t$ 是任务权重，约束条件为不能访问 $t < current$ 的旧数据。

### 任务间知识蒸馏损失

为保持旧模型的知识，常用知识蒸馏：

$$
L_{KD} = -\sum_i p_{old}(x_i) \log p_{new}(x_i)
$$

其中 $p_{old}$ 是旧模型的输出概率分布，$p_{new}$ 是当前模型的输出。

## 详细说明

### 1. 增量学习分类

#### 1.1 任务增量学习（Task-Incremental Learning）

模型在每个增量阶段学习一个独立任务，推理时需要知道当前任务ID。

**特点**：
- 任务边界清晰
- 可使用任务标识符选择对应输出头
- 相对容易实现

**代表方法**：PikeNet、DenseCollaborativeLearning

#### 1.2 领域增量学习（Domain-Incremental Learning）

学习相同任务的不同数据分布，不添加新任务标识。

**特点**：
- 任务不变，只变化数据分布
- 无需任务ID即可推理
- 需要处理分布偏移

**代表方法**：Domain-Adaptive方法

#### 1.3 类增量学习（Class-Incremental Learning）

每个阶段学习新的类别，旧类别数据不可用。

**特点**：
- 类别数逐渐增加
- 旧类别数据无法访问
- 最具挑战性的设置

**代表方法**：iCaRL、EEIL、LwF

### 2. 核心技术路线

#### 2.1 参数隔离方法

通过为不同任务分配独立参数来避免干扰：

```python
class PackNet(nn.Module):
    """PackNet：保留并冻结重要权重用于旧任务"""
    def __init__(self, input_dim, hidden_dim, num_tasks, prune_ratio=0.5):
        super().__init__()
        self.num_tasks = num_tasks
        self.prune_ratio = prune_ratio
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, hidden_dim)
        self.fc_out = nn.ModuleList([nn.Linear(hidden_dim, 10) for _ in range(num_tasks)])
        self.task_mask = [None] * num_tasks
        self.used_params = []  # 记录已分配给旧任务的参数

    def forward(self, x, task_id):
        x = torch.relu(self.fc1(x))
        x = torch.relu(self.fc2(x))
        return self.fc_out[task_id](x)

    def prune_and_freeze(self, task_id):
        """为新任务腾出参数空间"""
        if task_id == 0:
            return

        # 计算每个权重的重要性（梯度绝对值）
        importance = torch.abs(self.fc1.weight.grad) + torch.abs(self.fc2.weight.grad)

        # 创建掩码，标记已用参数
        mask = torch.ones_like(importance)
        for prev_task in range(task_id):
            mask[self.task_mask[prev_task]] = 0

        # 选择最低重要性的可复用参数
        importance[mask == 0] = float('inf')
        num_to_prune = int(importance.numel() * self.prune_ratio)
        prune_indices = torch.argsort(importance.flatten())[:num_to_prune]

        # 保存新任务的参数掩码
        self.task_mask[task_id] = prune_indices

        # 冻结旧任务参数
        for idx in self.used_params:
            for param in [self.fc1.weight, self.fc1.bias, self.fc2.weight, self.fc2.bias]:
                param.requires_grad = False

        self.used_params.extend(prune_indices.tolist())
```

#### 2.2 正则化方法

通过在损失函数中添加正则项来约束参数更新：

```python
class LwF_Loss(nn.Module):
    """Learning without Forgetting：知识蒸馏方法"""
    def __init__(self, temperature=2.0, alpha=0.5):
        super().__init__()
        self.temperature = temperature
        self.alpha = alpha  # 旧知识权重

    def forward(self, new_logits, old_logits, target):
        """
        new_logits: 当前模型对新样本的输出
        old_logits: 旧模型对新样本的输出（知识蒸馏目标）
        target: 真实标签
        """
        # 交叉熵损失（监督信号）
        ce_loss = nn.functional.cross_entropy(new_logits, target)

        # 知识蒸馏损失（保持旧知识）
        if old_logits is not None:
            soft_target = nn.functional.softmax(old_logits / self.temperature, dim=-1)
            log_probs = nn.functional.log_softmax(new_logits / self.temperature, dim=-1)
            kd_loss = -torch.sum(soft_target * log_probs) * (self.temperature ** 2)
            total_loss = self.alpha * kd_loss + (1 - self.alpha) * ce_loss
        else:
            total_loss = ce_loss

        return total_loss
```

#### 2.3 原型网络方法（用于分类）

```python
class PrototypeNetwork(nn.Module):
    """基于原型的类增量学习方法"""
    def __init__(self, input_dim, hidden_dim, num_classes):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )
        self.num_classes = num_classes
        self.class_prototypes = {}  # 存储每个类的原型

    def compute_prototypes(self, dataloader, device):
        """计算每个类的原型（类中心）"""
        self.eval()
        class_features = {c: [] for c in range(self.num_classes)}

        with torch.no_grad():
            for batch in dataloader:
                if len(batch) == 2:
                    x, y = batch
                else:
                    x, y, _ = batch
                x, y = x.to(device), y.to(device)
                features = self.encoder(x)

                for c, f in zip(y, features):
                    c = c.item()
                    if c not in class_features:
                        class_features[c] = []
                    class_features[c].append(f)

        # 计算原型
        for c in class_features:
            if class_features[c]:
                self.class_prototypes[c] = torch.stack(class_features[c]).mean(dim=0)

    def predict(self, x, device):
        """基于原型的最近邻分类"""
        self.eval()
        features = self.encoder(x)

        min_dist = float('inf')
        pred_class = 0

        for c, proto in self.class_prototypes.items():
            dist = torch.norm(features - proto.to(device), p=2)
            if dist < min_dist:
                min_dist = dist
                pred_class = c

        return pred_class
```

### 3. 评估指标

| 指标 | 说明 |
|------|------|
| **Average Accuracy** | 所有任务完成后的平均准确率 |
| **Forgetting Rate** | 每个任务性能的下降程度 |
| **Transfer** | 任务间的正/负迁移 |
| **Memory Bounded** | 固定内存约束下的性能 |

### 4. 挑战与局限

- **容量饱和**：网络容量有限，无法无限学习新任务
- **分离边界模糊**：任务间的知识难以完全解耦
- **评估标准不统一**：不同论文使用不同的评估协议
- **计算效率**：随任务增加，训练时间可能增长

## 应用场景

| 领域 | 应用 |
|------|------|
| 自动驾驶 | 持续学习新的驾驶场景和路况 |
| 智能客服 | 不断学习新的用户问题和回答 |
| 医疗诊断 | 持续更新疾病诊断模型 |
| 推荐系统 | 实时学习用户兴趣变化 |
| 工业检测 | 持续学习新的产品缺陷类型 |

## 相关概念

- **灾难性遗忘（Catastrophic Forgetting）**：增量学习的核心挑战
- **任务迁移（Task Transfer）**：知识在不同任务间的正向迁移
- **元学习（Meta Learning）**：学习如何快速适应新任务
- **多任务学习（Multi-Task Learning）**：同时学习多个任务
- **在线学习（Online Learning）**：数据流式到达时的持续学习
