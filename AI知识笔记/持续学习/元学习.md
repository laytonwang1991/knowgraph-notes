# 元学习（Meta Learning）

## 一句话定义

元学习是让模型学习如何学习的研究范式，通过从多个相关任务中提取元知识，使模型能够在少量样本下快速适应新任务。

## 核心公式

### 元学习的目标函数

$$
\min_{\theta} \sum_{\mathcal{T}_i \sim p(\mathcal{T})} \mathcal{L}_{\mathcal{T}_i}(f_{\theta})
$$

其中 $\mathcal{T}_i$ 是从任务分布 $p(\mathcal{T})$ 中采样的任务，$f_{\theta}$ 是参数化模型。

### MAML 算法

MAML 寻找一组能够快速适应新任务的初始化参数：

$$
\theta^* = \theta - \beta \nabla_\theta \sum_{\mathcal{T}_i} \mathcal{L}_{\mathcal{T}_i}(f_{\theta - \alpha \nabla_\theta \mathcal{L}_{\mathcal{T}_i}(f_{\theta})})
$$

其中 $\alpha$ 是内层循环学习率，$\beta$ 是外层循环学习率。

### Reptile 算法

Reptile 采用简化的逐步更新策略：

$$
\theta \leftarrow \theta + \beta \cdot (\tilde{\theta} - \theta)
$$

其中 $\tilde{\theta}$ 是在采样任务上多次更新的参数。

## 详细说明

### 1. 元学习三大范式

#### 1.1 基于优化的元学习

通过优化过程本身实现快速学习：

| 算法 | 核心思想 | 适用场景 |
|------|----------|----------|
| **MAML** | 寻找最优初始化点 | 少样本分类、快速强化学习 |
| **Reptile** | 简化的元更新 | 简单任务、易于实现 |
| **LSTM Meta-Learner** | 用LSTM学习优化器 | 复杂优化场景 |

#### 1.2 基于度量的元学习

学习样本间的相似度度量：

| 算法 | 度量方式 | 适用场景 |
|------|----------|----------|
| **Siamese Network** | L1/L2距离 | 验证任务 |
| **Matching Network** | 注意力加权 | 少样本分类 |
| **Prototypical Network** | 原型距离 | 少样本分类 |
| **Relation Network** | 可学习度量 | 少样本关系判断 |

#### 1.3 基于记忆的元学习

通过外部记忆模块实现知识存储：

| 算法 | 记忆机制 | 适用场景 |
|------|----------|----------|
| **MANN** | Neural Turing Machine | 序列任务 |
| **Memory-Augmented NN** | 内容寻址记忆 | 快速绑定 |
| **Meta Network** | 慢速/快速权重 | 多任务学习 |

### 2. MAML 算法详解

**Model-Agnostic Meta-Learning (MAML)** 是最经典的元学习方法：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class MAML(nn.Module):
    def __init__(self, model, inner_lr=0.01, outer_lr=0.001):
        super().__init__()
        self.model = model
        self.inner_lr = inner_lr  # 内层循环学习率（任务适应）
        self.outer_lr = outer_lr  # 外层循环学习率（元更新）

    def inner_update(self, loss, params=None):
        """内层循环：在单个任务上快速适应"""
        if params is None:
            params = dict(self.model.named_parameters())

        grads = torch.autograd.grad(loss, params.values(),
                                     create_graph=True,
                                     retain_graph=True)

        # 计算任务适应的参数
        adapted_params = {name: param - self.inner_lr * grad
                          for (name, param), grad in zip(params.items(), grads)}
        return adapted_params

    def forward(self, support_x, support_y, query_x, query_y, task_id):
        """
        MAML 单任务前向
        support_x/y: 支持集（用于计算适应方向）
        query_x/y: 查询集（用于元更新）
        """
        # 阶段1：在支持集上计算初始损失并更新
        logits = self.model(support_x)
        loss = F.cross_entropy(logits, support_y)
        adapted_params = self.inner_update(loss)

        # 阶段2：在查询集上评估适应后的模型
        # 使用 adapted_params 但不构建计算图（避免二阶优化）
        with torch.no_grad():
            logits = self.model(query_x)
            query_loss = F.cross_entropy(logits, query_y)

        return query_loss

    def meta_learn(self, task_batch, optimizer):
        """
        外层循环：跨任务元更新
        task_batch: 一个batch中的多个任务
        """
        optimizer.zero_grad()
        meta_loss = 0

        for task in task_batch:
            # 对每个任务执行内层循环
            task_loss = self.forward(**task)
            meta_loss += task_loss

        # 元更新：基于所有任务的累积损失
        meta_loss.backward()
        optimizer.step()

        return meta_loss.item() / len(task_batch)
```

### 3. Prototypical Networks

原型网络通过计算类原型实现少样本分类：

```python
class PrototypicalNetwork(nn.Module):
    def __init__(self, encoder, hidden_dim):
        super().__init__()
        self.encoder = encoder

    def compute_prototypes(self, support_x, support_y):
        """计算每个类的原型"""
        features = self.encoder(support_x)
        classes = torch.unique(support_y)

        prototypes = {}
        for c in classes:
            c = c.item()
            mask = (support_y == c)
            prototypes[c] = features[mask].mean(dim=0)

        return prototypes

    def forward(self, support_x, support_y, query_x, query_y, n_shot):
        """
        n_shot: 每个类的样本数
        """
        # 计算类原型
        prototypes = self.compute_prototypes(support_x, support_y)

        # 查询样本分类
        query_features = self.encoder(query_x)
        logits = []

        for q_feat in query_features:
            distances = []
            for c, proto in prototypes.items():
                d = torch.norm(q_feat - proto, p=2)
                distances.append(-d)  # 距离取负作为logit
            logits.append(torch.stack(distances))

        logits = torch.stack(logits)

        # 计算损失
        query_classes = torch.tensor(list(prototypes.keys())).unsqueeze(1).expand(-1, n_shot).flatten()
        loss = F.cross_entropy(logits, query_classes.to(logits.device))

        return loss, logits

    def predict(self, support_x, support_y, query_x, n_shot):
        """推理：给定支持集，对查询样本分类"""
        prototypes = self.compute_prototypes(support_x, support_y)
        query_features = self.encoder(query_x)

        predictions = []
        for q_feat in query_features:
            distances = {c: torch.norm(q_feat - proto, p=2).item()
                        for c, proto in prototypes.items()}
            pred = min(distances, key=distances.get)
            predictions.append(pred)

        return predictions
```

### 4. Reptile 算法

Reptile 是对 MAML 的简化版本：

```python
class Reptile(nn.Module):
    def __init__(self, model, inner_steps=5, inner_lr=0.01, outer_lr=0.001):
        super().__init__()
        self.model = model
        self.inner_steps = inner_steps  # 内层迭代次数
        self.inner_lr = inner_lr
        self.outer_lr = outer_lr

    def inner_loop(self, task_batch):
        """在任务上执行多次梯度更新"""
        # 保存原始参数
        original_params = {n: p.clone() for n, p in self.model.named_parameters()}

        for task in task_batch:
            # 复制当前模型参数
            current_params = {n: p.clone() for n, p in self.model.named_parameters()}

            # 多次内部更新
            for _ in range(self.inner_steps):
                loss = self.compute_task_loss(task, current_params)
                grads = torch.autograd.grad(loss, current_params.values())

                # 梯度更新
                current_params = {n: p - self.inner_lr * g
                                 for (n, p), g in zip(current_params.items(), grads)}

            # 更新模型参数到新位置
            for n, p in self.model.named_parameters():
                p.data = p.data + (current_params[n] - original_params[n]) / len(task_batch)

    def compute_task_loss(self, task, params):
        """使用给定参数计算任务损失"""
        raise NotImplementedError

    def meta_learn(self, task_batch, optimizer):
        """元学习主循环"""
        optimizer.zero_grad()
        self.inner_loop(task_batch)
        optimizer.step()
```

### 5. Few-Shot Learning 配置

| 配置 | 支持集 | 查询集 | 说明 |
|------|--------|--------|------|
| 1-shot | 1样本/类 | 若干/类 | 极度受限 |
| 5-shot | 5样本/类 | 若干/类 | 常用配置 |
| 10-shot | 10样本/类 | 若干/类 | 相对宽松 |
| N-way K-shot | N类 K样本/类 | 若干/类 | 标准表述 |

## 应用场景

| 领域 | 应用 | 说明 |
|------|------|------|
| 计算机视觉 | 细粒度图像分类 | 稀有物种、定制商品识别 |
| 自然语言处理 | 文本分类 | 少样本意图识别、情感分析 |
| 机器人 | 快速技能学习 | 少量演示即可学习新动作 |
| 推荐系统 | 冷启动 | 新用户/商品快速建模 |
| 药物发现 | 分子属性预测 | 罕见分子性质预测 |

## 相关概念

- **迁移学习（Transfer Learning）**：将知识从源任务迁移到目标任务
- **多任务学习（Multi-Task Learning）**：同时学习多个相关任务
- **持续学习（Continual Learning）**：随时间持续学习新任务
- **零样本学习（Zero-Shot Learning）**：无样本情况下的泛化
- **少样本学习（Few-Shot Learning）**：极少量样本的学习
