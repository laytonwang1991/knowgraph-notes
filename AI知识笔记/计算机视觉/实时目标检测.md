# 实时目标检测

## 一句话定义

实时目标检测是指在视频流或高帧率场景中，以不低于 25-30 FPS 的速度完成目标的定位（边界框）与分类的深度学习技术，其核心挑战在于速度与精度的权衡（Speed-Accuracy Trade-off）。

## 核心公式

### 1. 非极大值抑制（NMS）

$$iou = \frac{B_{pred} \cap B_{gt}}{B_{pred} \cup B_{gt}}$$

$$s_i = \begin{cases} s_i & \text{if } iou < threshold \\ 0 & \text{otherwise} \end{cases}$$

其中 $B_{pred}$ 为预测边界框，$B_{gt}$ 为真实边界框，$s_i$ 为第 $i$ 个边界框的置信度分数。

### 2. mAP（Mean Average Precision）

$$AP = \int_0^1 P(r) \, dr$$

$$mAP = \frac{1}{N} \sum_{i=1}^{N} AP_i$$

### 3. FPS 计算

$$FPS = \frac{1}{T_{inference} + T_{preprocessing} + T_{postprocessing}}$$

### 2. 实时检测器架构对比

#### 2.1 YOLO 系列（单阶段回归方法）

- **核心思想**：将检测任务转化为单次全局回归，直接预测边界框和类别概率
- **优势**：推理速度快，适合端侧部署
- **代表**：YOLOv5-v12, YOLOX, YOLOv7, YOLOv8, YOLOv10

#### 2.2 RT-DETR（Real-Time DETR）

- **核心思想**：将 Transformer 的全局注意力机制引入实时检测
- **骨干网络**：HGNet（华为诺亚方舟实验室）
- **优势**：在保证实时速度的同时享有 Transformer 的全局建模能力
- **性能**：FPS ~110, mAP ~54.8

#### 2.3 EfficientDet

- **核心思想**：采用复合缩放（Compound Scaling）+ BiFPN 特征金字塔
- **优势**：参数量少，效率高
- **代表**：EfficientDet-D0 ~ D7

### 3. 速度与精度的权衡策略

#### 3.1 模型层面的权衡

```
轻量化骨干网络：ResNet18 → MobileNetV3 → ShuffleNetV2
模型缩放策略：width scaling + depth scaling + resolution scaling
知识蒸馏：用大模型监督小模型
```

#### 3.2 输入分辨率的影响

| 输入尺寸 | 参数量变化 | FPS 变化 | mAP 变化 |
|----------|------------|----------|----------|
| 320x320 | -40% | +60% | -8% |
| 416x416 | baseline | baseline | baseline |
| 512x512 | +40% | -35% | +5% |
| 640x640 | +80% | -55% | +12% |

#### 3.3 推理优化技术

- **TensorRT 加速**：INT8/FP16 量化，层融合，内核自动调优
- **剪枝（Pruning）**：移除不重要的通道或层
- **知识蒸馏（Distillation）**：用大模型监督小模型
- **结构重参数化**：推理时多分支合并为单分支

### 4. 后处理优化：NMS 的替代方案

传统 NMS 计算开销大，以下方法可降低延迟：

- **Soft-NMS**：将重叠框的分数衰减而非直接删除
- **Center-NMS**：基于中心点距离抑制，减少比较次数
- **Matrix NMS**：矩阵运算并行化，加速 NMS

## 代码示例

### 使用 Ultralytics YOLOv8 进行实时推理

```python
from ultralytics import YOLO
import cv2

# 加载模型
model = YOLO("yolov8n.pt")  # n=nano, s=small, m=medium, l=large, x=x-large

# 实时摄像头推理
cap = cv2.VideoCapture(0)
while cap.isOpened():
    ret, frame = cap.read()
    if not ret:
        break

    # 推理
    results = model(frame, conf=0.5, iou=0.45, verbose=False)

    # 绘制结果
    annotated_frame = results[0].plot()
    cv2.imshow("YOLOv8 Real-Time Detection", annotated_frame)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

### 使用 RT-DETR 进行实时推理

```python
from transformers import RTDetrForObjectDetection, RTDetrImageProcessor
from PIL import Image

# 加载预训练模型
model = RTDetrForObjectDetection.from_pretrained("PekingU/rtdetr_resnet50")
processor = RTDetrImageProcessor.from_pretrained("PekingU/rtdetr_resnet50")

# 推理
image = Image.open("street.jpg")
inputs = processor(images=image, return_tensors="pt")
outputs = model(**inputs)

# 后处理
results = processor.post_process_object_detection(outputs, target_sizes=[(image.height, image.width)])
for result in results:
    for score, label_id, box in zip(result["scores"], result["labels"], result["boxes"]):
        box = [round(i, 2) for i in box.tolist()]
        print(f"Class: {model.config.id2label[label_id.item()]}, Score: {score.item():.3f}, Box: {box}")
```

### TensorRT INT8 量化加速

```python
import torch
from torch2trt import torch2trt

# 加载 PyTorch 模型
model = YOLO("yolov8m.pt")
model.model.eval()

# 转换为 TensorRT FP16
x = torch.randn(1, 3, 640, 640).cuda()
model_trt = torch2trt(model.model, [x], fp16=True)

# 推理速度对比
import time
with torch.no_grad():
    # PyTorch
    t1 = time.time()
    for _ in range(100):
        _ = model.model(x)
    print(f"PyTorch: {(time.time()-t1)/100*1000:.2f} ms/frame")

    # TensorRT
    x_trt = x.cuda()
    t2 = time.time()
    for _ in range(100):
        _ = model_trt(x_trt)
    print(f"TensorRT: {(time.time()-t2)/100*1000:.2f} ms/frame")
```

## 应用场景

| 场景 | 需求帧率 | 代表模型 | 特殊要求 |
|------|----------|----------|----------|
| 自动驾驶 | 30+ FPS | YOLOv8, RT-DETR | 低延迟，高召回率 |
| 视频监控 | 25+ FPS | YOLOv5, EfficientDet | 多目标，长时推理 |
| 无人机避障 | 50+ FPS | YOLOv8n, NanoTrack | 极低延迟，轻量化 |
| 工业质检 | 60+ FPS | YOLOv7, 自定义 | 高精度，定制化 |
| 直播特效 | 30+ FPS | YOLOX, SAM+YOLO | 类别定制，人像分割 |
| AR/VR | 90+ FPS | 轻量级YOLO | 极致低延迟，边缘部署 |

## 相关概念

- **One-Stage vs Two-Stage**：实时检测通常采用 One-Stage 架构（无候选区域生成阶段）
- **Anchor-Based vs Anchor-Free**：YOLOv8 开始转向 Anchor-Free，简化后处理
- **NMS（Non-Maximum Suppression）**：去除重复检测框的后处理步骤
- **mAP（Mean Average Precision）**：评估检测精度的核心指标
- **FLOPs（浮点运算次数）**：衡量模型计算复杂度的指标
- **TensorRT**：NVIDIA 高性能深度学习推理引擎
