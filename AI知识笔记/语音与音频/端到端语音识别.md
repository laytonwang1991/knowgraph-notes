# 端到端语音识别 (End-to-End Speech Recognition)

## 一句话定义

端到端语音识别通过深度学习模型直接从原始音频映射到文本，消除了传统 ASR 系统中独立的声学模型、语言模型和词典组件。

## 核心公式

### CTC 损失函数
$$L_{CTC} = -\sum_{(X,Y)} \log P(Y|X) = -\sum_{(X,Y)} \log \sum_{A \in \mathcal{B}(X,Y)} \prod_{t=1}^{T} P(a_t|x_t)$$

其中 $\mathcal{B}(X,Y)$ 表示所有能映射到标签序列 $Y$ 的对齐路径，$a_t$ 是时刻 $t$ 的输出符号。

### Listen-Attend-Spell (LAS) 模型
$$h_{enc} = \text{Encoder}(X) \quad \text{(音频编码)}$$
$$c_i = \text{Attention}(h_{enc}, h_{dec,i-1}) \quad \text{(上下文向量)}$$
$$h_{dec,i} = \text{Decoder}(h_{dec,i-1}, y_{i-1}, c_i) \quad \text{(解码)}$$
$$P(y_i|Y_{<i}, X) = \text{softmax}(W h_{dec,i}) \quad \text{(输出分布)}$$

### Wav2Vec 2.0 对比损失
$$L = -\log \frac{\exp(\text{sim}(z, c)/\kappa)}{\sum_{\tilde{z}} \exp(\text{sim}(z, \tilde{z})/\kappa)}$$

其中 $z$ 是正确上下文 $c$ 对应的潜在表示，$\tilde{z}$ 是干扰项。

## 详细说明

### 1. 为什么需要端到端模型？

| 传统 ASR 组件 | 问题 | 端到端优势 |
|--------------|------|-----------|
| 声学模型 | 需人工设计特征，独立训练 | 自动学习联合表示 |
| 语言模型 | 与声学模型分离，资源密集 | 与声学联合优化 |
| 词典/发音字典 | 需大量人工维护 | 无需显示词典 |
| 强制对齐 | 训练复杂，错误累积 | 隐式对齐 |

### 2. CTC (Connectionist Temporal Classification)

**核心思想**：允许预测重复符号和空白符，无需帧级别对齐。

- **空白符 (blank)**：表示无输出
- **多对一映射**：将预测序列折叠为最终输出
- **前向-后向算法**：高效计算所有可能对齐的概率

**代表模型**：
- Deep Speech (Baidu, 2014)
- Jasper (NVIDIA, 2019)
- QuartzNet (NVIDIA, 2020)

### 3. Attention-based 模型 (LAS/RNN-T)

**Listen-Attend-Spell (LAS)**：
- Encoder-Decoder 架构
- 全局 Attention 机制
- 自回归解码

**RNN Transducer (RNN-T)**：
- 结合 CTC 和 attention
- 支持流式识别
- 输出与输入长度独立

### 4. Transformer ASR

**Conformer (Google, 2020)**：
$$H_{out} = \text{FFN} + \text{MHSA} + \text{ConvModule} + \text{FFN}$$

- 结合卷积的局部建模能力
- Transformer 的全局 attention
- 当前 SOTA 骨干网络

**Branching**：
- Switch Transformer
- Efficient Conformer

### 5. 自监督学习: Wav2Vec 系列

**Wav2Vec 2.0 (Facebook, 2020)**：
```
原始音频 → CNN Encoder → 潜在表示 z
                          ↓
              Transformer Encoder → 上下文表示 c
                          ↓
                   对比损失学习
```

**HuBERT (Facebook, 2021)**：
- 掩码预测任务
- 离线聚类提供目标

**WavLM (Microsoft, 2022)**：
- 结合语音增强预训练
- 说话人分离任务

```python
# 使用 Hugging Face Transformers 进行端到端 ASR
from transformers import pipeline

# 加载 Whisper 模型（OpenAI 的端到端模型）
asr_pipeline = pipeline(
    "automatic-speech-recognition",
    model="openai/whisper-medium"
)

# 音频文件识别
result = asr_pipeline("audio.wav")
print(result["text"])
```

```python
# 使用 Wav2Vec 2.0 进行语音识别
from transformers import Wav2Vec2ForCTC, Wav2Vec2Processor
import torch

processor = Wav2Vec2Processor.from_pretrained("facebook/wav2vec2-base-960h")
model = Wav2Vec2ForCTC.from_pretrained("facebook/wav2vec2-base-960h")

# 音频输入
input_values = processor(audio_array, sampling_rate=16000, return_tensors="pt").input_values

# 推理
logits = model(input_values).logits
predicted_ids = torch.argmax(logits, dim=-1)
transcription = processor.batch_decode(predicted_ids)[0]
```

```python
# 使用 CTC 实现简单端到端识别
import torch
import torch.nn as nn

class SimpleCTCModel(nn.Module):
    def __init__(self, input_dim, hidden_dim, num_classes):
        super().__init__()
        self.cnn = nn.Conv1d(input_dim, hidden_dim, kernel_size=5, stride=2)
        self.rnn = nn.GRU(hidden_dim, hidden_dim, num_layers=2, batch_first=True, bidirectional=True)
        self.fc = nn.Linear(hidden_dim * 2, num_classes)
        self.ctc_loss = nn.CTCLoss(blank=0, reduction='mean')

    def forward(self, x, targets, input_lengths, target_lengths):
        # x: (batch, time, features)
        x = self.cnn(x.transpose(1, 2)).transpose(1, 2)
        x, _ = self.rnn(x)
        x = self.fc(x)
        log_probs = torch.log_softmax(x, dim=-1)
        loss = self.ctc_loss(log_probs.transpose(0, 1), targets, input_lengths, target_lengths)
        return loss

    def decode(self, x):
        # Greedy decode
        log_probs = torch.log_softmax(self.fc(self.rnn(self.cnn(x.transpose(1, 2))[0])[0].transpose(0, 1), -1]), dim=-1)
        return torch.argmax(log_probs, dim=-1)
```

| 场景 | 应用 | 代表模型 |
|------|------|---------|
| 实时语音转文字 | 会议记录、字幕生成 | Whisper, Conformer |
