# 专家系统

## 一句话定义

> 专家系统是基于知识库和推理引擎的AI程序，通过将领域专家的知识编码为规则，模拟专家进行特定领域的复杂推理和决策。

## 核心架构

**专家系统的基本结构：**

```
+------------------+
|   用户界面       |
+--------+---------+
         |
+--------v---------+
|   推理引擎       | <-------> 知识库
+--------+---------+          (规则库 + 事实库)
         |
+--------v---------+
|   知识获取模块   |
+------------------+
```

**推理引擎的核心循环：**
```
WHILE NOT (goal reached OR no more rules) DO
    MATCH: Find applicable rules
    SELECT: Choose one rule (conflict resolution)
    EXECUTE: Fire the rule, update working memory
END WHILE
```

## 详细说明

### 1. 历史背景

**复兴契机：**
- 1970年代后期AI走出寒冬
- 专家系统被视为AI商业化的突破口
- 日本第五代计算机计划推波助澜（1982年）

**发展脉络：**
| 时间 | 里程碑 |
|------|--------|
| 1965年 | DENDRAL系统诞生（有机化学专家系统） |
| 1972年 | MYCIN系统（医学诊断专家系统） |
| 1979年 | XCON系统（DEC计算机配置专家系统） |
| 1980年代 | 专家系统商业化高潮 |

### 2. 代表性系统

**MYCIN（1972）：**
- 用途：细菌感染疾病诊断与治疗推荐
- 规则数：约450条产生式规则
- 特点：不确定性推理（置信度因子）
- 准确率：在测试中达到专家水平

**DENDRAL（1965）：**
- 用途：根据质谱数据推断分子结构
- 开创：首个成功的专家系统
- 方法：基于规则的推理 + 约束满足

**XCON（1979）：**
- 用途：为DEC VAX计算机系统自动配置订单
- 规则数：超过10,000条规则
- 价值：每年为DEC节省数百万美元

### 3. 技术组成

**知识表示方法：**
| 方法 | 说明 | 典型应用 |
|------|------|----------|
| 产生式规则 | IF-THEN形式，易于理解 | MYCIN |
| 框架表示 | 面向对象的知识结构 | 专家系统外壳 |
| 语义网络 | 图结构表示概念关系 | 自然语言理解 |
| 逻辑编程 | 基于一阶谓词逻辑 | Prolog系统 |

**推理策略：**
1. **正向链（Forward Chaining）**：从已知事实出发，触发适用规则
2. **反向链（Backward Chaining）**：从目标出发，反向寻找支持证据
3. **混合推理**：结合正向和反向推理

**不确定性处理：**
- MYCIN的置信度因子（Certainty Factors）
- 概率推理（Bayesian Networks的雏形）
- 模糊逻辑处理模糊性

### 4. 局限性

**知识获取瓶颈：**
| 问题 | 影响 |
|------|------|
| 专家时间稀缺 | 知识工程师与专家的沟通成本极高 |
| 隐性知识难表达 | 专家的经验直觉难以形式化 |
| 规则冲突 | 专家之间意见不一致 |

**技术局限：**
- 缺乏鲁棒性：输入偏离预期时容易失败
- 难以学习：无法从数据中自动获取规则
- 维护困难：规则库膨胀后难以保持一致性
- 无法处理新情况：规则未覆盖即无法推理

**最终衰落：**
- 1980年代后期问题逐渐暴露
- 专家系统维护成本远超预期
- 实用性有限，难以大规模推广
- 1990年代被统计学习方法逐步取代
